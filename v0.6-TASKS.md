### 决策：删除 dnt/npm 发布管线

**理由**：
1. JSR 自动提供 npm 兼容层（`npm install @lessjs/core` 通过 JSR mirror 工作）
2. dnt 的 `postBuild()` 路径问题是已知痛点（相对路径混乱）
3. 5 个 `_build_npm.ts` 维护成本高（版本同步、依赖映射、类型声明）
4. 零 npm 直接消费者证据——Deno 用户用 JSR，Node 用户用 JSR npm mirror
5. `package.json` 保留仅作 TS 编辑器支持，不发布到 npm registry
