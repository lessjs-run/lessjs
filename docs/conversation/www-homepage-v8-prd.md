## 2. 核心叙事策略

### 2.1 LessJS 的核心故事（一句话）

> **LessJS 是唯一用浏览器原生 DSD 实现零 JS 首屏、同时让 Lit/React/Vanilla 组件在同一页面共存运行的全栈框架。**

关键词：**原生能力**（不是模拟）、**零 JS**（不是接近零）、**多框架共存**（不是多框架支持）、**全栈**（不是纯前端）。

### 2.2 访问者心理路径

目标用户是**前端架构师/全栈开发者**，他们在寻找 Next.js 的替代方案。心理路径应该是：

```
"这是什么？"       →  Hero
"凭什么零 JS？"    →  Code Strip（技术证明）
"别人做不到吗？"   →  Benchmark（竞争数据）
"最独特的是什么？" →  Multi-framework（差异尖峰）
"完整系统什么样？" →  Bento Grid（全景图）
"我能上手吗？"     →  Quick Start（行动终点）
```

为什么这个顺序：**Multi-framework 是 LessJS 唯一具备、所有竞品都不具备的差异化能力**。它必须出现在 Benchmark（证明不比别人差）之后、Bento（全景）之前——作为"竞争数据之上的差异尖峰"。

### 2.3 区块重排序

| 旧顺序 (v7)       | 新顺序 (v8)            | 节奏类型       | 理由                     |
| ----------------- | ---------------------- | -------------- | ------------------------ |
| ① Hero            | ① Hero                 | **暗色沉浸**   | 开场，精简化             |
| ② Code Strip      | ② Code Strip           | **暗色连续**   | 紧接 Hero，无缝证明零 JS |
| ③ Bento Grid      | ③ Benchmark            | **浅色叙事 A** | 竞争数据建立信任         |
| ④ Benchmark       | ④ Multi-framework      | **浅色叙事 B** | 差异尖峰，最独特卖点     |
| ⑤ Multi-framework | ⑤ Bento Grid           | **浅色叙事 C** | 全景收束，展示完整系统   |
| ⑥ Quick Start     | ⑥ Quick Start + Footer | **暖灰谢幕**   | 行动号召 + 安静离开      |

### 2.4 节奏改革：打破单调

v7 六个区块完全相同的节奏（`sec-divider → sec-lbl → sec-title → sec-bd`），滚动体验像读 PPT。v8 改革：

```
Hero          ─── 无分割线，全屏沉浸式
  ↓ (暗→暗，连续滚入)
Code Strip    ─── 无分割线，暗色延续
  ↓ (暗→浅，戏剧性翻转)
Benchmark     ─── 仅用品牌色细线分割，不用 divider
  ↓ (浅→浅，微变化)
Multi-fw      ─── 不用分割线，用交互 Tab 自身作为视觉边界
  ↓ (浅→浅，微变化)
Bento Grid    ─── 品牌色光晕分割
  ↓ (浅→暖灰，色温过渡)
Quick Start   ─── 渐变背景过渡到暖灰
  ↓ (暖灰→暖灰，连续)
Footer        ─── 自然衔接
```

关键原则：**禁用 `sec-divider` 统一分割线**。每个区块的顶部过渡是区块设计的一部分，不是通用的"分隔符组件"。

## 3. 区块级详细设计

### 3.1 区块 ① Hero ——「一句话说清楚」

**目的**：3 秒内回答"LessJS 是什么"和"为什么我要继续看"。

**视觉基调**：暗色沉浸（`#26215C → #111127` 渐变），品牌色呼吸光晕。

**内容精简**（v7 → v8）：

| 保留           | 移除/降级                                  |
| -------------- | ------------------------------------------ |
| Logo + 品牌名  | ~~4 个 stat 数字条~~（移到 Benchmark）     |
| 一行核心定位语 | ~~less-term-demo~~（移到 Code Strip 下方） |
| 双 CTA 按钮    | ~~描述段落~~（改为单行短句）               |

**Hero 新内容结构**：

```
┌─ Logo + "LessJS" ─────────────────────────────┐
│                                                │
│  全栈框架 · 零 JS 首屏 · 多框架共存              │  ← 一行定位语，品牌色渐变高亮
│                                                │
│  DSD 原生渲染，浏览器零 JS 看到完整页面          │  ← 副标题，一行，rgba(255,255,255,0.45)
│                                                │
│  [开始使用 →]  [理解定位]                       │  ← CTA 不变
│                                                │
│  (背景: 呼吸光晕动画)                           │

```

**关键交互**：

- 呼吸光晕动画（已有，保留 8s ease-in-out）
- H1 中"零 JS 首屏"和"多框架共存"用品牌色渐变高亮
- CTA hover 上浮 + 阴影增强（已有）

**过渡到下一区块**：无分割线，暗色背景平滑延续到 Code Strip。

### 3.2 区块 ② Code Strip ——「眼见为实的零 JS」

**目的**：用代码对比证明"零 JS 首屏"不是口号，是浏览器原生能力。

**视觉基调**：暗色连续（`#111127 → #0f0f1a`），与 Hero 无缝衔接。

**内容保留**（v7 优秀设计，不改）：

- 左栏：你的组件源码（LitElement）
- 右栏：SSG 输出的 DSD HTML
- "0 KB JS" 品牌色徽章
- macOS 三色窗口装饰

**关键交互**：

- 滚动进入时，右侧 DSD 代码块中的 `<template shadowrootmode="open">` 行短暂高亮（1.5s 品牌色脉冲）
- "0 KB JS" 徽章呼吸动画（2s 周期）

**与前后区块的过渡**：

- **前**：Hero 暗色 → Code Strip 暗色，无分割线，通过代码对比区的视觉重量自然区分
- **后**：Code Strip 暗色 → Benchmark 浅色，**戏剧性翻转**——用一道品牌色渐变线（`#534AB7 → transparent`）作为"幕布升起"的隐喻

### 3.3 区块 ③ Benchmark ——「零就是零，无法追平」

**目的**：用竞争数据建立信任——"LessJS 不是比竞品好一点，是在一个竞品无法追平的维度上完胜"。

**视觉基调**：浅色叙事（`#ffffff`），用数据可视化创造视觉锚点。

**内容增强**（v7 → v8）：

```

│                                                 │
│  首屏 JS：零就是零                               │  ← 标题

│  ┌── 水平条形图（已有，保留）─────────────────┐   │
│  │ LessJS    ████ 0 KB                       │   │
│  │ Astro     ████ 0 KB                       │   │
│  │ Fresh     ████████████ ~23 KB             │   │
│  │ Next.js   ████████████████████████ ~90 KB │   │
│  └──────────────────────────────────────────┘   │
│                                                 │
│  ★ Unlike Astro, LessJS achieves zero-JS       │
│    without a "no-JS-by-default" policy —        │
│    DSD is a browser primitive, not a convention.│
│                                                 │
│  ┌──────────────┐ ┌──────────────┐             │
│  │ v0.19.0      │ │ 681 tests    │             │  ← Hero 移来的 stat
│  │ 最新版本      │ │ 全部通过      │             │

│  零 JS 首屏是浏览器原生能力，工程优化无法追平。    │
└─────────────────────────────────────────────────┘
```

**关键交互**：

- 条形图宽度动画（已有，进入视口时触发）
- 新增：stats 数字进入视口时计数动画（0 → 目标值，1.5s ease-out）
- 新增：Astro 对比说明用小字 callout，hover 展开完整解释

**新增内容**（与 v7 差异）：

- **从 Hero 移来的 4 个 stat 卡片**放在 Benchmark 底部——它们是"信任数据"的一部分，不应在 Hero 抢占注意力
- **Astro 对比说明**：LessJS 和 Astro 都是 0 KB，但方式根本不同——Astro 是"默认不加 JS"的策略约定，LessJS 是浏览器原生 DSD 解析。这一行是 LessJS 对 Astro 的核心防御。

**与前后区块的过渡**：

- **前**：品牌色渐变线（`#534AB7 → transparent`）标记暗→浅的翻转
- **后**：无分割线，直接进入 Multi-framework

### 3.4 区块 ④ Multi-framework ——「任意框架，同一个 Island」

**目的**：展示 LessJS 的唯一差异化能力——Lit/React/Vanilla 组件在同一页面上共存运行。这是**全页的情感高点**。

**视觉基调**：浅色叙事，但通过交互式 showcase 组件的视觉丰富度自然成为页面焦点。这是唯一一个不需要 section label 的区块——让交互本身说话。

**内容保留**（v7 优秀设计）：

- `less-showcase-panel` 组件（已有，保留）
- 三 Tab 切换：Shoelace(Lit) / React 19 / Media Chrome(Vanilla)
- 适配器说明 + 安装命令

**内容增强**（v7 → v8）：

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  任意框架，同一个 Island                          │  ← 标题（无 section label）
│                                                 │
│  ┌──────────────────────────────────────────┐   │
│  │  [◉ Lit/Shoelace] [○ React 19] [○ Vanilla] │ ← Tab 栏
│  ├──────────────────┬───────────────────────┤   │
│  │                  │                       │   │
│  │  真实交互组件     │  源代码               │   │
│  │  (showcase)      │  (code block)         │   │
│  │                  │                       │   │
│  ├──────────────────┴───────────────────────┤   │
│  │ @lessjs/adapter-lit — DSD 原生渲染       │   │
│  │ npm i @lessjs/adapter-lit               │   │
│  └──────────────────────────────────────────┘   │
│                                                 │
│  一行解释：浏览器只加载当前 Tab 组件的 JS，       │
│  其余框架的组件保持 DSD 零 JS 静态渲染。          │
└─────────────────────────────────────────────────┘
```

**关键交互**：

- Tab 切换时，左侧 showcase island 平滑过渡（已有的 `less-showcase-panel`）
- 新增：Tab 切换时，右侧代码块有微弱的品牌色流光（200ms）
- 新增：右下角加一个微型信息图标，hover 显示"此组件由真实 Lit/React/Vanilla 代码渲染，非 iframe 模拟"

**为什么这个区块不需要 section label**：
Multi-framework 是展示区，不是阅读区。Label 是阅读导航工具——这里用户在做的是"玩"交互，不是"读"内容。省掉 label，让展示组件撑满视觉空间，自然成为页面焦点。

**与前后区块的过渡**：

- **前**：Benchmark 直接滚入，无分隔线
- **后**：品牌色光晕分割（`radial-gradient` 细线），暗示"展示结束，回到阅读"

### 3.5 区块 ⑤ Bento Grid ——「一个产品，三重能力」

**目的**：在用户被 Multi-framework 震撼后，给出完整的产品全景。三支柱不再平等——WC 渲染引擎是核心。

**视觉基调**：浅色叙事（`#ffffff`），但通过卡片视觉层级打破"三卡平等"。

**核心改变：建立视觉层级**：

```
v7（三卡平等）：               v8（WC Engine 主导）：
┌──────────┐ ┌─────┐         ┌─────────────────────┐
│          │ │  D  │         │                     │
│    F     │ ├─────┤         │   WC 渲染引擎 (D)    │  ← 宽卡，品牌色背景
│          │ │  I  │         │   DSD · Island ·     │
└──────────┘ └─────┘         │   Lit/React/Vanilla  │
                              └─────────────────────┘
                              ┌──────────┐┌─────────┐
                              │ 全栈框架  ││Registry │  ← 两张小卡
                              │  (F)     ││ Hub (I) │
                              └──────────┘└─────────┘
```

**设计规范**：

- **WC 渲染引擎卡片（主导）**：
  - 全宽或 2fr 宽度，品牌色淡背景 `rgba(83,74,183,0.04)`
  - 卡片内部增加 3 个子要点：DSD 渲染 / Island 架构 / 多框架适配器
  - 左侧品牌色竖线（2px）作为视觉锚点
  - Hover：品牌色竖线发光 + 边框变品牌色

- **全栈框架卡片 + Registry Hub 卡片（辅助）**：
  - 并排两张小卡，保持现有样式
  - Hover 保留现有效果

**内容精简**：

- 每张卡片 1 行标题 + 1 行描述（已有，保留）
- WC Engine 卡片新增 3 个 pill 标签：`DSD` `Island` `Multi-adapter`

**与前后区块的过渡**：

- **前**：品牌色光晕分割线进入
- **后**：无分割线，浅色背景通过微妙的渐变过渡到暖灰

## 8. 一句话总结

**v8 不改组件、不改技术栈、不改数据，只改一件事：叙事顺序和视觉节奏。** 把 Multi-framework 提前到竞争数据之后作为差异尖峰，把 Bento 后移到全景收束，让 WC Engine 在 Bento 中视觉主导，打破 6 区块统一分割线节奏——用过渡设计本身讲述"暗→浅→暖灰"的三幕故事。
