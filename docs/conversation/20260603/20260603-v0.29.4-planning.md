# 2026-06-03 v0.29.4 Planning

## Context

v0.29.3 resolved CodeQL alerts. Two remaining architecture debts were identified:

1. `callComponent()` in `render-ir.ts` misses prototype pollution filtering — SSR's `injectProps()` already has `DANGEROUS_KEYS` guard, IR path doesn't.
2. `renderDsd()` returns HTML string, forcing `mergeDsdHostHtmlWithLightDom` to do fragile string manipulation on HTML.

## Decision

Fix both in v0.29.4. This closes architecture debt before moving to JamFullStack features.

After v0.29.4, the rendering pipeline is architecturally complete:
- One IR (`RenderNode`)
- One serializer (`serializeRenderNode`)
- One render path (`renderToNode`, async-only)
- One public API (`renderDsdTree`)
- Trust boundaries are explicit
- No string manipulation on rendered output

## What Changes

### callComponent prototype pollution

```diff
for (const [key, value] of Object.entries(props)) {
+  if (DANGEROUS_KEYS.has(key)) continue;
  (instance as Record<string, unknown>)[key] = value;
}
```

### renderDsd → RenderNode

```diff
// Before: renderToNode calls renderDsd, gets HTML string,
//   then does fragile string manipulation to insert light DOM
const dsdResult = await renderDsd(tagName, { componentClass, props });
return mergeDsdHostHtmlWithLightDom(dsdResult.html, tagName, childNodes);

// After: lightDom passed to renderDsd, handled at IR level
const dsdResult = await renderDsd(tagName, {
  componentClass, props, lightDom: childNodes,
});
return trustedHtmlNode(dsdResult.html);

// mergeDsdHostHtmlWithLightDom — DELETED
```

## Post v0.29.4

Architecture debt is closed. Next target: JamFullStack (ISR, Edge KV, RPC maturation, Data Layer).
