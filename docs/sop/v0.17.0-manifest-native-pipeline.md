# SOP: v0.17.0 — Manifest-Native Pipeline

> Purpose: eliminate `PackageIslandMeta` and the backward-compat bridge,
> making the entire pipeline consume `LessPackageManifest` natively.
> This is the v0.16.1 plan plus the compat removal, merged into one
> breaking release.
>
> Release type: **minor (breaking internal API)**.
> No user-facing config changes — only package authors who imported
> `PackageIslandMeta` directly need to update.

## Goal

1. **Delete `PackageIslandMeta`** — the 3-field legacy type is gone.
2. **Delete `packageIslandFromManifest()`** — no bridge needed.
3. **adapter-vite consumes `LessPackageManifest` directly** — reads
   `manifest.declarations[].less` for tagName / module / hydrate.
4. **@lessjs/ui exports `manifest` only** — no `islands` array export.
5. **SSR-aware island rendering** — manifest flags drive render behavior.
6. **Build report integration** — `dsd-report.json` includes manifest decisions.

## Entry Conditions

- v0.16.0 `LessPackageManifest` + `LessRegistry` released on main.
- All CI gates green (fmt / lint / typecheck / test / build / e2e).

### Scope 1: Delete `PackageIslandMeta` + bridge

Remove from `packages/core/src/types.ts`:

- `PackageIslandMeta` interface (currently `@deprecated`)

Remove from `packages/core/src/registry.ts`:

- `packageIslandFromManifest()` function

Remove from `packages/core/src/index.ts`:

- `PackageIslandMeta` type export
- `packageIslandFromManifest` value export + `packageIslandFromManifest` alias

Remove from `packages/core/__tests__/package-manifest.test.ts`:

- Tests that verify `packageIslandFromManifest` behavior
  (these were testing the bridge; the bridge is gone)

### Scope 2: @lessjs/ui removes `islands` export

Current `packages/ui/src/index.ts`:

```ts
import type { PackageIslandMeta } from '@lessjs/core';
import { packageIslandFromManifest } from '@lessjs/core';
export const islands: PackageIslandMeta[] = packageIslandFromManifest(manifest);
```

After:

```ts
// Only export the manifest — consumers derive what they need
export { manifest } from './manifest.js';
```

Any consumer that previously did `import { islands } from '@lessjs/ui'`
must now read `manifest.declarations` directly. adapter-vite will be
updated in Scope 3.

### Scope 3: adapter-vite manifest-native pipeline

**Current flow** (v0.16.0):

```
@lessjs/ui → islands: PackageIslandMeta[]
  → adapter-vite reads PackageIslandMeta[]
    → entry-descriptor.ts maps to IslandDecl[]
```

**New flow** (v0.17.0):

```
@lessjs/ui → manifest: LessPackageManifest
  → adapter-vite reads manifest.declarations
    → filters by less.ssr / less.dsd / less.module
    → maps to IslandDecl[] internally
```

Key changes per file:

**`build-context.ts`**:

- `packageIslands: PackageIslandMeta[]` → `packageManifests: LessPackageManifest[]`
- Add `packageIslandDecls: IslandDecl[]` (pre-computed from manifests)

**`route-scanner.ts`**:

- `scanPackageIslands()` → `scanPackageManifests()`
- Returns `LessPackageManifest[]` instead of `PackageIslandMeta[]`
- `validatePackageIslandMeta()` → `validateAndExtractIslands()`
  - Calls `LessRegistry.validate()` then extracts `IslandDecl[]`

**`entry-descriptor.ts`**:

- Remove `packageIslands?: PackageIslandMeta[]` option
- Add `packageManifests?: LessPackageManifest[]` option
- Internal: extract `IslandDecl[]` from manifest declarations:
  ```ts
  manifest.declarations
    .filter((d) => d.less?.module)
    .map((d) => ({
      tagName: d.tagName,
      modulePath: d.less!.module!,
      isPackage: true,
      hydrate: d.less?.hydrate,
      ssr: d.less?.ssr,
      dsd: d.less?.dsd,
    }));
  ```

**`IslandDecl`** (entry-descriptor.ts):

- Add optional `hydrate?: string`, `ssr?: boolean`, `dsd?: boolean`
  fields to carry manifest metadata through to the render pipeline.

**`index.ts`** (adapter-vite):

- `packageIslands: PackageIslandMeta[]` → `packageManifests: LessPackageManifest[]`
- `scanPackageIslands()` call → `scanPackageManifests()`
- All downstream calls updated.

**`build-client.ts`**:

- Read `IslandDecl[]` from pre-computed context instead of `PackageIslandMeta[]`

**`build-ssg.ts`**:

- Read `IslandDecl[]` from context; use `ssr`/`dsd` flags for render decisions.

**`entry-renderer.ts`**:

- Read `IslandDecl[]` for SSR-aware rendering logic.

### Scope 4: SSR-aware package island rendering

Render behavior driven by manifest flags:

| `less.ssr` | `less.dsd` | Behavior                                          |
| ---------- | ---------- | ------------------------------------------------- |
| `true`     | `true`     | Include in SSR bundle, render with `renderDSD()`  |
| `true`     | `false`    | Include in SSR bundle, pure-island path           |
| `false`    | —          | Skip SSR, client bundle only                      |
| undefined  | undefined  | Default: `ssr: true, dsd: true` (backward compat) |

Implementation in `build-ssg.ts`:

- When generating the SSR entry, filter `IslandDecl[]` by `ssr !== false`
  to decide which packages enter the SSR bundle.
- Pass `dsd` flag to `renderDSD()` options.

### Scope 5: Build report + client entry integration

- `dsd-report.json` includes per-package-island render decisions
  (ssr/dsd flags, hydration strategy).
- Client entry generation uses CEM metadata (events, slots, attributes)
  from manifest declarations where available.
- Invalid manifests fail early (before code generation).
