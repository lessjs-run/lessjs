# SOP: v0.18.2 - `less add` Safe Install Flow

> Purpose: make package installation, registration, and rendering setup
> repeatable while preserving the v0.18 admission rules.
>
> Release type: **minor patch (CLI + project mutation)**.

## Entry Criteria

- `less validate-manifest` exists and returns stable JSON.
- Compatibility tiers are stable enough for local project changes.
- v0.17.4 SSR admission planning prevents accidental SSR imports.

## Step-by-Step Execution

### Step 1: Implement Dry Run First

Command:

```sh
less add @scope/wc-package --dry-run
```

Dry run must print:

- package source and version
- manifest path used
- compatibility tier
- tags to register
- generated config changes
- warnings and rejected tags

Dry run must not change files.

### Step 2: Resolve And Validate Package

Resolve package from:

- local path
- JSR package
- npm package

Then run the v0.18.1 validator. Stop immediately if validation fails.

### Step 3: Update Project Files Safely

Only after validation passes:

- update LessJS config
- add package to package island registry
- generate or update client registration entry
- update lockfile through the package manager when needed

Every file mutation must be listed before it happens.

### Step 4: Preserve Rollback Safety

If any write fails:

- stop immediately
- print changed files
- print recovery command or manual recovery steps
- do not continue to build

### Step 5: Build Integration

After install:

- SSR-capable tags enter SSR admission plan
- client-only tags enter client registration only
- rejected tags never enter generated files

## Verification

- [ ] `less add ./fixtures/ssr-capable --dry-run` changes no files
- [ ] `less add ./fixtures/client-only --dry-run` reports client-only outcome
- [ ] invalid package fails before file writes
- [ ] real install updates expected files only
- [ ] rerunning install is idempotent
- [ ] uninstall/manual rollback instructions are printed when needed
- [ ] `deno task fmt && deno task lint && deno task typecheck`
- [ ] `deno task test`
- [ ] `deno task build`

## Exit Criteria

- Users can add a validated package without manual file edits.
- The command never treats unknown packages as SSR-capable.
- All generated changes are explainable and reviewable.

## Non-Goals

- Do not create a public Hub yet.
- Do not add publisher auth.
- Do not implement package signing.
- Do not bypass validation for convenience.
