# SOP: v0.19.1 - Hub Previews + Quality Gates

> Purpose: turn Hub records into useful user-facing package evidence: previews,
> quality signals, and install warnings.
>
> Release type: **minor patch (Hub product quality)**.

## Entry Criteria

- v0.19.0 Hub can ingest validation/build artifacts.
- v0.18 reports include compatibility tier and skip reasons.
- Snapshot rendering only runs for admitted packages/components.

## Step-by-Step Execution

### Step 1: Define Quality Score Inputs

Start with transparent signals:

- compatibility tier
- validation errors and warnings
- SSR/SSG render success
- bundle size
- hydration mode
- duplicate tag risk
- security audit result
- documentation/readme availability

Do not invent opaque rankings.

### Step 2: Generate Preview Artifacts

For each admitted component:

- render SSR/SSG snapshot
- capture visual preview when browser tooling is available
- store artifact path in Hub record
- skip client-only components unless a client preview runner exists

### Step 3: Show Install Warnings

Package detail pages must show:

- whether `less add` is safe
- whether component is SSR-capable or client-only
- what config changes will be generated
- known risks before install

### Step 4: Add Regression Checks

Track changes across package versions:

- compatibility tier changed
- bundle size increased
- render errors increased
- security status worsened

## Verification

- [ ] Hub page distinguishes SSR-capable, client-only, and rejected packages
- [ ] preview is absent when component did not pass admission
- [ ] quality score shows underlying reasons
- [ ] install warning matches `less validate-manifest` output
- [ ] regression check catches a compatibility downgrade fixture
- [ ] `deno task fmt && deno task lint && deno task typecheck`
- [ ] `deno task test`
- [ ] `deno task build`

## Exit Criteria

- Users can decide whether a package is safe to install from Hub evidence.
- Hub does not imply SSR support where validation did not prove it.
- v1.0 has a credible registry evidence model to freeze against.

## Non-Goals

- Do not add paid marketplace flows.
- Do not add publisher signing unless v0.19.0 proves ingestion quality.
- Do not execute unknown packages directly in the Hub request path.
