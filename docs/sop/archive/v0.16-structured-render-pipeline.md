# SOP: v0.16 Structured Render Pipeline

> Implementation guide for v0.16.0 release.
> Cross-session executable — read this file to resume work.
>
> Driven by ADR-0026. Completes deferred items from ADR-0025.

## Overview

v0.16 completes the Renderer Protocol by making `renderDSD()` return
`RenderOutput`, wiring `RenderHooks` into the pipeline, writing `dsd-report.json`
during SSG builds, enabling package island SSR registration via hooks, adding
pre-publish version checks, and establishing Playwright E2E baseline.

## Phase 1: `renderDSD()` → `Promise<RenderOutput>`

### 1.1 Change return type

File: `packages/core/src/render-dsd.ts`

```ts
// Before:
export async function renderDSD(...): Promise<string>

// After:
export async function renderDSD(...): Promise<RenderOutput>
```

At the end of the function, construct and return `RenderOutput`:

```ts
return {
  html: resultHtml,
  errors: collectedErrors,
  metrics: collector?.getReport() ?? emptyMetrics,
  hydrationHints: collectedHints,
};
```

### 1.2 Update `render-nested.ts`

- `renderNestedCustomElements()` must propagate errors from child renders
  up to the parent `RenderOutput.errors` array.
- Hydration hints from nested components also propagate up.

### 1.3 Update all callers

| File                                 | Change                                    |
| ------------------------------------ | ----------------------------------------- |
| `adapter-vite/src/cli/ssg-render.ts` | `const { html } = await renderDSD(...)`   |
| `adapter-vite/src/entry-renderer.ts` | Generated SSR code destructure `out.html` |
| `adapter-lit/src/ssr.ts`             | Lit pipeline uses `out.html`              |
| `core/__tests__/render-dsd.test.ts`  | Update all assertions                     |

### 1.4 Verify

- [ ] `deno task typecheck` passes
- [ ] All existing tests updated and passing
- [ ] No callers still assume `string` return

## Phase 2: RenderHooks Integration

### 2.1 Add `hooks` parameter to `renderDSD()`

File: `packages/core/src/render-dsd.ts`

```ts
export async function renderDSD(
  ...existing params...,
  hooks?: RenderHooks,
): Promise<RenderOutput>
```

### 2.2 Wire hooks into pipeline

```
beforeRender(input)  → before component instantiation
  ↓ instantiate + inject props
  ↓ adapter.render() or instance.render()
  ↓ renderNestedCustomElements()
  ↓ serialize + wrap
afterRender(output)   → after serialization complete
onError(error)       → on any RenderError (from any phase)
```

### 2.3 Write tests

- `packages/core/__tests__/render-hooks.test.ts`
  - beforeRender fires before instantiation
  - afterRender fires with full RenderOutput
  - onError fires on instantiation/render/serialize errors
  - Hooks are optional (undefined = no-op)

### 2.4 Verify

- [ ] All hook tests pass
- [ ] Pipeline behavior unchanged when hooks are undefined
- [ ] `deno task typecheck` passes

## Phase 3: DSD Report Output

### 3.1 Write `dsd-report.json` in SSG

File: `adapter-vite/src/cli/ssg-render.ts`

After all pages are rendered, aggregate `RenderOutput` results:

```ts
const report: DsdBuildReport = {
  version: 1,
  timestamp: new Date().toISOString(),
  totalPages: renderedPages.length,
  totalErrors: allErrors.length,
  errors: allErrors,
  metrics: aggregateMetrics(allMetrics),
};
await Deno.writeTextFile(
  join(outputDir, 'dsd-report.json'),
  JSON.stringify(report, null, 2),
);
```

### 3.2 Define `DsdBuildReport` type

File: `packages/core/src/types.ts`

```ts
interface DsdBuildReport {
  version: 1;
  timestamp: string;
  totalPages: number;
  totalErrors: number;
  errors: RenderError[];
  metrics: DsdReport;
}
```

### 3.3 Write integration test

- `packages/adapter-vite/__tests__/ssg-report.test.ts`
- SSG build produces `dsd-report.json` with expected structure

### 3.4 Verify

- [ ] `dsd-report.json` generated after SSG build
- [ ] Report contains correct error counts and metrics

## Phase 4: Package Islands SSR via RenderHooks

### 4.1 Add `beforeRender` hook in adapter-vite

File: `adapter-vite/src/entry-renderer.ts`

Generate SSR entry code that provides a `beforeRender` hook:

```ts
const hooks: RenderHooks = {
  beforeRender(input) {
    if (!customElements.get(input.tagName)) {
      // Lazy import + register package island
      // e.g., import('@openelement/ui/counter-island.js')
    }
  },
};
```

### 4.2 Build package island registry

Create a mapping of package island tagNames to their module paths, generated
at build time by `EntryDescriptor.islands` where `isPackage === true`.

### 4.3 Write tests

- Package island component renders in SSR
- `beforeRender` hook fires and registers the component
- Fallback: component not found → `RenderError` with `recoverable: false`

### 4.4 Verify

- [ ] Package island renders in SSR via hook
- [ ] Local islands still work without hooks (no regression)
- [ ] `deno task typecheck` passes
