### Implementation Details

#### Scope 1: SSR filtering in entry-renderer.ts

Changed the SSR island filter from:

```ts
const ssrIslands = desc.islands.filter((island) => !island.isPackage);
```

to:

```ts
const ssrIslands = desc.islands.filter((island) => !island.isPackage && island.ssr !== false);
```

Islands with `ssr === false` are excluded from SSR customElements.define()
registration. When `__ssr()` encounters an unregistered element, it outputs
an empty `<tag></tag>` placeholder, which gets upgraded client-side.

#### Scope 2: ManifestDecision type in core

Added `ManifestDecision` interface to `core/src/types.ts`:

```ts
interface ManifestDecision {
  tagName: string;
  packageName: string;
  ssr: boolean;
  dsd: boolean;
  hydrate?: string;
  renderPath: 'ssr+client' | 'client-only';
}
```

Extended `DsdBuildReport` with optional `manifestDecisions?: ManifestDecision[]`.

#### Scope 3: dsd-report manifest decisions

Added `buildManifestDecisions(ctx)` helper in `ssg-render.ts`:

- Reads `ctx.phase1.packageIslandDecls` + `ctx.phase1.packageManifests`
- Maps each island's manifest flags to a `ManifestDecision`
- `ssr !== false` â†’ `renderPath: 'ssr+client'`
- `ssr === false` â†’ `renderPath: 'client-only'`
- Returns `[]` when no ctx or no package island decls
