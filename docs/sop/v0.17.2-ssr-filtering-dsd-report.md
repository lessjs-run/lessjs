# SOP: v0.17.2 — SSR Filtering + dsd-report Manifest Decisions

> Purpose: implement the two deferred v0.17.0 SOP items — SSR-aware island
> filtering and manifest-driven render decisions in dsd-report.json.
>
> Release type: **patch (feature completion, no breaking change)**.

## Goal

1. **SSR filtering**: Package islands with `less.ssr: false` appear only in
   the client bundle, not in the SSR render output.
2. **dsd-report.json manifest decisions**: The build report includes per-island
   render decisions derived from manifest flags (ssr, dsd, hydrate).

### Scope 1: SSR filtering in build-ssg.ts

When generating the SSR entry, filter `IslandDecl[]` by `ssr !== false` to
decide which packages enter the SSR bundle:

```ts
const ssrDecls = packageIslandDecls.filter((d) => d.ssr !== false);
const clientOnlyDecls = packageIslandDecls.filter((d) => d.ssr === false);
```

- `ssrDecls` → included in SSR virtual entry + render output
- `clientOnlyDecls` → included in client island bundle only

### Scope 2: SSR filtering in entry-renderer.ts

When rendering a route, skip SSR rendering for islands with `ssr === false`:
emit a placeholder `<template>` or custom element tag without inner HTML.

### Scope 3: dsd-report manifest decisions

Add `manifestDecisions` field to `DsdBuildReport`:

```ts
interface ManifestDecision {
  tagName: string;
  packageName: string;
  ssr: boolean;
  dsd: boolean;
  hydrate?: string;
  renderPath: 'ssr+client' | 'client-only';
}

interface DsdBuildReport {
  // ... existing fields ...
  manifestDecisions: ManifestDecision[];
}
```
