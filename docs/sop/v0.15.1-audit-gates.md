# SOP: v0.15.1 Audit Gates

> Purpose: make the v0.15 renderer kernel safe to extend before v0.16 package
> protocol work starts.
>
> Release type: patch / hardening.

## Goal

v0.15.1 closes the audit blockers that would make v0.16 unsafe:

- unsafe package island metadata reaching generated JavaScript;
- package discovery depending on arbitrary package entry execution;
- generated service worker cache rules that are too broad;
- tests that can pass without checking the intended condition.

This version does not add the WC Package Protocol. It makes the current
`PackageIslandMeta` path strict enough to survive the migration.

## Scope

### 1. Safe package island code generation

Files:

- `packages/adapter-vite/src/entry-generators.ts`
- `packages/adapter-vite/src/route-scanner.ts`

Required changes:

- Generate tag names and module specifiers with `JSON.stringify()`.
- Add `validatePackageIslandMeta()` or an equivalent validator.
- Reject invalid `tagName`, unsafe `modulePath`, line breaks, control
  characters, quote breaking, and unsupported protocols.
- Add negative tests for malicious `tagName` and `modulePath`.

### 2. Static package discovery boundary

File:

- `packages/adapter-vite/src/route-scanner.ts`

Required changes:

- Keep `import(pkg).islands` only as trusted backward compatibility.
- Document that third-party packages must move toward static manifests:
  `custom-elements.json`, `lessjs.manifest.json`, or explicitly trusted
  manifest modules.
- Do not base future `less add` behavior on executing arbitrary package entry
  code.

### 3. Service worker cache boundary

File:

- `packages/adapter-vite/src/cli/ssg-render.ts`

Required changes:

- Cache only GET requests.
- Skip `/api/`, `/rpc/`, auth/session paths, and authorization-bearing requests.
- Cache only static destinations or known build assets.
- Add tests for API, POST, and authenticated requests.

### 4. False-positive test cleanup

Files:

- `www/__tests__/build-output.test.ts`
- `packages/create/__tests__/cli.test.ts`
- `packages/adapter-vite/__tests__/entry-generators.test.ts`
- `packages/adapter-vite/__tests__/build-manifest.test.ts`

Required changes:

- Make `www/__tests__/build-output.test.ts` check `www/dist`.
- Fail clearly if `www/dist` does not exist.
- Replace boolean `assertExists(...)` with `assert(...)` or
  `assertEquals(condition, true)`.
- Add a repo check rejecting new `assertExists(<boolean expression>)`.

## Exit Criteria

- [x] No unsafe package island metadata can reach generated JS.
- [x] Generated SW does not cache API/auth/user-specific responses.
- [x] Build-output tests fail when `www/dist` is missing.
- [x] The current v0.15 behavior remains backward compatible.

## Decision

v0.15.1 must be completed before starting any public v0.16 registry or
one-command install work.
