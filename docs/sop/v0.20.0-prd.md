## 1. 产品目标

将 LessJS 从"Lit + DSD"混合架构升级为**原生优先的 Ocean-Island 架构**：

- **海洋（DSD 组件）**：零框架依赖，纯原生 HTMLElement，SSR 渲染 Shadow DOM
- **岛屿（Pure Island）**：按需选框架（Lit/FAST/Preact/React），仅服务客户端交互
- **CSS 栈**：Open Props（设计 token）+ CSSStyleSheet（组件样式）+ CSS Parts（外部定制），三层极简原生栈

**一句话**：LessJS 的海洋是封装好的 Web Components，而其他框架的海洋只是裸 HTML。

## 2. 当前痛点

### 2.1 Lit 与 DSD 的架构冲突

```
Lit 的核心假设：render() 是 DOM 的唯一真相源
DSD 的现实：    <template shadowrootmode> 已渲染最终 DOM

结果：
  - render() 在 DSD 下返回 nothing（hack）
  - @click 在 SSR 时被剥离，用 hydrateEvents 替代
  - Lit 的 DOM diffing 完全不触发
  - Lit 的模板引擎在 DSD 路径下完全绕过
```

10 个 UI 组件中，9 个 DSD 组件的 Lit 能力利用率极低——只有 `css` tagged template 和 `static properties` 真正被使用。

### 2.2 框架兼容性伪命题

当前 `@openelement/adapter-lit` 将 DSD 逻辑焊死在 Lit 上：

- 想用 FAST 写 DSD 组件 → 必须 extends DsdLitElement → 强制依赖 Lit
- 想用 Preact 写 DSD 组件 → 同上
- 这不是"兼容"，是被**绑架**

### 2.3 CSS 方案混乱

三种样式方案并存：

- Lit `css` tagged template（实际是 adoptedStyleSheets）
- 内联 cssText（less-search overlay）
- CSS 变量（Open Props 内联在 design-tokens.ts）

问题：风格不统一，内联 cssText 不可复用。

### 2.4 渲染管线混乱

```
当前：Lit render() → adapter-lit.interpolate() → renderDsd() → HTML
问题：
  - Lit 的模板解析是多余的（DSD 下不需要）
  - 适配器既要处理 Lit 又要处理 DSD，职责不清
```

## 3. 目标架构

### 3.1 Ocean-Island 模型

```
┌──────────────────────────────────────────────────────┐
│                    v0.20.0 架构                        │
├──────────────────────────────────────────────────────┤
│                                                       │
│  海洋（~80% 页面内容）                                  │
│  ┌─────────────────────────────────────────────────┐ │
│  │  DsdElement (原生 HTMLElement)                    │ │
│  │  ├── render(): string → SSR 直接输出 DSD HTML    │ │
│  │  ├── hydrateEvents → 声明式事件绑定               │ │
│  │  ├── CSSStyleSheet + adoptedStyleSheets          │ │
│  │  ├── attributeChangedCallback → 手动 DOM 同步    │ │
│  │  └── ::part() + :state() → 标准外部 API          │ │
│  │                                                  │ │
│  │  零 Lit 依赖，零框架开销                          │ │
│  └─────────────────────────────────────────────────┘ │
│                                                       │
│  岛屿（~20% 页面内容）                                  │
│  ┌─────────────────────────────────────────────────┐ │
│  │  Pure Island                                     │ │
│  │  ├── Lit/LitElement → 需要 reactivity 时用       │ │
│  │  ├── FAST/FASTElement → 同上                     │ │
│  │  ├── Preact → 同上                                │ │
│  │  └── 策略: eager / lazy / visible / idle         │ │
│  │                                                  │ │
│  │  按需选框架，各框架适配器仅服务 Island SSR        │ │
│  └─────────────────────────────────────────────────┘ │
│                                                       │
│  CSS 栈（全原生）                                       │
│  ┌─────────────────────────────────────────────────┐ │
│  │  @property          — CSS 变量类型安全 (Houdini)  │ │
│  │  CSS Custom Props   — 设计 token                  │ │
│  │  CSSStyleSheet      — 组件样式（一次解析全局共享）│ │
│  │  CSS Parts ::part() — 外部定制 API                │ │
│  │  :state()           — 组件状态伪类                │ │
│  └─────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

### 3.2 渲染管线

```
海洋:
  DsdElement.render(): string
    → renderDsd()
    → <template shadowrootmode="open">（浏览器原生解析）
    → DsdElement.hydrateEvents()
    → 可交互 ✅

岛屿:
  LitElement.render() / FASTElement.template
    → Adapter（仅 Island SSR）
    → renderDsd()
    → 空壳 custom element
    → 客户端框架接管
    → 可交互 ✅
```

## 5. 验收标准

### P0 验收

- [ ] `npm pack` 体积：UI 包（不含 Lit）< 15KB gzip
- [ ] SSR 输出：所有 DSD 组件 HTML 与当前一致
- [ ] 水合行为：hydrateEvents 全部正常工作
- [ ] CSS Parts：`::part()` 在所有组件上可用
- [ ] 零回归：现有页面视觉和行为不变

### P1 验收

- [ ] Lit 依赖：DSD 组件零 Lit import
- [ ] Bundle 报告：对比 v0.19.0 减小 ≥ 6KB gzip
- [ ] 组件数量：10/10 组件迁移完成（含 hero-ping 保留 Lit）

## 6. 风险与依赖

| 风险                                | 概率 | 缓解                           |
| ----------------------------------- | ---- | ------------------------------ |
| `less-layout`（1200行）迁移复杂度高 | 高   | 分步迁移，先属性和事件，后样式 |
| Lit SSR 适配器与 DsdElement 集成    | 中   | 适配器仅服务 Island，互不干扰  |
| CSSStyleSheet 兼容性                | 低   | 97%+ 浏览器支持                |
| @property 兼容性（Safari）          | 中   | 降级到普通 CSS 变量            |
