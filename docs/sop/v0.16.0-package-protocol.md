# SOP: v0.16.0 WC Package Protocol

> Purpose: introduce the CEM-compatible LessJS package manifest and local
> registry without yet requiring build-time SSR-aware package integration.
>
> Release type: minor protocol release.

## Goal

v0.16.0 turns the current 3-field `PackageIslandMeta` into a real package
protocol:

- `LessPackageManifest`
- CEM-compatible component metadata
- LessJS SSR/DSD/hydration declarations
- `LessRegistry`
- validation and index generation

This is the first version where LessJS can describe WC packages as data instead
of only executing package code.

## Scope

### 1. Manifest type system

Files:

- `packages/core/src/types.ts`
- `packages/core/src/index.ts`

Required types:

- `LessPackageManifest`
- `LessModule`
- `LessDeclaration`
- `LessElementExtensions`
- `LessPackageExtensions`
- CEM-compatible attributes, members, events, slots, css properties, css parts
- `LessExport`

### 2. Backward compatibility adapter

File:

- `packages/core/src/types.ts` or `packages/core/src/registry.ts`

Required function:

```ts
packageIslandFromManifest(manifest: LessPackageManifest): PackageIslandMeta[]
```

Rules:

- Preserve existing `@openelement/ui` package island behavior.
- Mark old `PackageIslandMeta` path as deprecated, not removed.
- Removal target: v0.18 or later.

### 3. `@openelement/ui` manifest

Files:

- `packages/ui/src/manifest.ts`
- `packages/ui/src/index.ts`

Required behavior:

- Export `manifest`.
- Generate or preserve `islands` from `packageIslandFromManifest(manifest)`.
- Include enough CEM metadata for docs and validation to consume.

### 4. Local registry

File:

- `packages/core/src/registry.ts`

Required API:

```ts
register(manifest: LessPackageManifest): void
getByTagName(tagName: string): LessDeclaration | undefined
getAll(): LessPackageManifest[]
validate(manifest: LessPackageManifest): ValidationResult
generateIndex(): RegistryIndex
```

Validation must reject:

- missing required fields;
- invalid custom element names;
- unsafe module paths;
- invalid layer or hydrate strategy;
- duplicate tag names inside a package;
- unresolved export declaration references;
- registry-level duplicate tag names unless explicitly allowed.

- [ ] `packages/core/__tests__/registry.test.ts`
- [ ] `@openelement/ui` generated `islands` matches previous behavior.

## Exit Criteria

- v0.16.0 can ship even if build-time SSR-aware package rendering is deferred.
- Package metadata is static, validated, and tool-consumable.
- LessJS has a credible local registry foundation for future hub work.
