# SOP: v0.15.2 RenderOutput + RenderHooks

> Purpose: finish the v0.15 renderer protocol items that were deferred from
> ADR-0025.
>
> Release type: minor-compatible internal protocol hardening.

## Goal

v0.15.2 makes the renderer kernel return structured data instead of a raw HTML
string. This is required before package manifests can declare SSR behavior in
v0.16.

## Scope

### 1. Change `renderDsd()` return type

File:

- `packages/core/src/render-dsd.ts`

Required change:

```ts
export async function renderDsd(...): Promise<RenderOutput>
```

The returned object includes:

- `html`
- `errors`
- `metrics`
- `hydrationHints`

### 2. Update all callers

Files:

- `packages/adapter-vite/src/cli/ssg-render.ts`
- `packages/adapter-vite/src/entry-renderer.ts`
- `packages/adapter-lit/src/ssr.ts`
- `packages/core/src/render-nested.ts`
- `packages/core/__tests__/render-dsd.test.ts`

Required changes:

- Consumers use `out.html`.
- Nested custom element rendering propagates child errors and hydration hints.
- Tests assert the full `RenderOutput` shape.

### 3. Wire `RenderHooks`

Files:

- `packages/core/src/render-dsd.ts`
- `packages/core/src/types.ts`

Required behavior:

- `beforeRender(input)` runs before instantiation.
- `afterRender(output)` runs after serialization.
- `onError(error)` runs for classified render errors.
- Hooks are optional and must not change behavior when omitted.

## Verification

- [ ] `deno task typecheck`
- [ ] `deno task test packages/core`
- [ ] No caller still assumes `renderDsd()` returns `string`.
- [ ] Hook tests cover success, failure, and undefined hooks.

## Exit Criteria

- `RenderOutput` is the stable renderer return contract.
- `RenderHooks` is real pipeline behavior, not only a type definition.
- v0.16 manifest work can rely on structured SSR output.
