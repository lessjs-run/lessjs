# SOP: v0.19.0 — Platform + Hub

> Purpose: build the public Registry Hub on top of the Universal WC Engine.
> Hub lives in `packages/hub/` within the main monorepo.
>
> Release type: **minor (new subsystem, backward compatible)**.

## Goal

1. **Hub API** — public package registry with search, browse, and metadata.
2. **Snapshot previews** — per-package SSR/SSG rendering for visual preview.
3. **Bundle cost analysis** — KB per component, per-package.
4. **Security audit** — dependency scanning and CVE marking.
5. **Edge runtime** — at least 2 edge runtimes supported.
6. **Scoped Custom Element Registries** — multi-version tag isolation.

### Scope 1: Hub API

Hono-based API service deployed on Deno Deploy:

```
GET  /api/packages              — list packages
GET  /api/packages/:name        — package detail
GET  /api/packages/:name/:version — version detail
GET  /api/search?q=button       — search
POST /api/packages              — publish (auth required)
```

Storage: Deno KV or SQLite via `packages/hub/db/`.

### Scope 2: Hub Web UI

Simple browse + search interface:

- Package list with SSR preview thumbnails
- Package detail page with manifest, bundle cost, security status
- Search with quality score ranking

### Scope 3: Snapshot previews

Uses the Universal WC Engine (v0.18) to render each package's components:

- Run `renderDSD()` for each declaration
- Capture screenshot with headless browser
- Store as static assets in Hub storage

### Scope 4: Bundle cost analyzer

For each package declaration:

- Client JS size (KB gzipped)
- SSR HTML size
- Hydration cost
- Dependency tree

### Scope 5: Security audit

- Scan `deno.lock` / `package-lock.json` for known CVEs
- Mark packages with security issues
- Report dependency depth and transitive risk

### Scope 6: Edge runtime adapters

- `@lessjs/adapter-deno-deploy` — Deno Deploy specific
- `@lessjs/adapter-cloudflare` — Cloudflare Workers
- Common interface: `EdgeAdapter` extends `RendererProtocol`

### Scope 7: Scoped Custom Element Registries

Browser API: `new CustomElementRegistry()` per scope.
LessJS integration:

- `@lessjs/core` scoped registry API
- Build-time scope assignment per package
- Runtime scope isolation

### Scope 8: Publisher auth + package signing

- Deno Deploy OAuth for publisher accounts
- Package signing with Ed25519
- Verify signature on install

## Key Risk

Hub is not a pure code project — it involves:

- Web infrastructure (Deno Deploy, storage, CI/CD)
- Ongoing operational cost
- Security policy and governance
- Community moderation

The monorepo approach keeps development tight, but deployment may require
separate CI pipelines for the Hub service vs the framework packages.
