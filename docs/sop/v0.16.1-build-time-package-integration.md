# SOP: v0.16.1 Build-Time Package Integration

> Purpose: connect `LessPackageManifest` to the build pipeline after the
> protocol and validation layer are stable.
>
> Release type: minor-compatible integration release.

## Goal

v0.16.1 makes package manifests affect build behavior:

- package island registration uses manifest metadata;
- SSR-aware package islands can enter the SSR/SSG path;
- client island manifests use CEM metadata;
- generated code still follows the v0.15.1 safety rules.

## Entry Conditions

- v0.15.1 audit gates are complete.
- v0.15.2 `RenderOutput` and `RenderHooks` are complete.
- v0.15.3 `dsd-report.json` and release gate are complete.
- v0.16.0 `LessPackageManifest` and `LessRegistry` are complete.
- `LessRegistry.validate()` rejects unsafe module paths and invalid tag names.
- `www/__tests__/build-output.test.ts` validates `www/dist`.

## Scope

### 1. Adapter reads manifests from packages

Files:

- `packages/adapter-vite/src/route-scanner.ts`
- `packages/adapter-vite/src/build-client.ts`
- `packages/adapter-vite/src/entry-renderer.ts`

Required behavior:

- Prefer static manifest entry points.
- Keep trusted legacy `islands` fallback.
- Register manifests with `LessRegistry`.
- Populate package island metadata from manifest declarations.

### 2. SSR-aware package island rendering

Rules:

- `less.ssr: true` and `less.dsd: true`: include in SSR bundle and render
  with `renderDSD()`.
- `less.ssr: true` and `less.dsd: false`: include in SSR bundle, but use the
  pure-island path.
- `less.ssr: false`: skip SSR and include only in the client bundle.

### 3. Client entry generation

Required behavior:

- Use CEM metadata for events, slots, attributes, parts, and tokens where
  currently available.
- All generated import specifiers and tag maps use safe literal generation.
- Invalid manifests fail before code generation.

### 4. Build report integration

Required behavior:

- `dsd-report.json` includes package island render decisions.
- Failed package SSR is reported with actionable errors.
- Client-only fallback is explicit in the report.

## Verification

- [ ] Package island with `less.ssr: true` appears in SSG output.
- [ ] Package island with `less.ssr: false` appears only in client output.
- [ ] Invalid manifest fails before generated code is emitted.
- [ ] Local islands are unaffected.
- [ ] `deno task test`
- [ ] `deno task build`
- [ ] `deno task test:e2e`

## Exit Criteria

- v0.16.1 proves that the package protocol can drive real rendering behavior.
- If this integration is unstable, it should move to v0.17 instead of blocking
  v0.16.0.
