# SOP-003: Runtime and App Facades

> Version: v0.23.0\
> Priority: P0\
> Status: PLANNED\
> Depends on: SOP-001, SOP-002

## Objective

Separate two different user-facing conveniences:

- component authoring imports;
- application configuration imports.

## Current Problem

`@lessjs/app` correctly owns `lessjs()` configuration composition, but
component authoring convenience still tends to fall back to `@lessjs/core`.
That pulls `core` toward a batteries-included facade instead of a runtime
kernel.

## Target Contract

Use separate facades:

```text
@lessjs/runtime
  authoring facade for DsdElement, html, StyleSheet, signal, computed, effect

@lessjs/app
  configuration facade for lessjs(), content, i18n, and adapter-vite assembly

@lessjs/core
  runtime kernel, not the product facade
```

The exact package name can change, but the responsibilities must stay separate.

## Procedure

### Step 1: Decide Whether to Add `@lessjs/runtime`

- [ ] Audit generated templates, docs, UI components, and examples for
      component authoring imports.
- [ ] Decide whether `@lessjs/runtime` is worth the extra package.
- [ ] If rejected, document the chosen alternative and keep `core` small.

Acceptance:

- [ ] The decision is recorded in the v0.23 implementation notes.
- [ ] The authoring import path is consistent across docs and templates.

### Step 2: Define Runtime Facade Exports

- [ ] Export `DsdElement`, `html`, `unsafeHTML`, and template types.
- [ ] Export `StyleSheet` from its canonical owner.
- [ ] Export signal helpers from `@lessjs/signals`.
- [ ] Avoid exporting Vite, content, i18n, or build contracts.

Acceptance:

- [ ] The facade has no Vite dependency.
- [ ] The facade does not create a publish cycle.

### Step 3: Keep `@lessjs/app` Configuration-Focused

- [ ] Keep `lessjs()` as the recommended Vite config entry.
- [ ] Keep `content` and `i18n` composition in `@lessjs/app`.
- [ ] Do not move runtime component APIs into `@lessjs/app`.
- [ ] Ensure `@lessjs/app` imports contracts from the contracts package, not
      adapter internals.

Acceptance:

- [ ] `vite.config.ts` generated by `@lessjs/create` remains small.
- [ ] `@lessjs/app` remains the only recommended config facade.

### Step 4: Migrate Templates and Docs

- [ ] Update `@lessjs/create` templates.
- [ ] Update README quickstart examples.
- [ ] Update package READMEs where imports change.
- [ ] Keep compatibility examples for one minor release if needed.

Acceptance:

- [ ] Fresh generated projects build locally and from JSR.
- [ ] Public docs do not recommend importing implementation packages for
      ordinary app authoring unless that is deliberate.

## Exit Criteria

- App configuration and component authoring are separate concepts.
- `@lessjs/core` is not forced to be the DX barrel.
- Generated templates teach the intended architecture.
