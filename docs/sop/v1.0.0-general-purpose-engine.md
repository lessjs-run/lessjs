# SOP: v1.0.0 — General-Purpose Engine

> Purpose: LessJS becomes the rendering kernel that any WC ecosystem can plug
> into. API freeze — no breaking changes without major version bump.
>
> Release type: **major (API freeze, stability guarantee)**.

## Goal

1. **Zero-config SSR/SSG** for any CEM manifest WC package.
2. **Plugin system** — custom adapters, validators, renderers.
3. **API freeze** — stable public API with backward compatibility guarantee.
4. **Plugin marketplace** — hosted on Hub.

## Scope

| # | Item                            | Location                    | Est.   |
| - | ------------------------------- | --------------------------- | ------ |
| 1 | API audit + freeze              | All packages                | 2 sess |
| 2 | Plugin system interface         | `packages/core/`            | 3 sess |
| 3 | Full a11y audit integration     | `packages/hub/audit/`       | 2 sess |
| 4 | Performance regression tracking | `packages/hub/analyzer/`    | 1 sess |
| 5 | Plugin marketplace (Hub)        | `packages/hub/marketplace/` | 2 sess |
| 6 | Comprehensive docs + migration  | `docs/`, `www/`             | 2 sess |

### Scope 1: API audit + freeze

- Audit all exported types and functions across 10 packages
- Mark unstable APIs with `@experimental` or remove
- Document stability guarantees per package
- Breaking changes only with major version bump

### Scope 2: Plugin system

```ts
interface LessPlugin {
  name: string;
  adapters?: RendererAdapter[];
  validators?: ManifestValidator[];
  renderers?: CustomRenderer[];
  hooks?: PluginHooks;
}
```

- Register via `lessjs({ plugins: [myPlugin] })`
- Access to build context, manifest registry, render pipeline

### Scope 3: Full a11y audit

- axe-core integration for rendered components
- Per-component a11y score in dsd-report.json
- WCAG 2.1 AA compliance reporting

### Scope 4: Performance regression tracking

- Per-package-version render time benchmarks
- Bundle size delta tracking
- Memory usage profiling
- Automated regression detection in CI

### Scope 5: Plugin marketplace

- Browse and install plugins via Hub
- Plugin quality score (tests, docs, maintenance)
- Plugin compatibility matrix (LessJS version × plugin version)

### Scope 6: Comprehensive docs + migration

- v1.0 migration guide from v0.x
- API reference (auto-generated from types)
- Getting started for npm, bun, and Deno users
- Architecture deep-dive

## Key Risk

API freeze is a commitment. Once v1.0 ships, any breaking change requires
v2.0. This means:

- Must audit all exports carefully before freeze
- `@experimental` APIs can change but must be clearly marked
- Some features may need to ship as `@experimental` in v1.0 and stabilize in v1.1
