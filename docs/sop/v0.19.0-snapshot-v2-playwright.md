# SOP: v0.19.0 Phase 3 — Real Browser Snapshot Rendering

> Purpose: Replace happy-dom snapshot rendering with Playwright-based real
> browser rendering for Hub component previews.
>
> **One-shot delivery** — replace the entire happy-dom pipeline in one pass.
>
> ADR: [ADR-0032](../adr/0032-real-browser-snapshot-rendering.md)

## Strategy

Replace the fragile happy-dom simulation with real Chromium rendering via Playwright.
The existing `renderSnapshotLit()` for @lessjs/ui SSR-capable components stays unchanged;
only the client-only npm package path (Shoelace, Media Chrome) switches from happy-dom
to Playwright.

**Why now?** The happy-dom approach has accumulated 7+ distinct bugs (type incompatibilities,
slot extraction, adoptedStyleSheets, global pollution, :host replacement, Media Chrome
isShimmed, subprocess overhead). Each fix adds complexity. The pattern is clear:
DOM simulation will always be a leaky abstraction. A real browser eliminates
the entire class of problems.

## Entry Criteria

- v0.19.0 Phase 2 complete (component detail pages, less add CLI)
- Playwright + Chromium available in CI (already true for e2e tests)
- `renderSnapshotLit()` working for @lessjs/ui components (existing, no changes)

## Step-by-Step Execution

### Step 1: Create `snapshot-playwright.ts` — Core Renderer

Implement the Playwright-based snapshot renderer:

```typescript
// packages/hub/src/snapshot-playwright.ts

interface PlaywrightRenderOptions {
  /** Package import specifier (e.g. '@shoelace-style/shoelace/dist/components/alert/alert.js') */
  importSpec: string;
  /** Custom element tag name */
  tagName: string;
  /** Demo attributes to set on the element */
  demoAttrs?: Record<string, string>;
  /** Demo slot content (innerHTML) */
  demoSlots?: string;
  /** Package theme CSS to inject */
  themeCss?: string;
  /** Timeout in ms (default 5000) */
  timeout?: number;
}

interface PlaywrightRenderResult {
  html: string; // Serialized component HTML
  shadowHtml: string; // Shadow DOM inner HTML (with styles)
  success: boolean;
  error?: string;
}
```

**Core flow**:

1. Launch Chromium browser (or reuse existing instance)
2. For each component, create a new page
3. Serve inline fixture HTML via temp HTTP server
4. Navigate to fixture page
5. Wait for component to render (`waitForSelector` + settle time)
6. `page.evaluate()` to:
   - Serialize `adoptedStyleSheets` → `<style>` tags
   - Capture `shadowRoot.innerHTML`
   - Extract slot assignments from light DOM
   - Replace `<slot>` with fallback content
7. Post-process: sanitize, wrap in snapshot-preview container
8. Close page

**Files**:

- `packages/hub/src/snapshot-playwright.ts` — New, ~200 lines

### Step 2: Fixture HTML Generation

Generate inline HTML pages that import and instantiate each component:

```html
<!DOCTYPE html>
<html>
<head>
  <script type="module">
    import '@shoelace-style/shoelace/dist/components/alert/alert.js';
  </script>
  <style>
    /* Shoelace theme variables */
    :root { ... }
  </style>
</head>
<body>
  <sl-alert open variant="primary">This is an alert message.</sl-alert>
</body>
</html>
```

Key decisions:

- **No file I/O**: fixture HTML generated in-memory, served via temp HTTP server
- **One page per component**: no cross-component interference
- **`DEMO_ATTRS` + `DEMO_SLOTS` reused**: same config maps as happy-dom,
  just applied in fixture HTML instead of JavaScript manipulation

**Function**: `generateFixtureHtml(options: FixtureOptions): string`

### Step 3: Temp HTTP Server

A lightweight `Deno.serve` instance to serve fixture pages:

```typescript
// Start temp server
const controller = new AbortController();
const server = Deno.serve({
  port: 0, // random available port
  signal: controller.signal,
  handler: (req) => {
    const url = new URL(req.url);
    // Serve fixture page
    if (url.pathname === '/fixture') {
      return new Response(fixtureHtml, {
        headers: { 'Content-Type': 'text/html' },
      });
    }
    // Proxy node_modules for module resolution
    // (Deno's file:// resolution works for npm: specifiers)
    return new Response('Not found', { status: 404 });
  },
});
```

**Why a server?** ES module imports need HTTP(S) URL resolution. Bare specifiers
like `@shoelace-style/shoelace/...` need the Deno resolver to work, which only
applies to `file://` or `http://` imports served through Deno's module system.

**Alternative considered**: Serve from `file://` directly. Problem: browsers
block `file://` module imports for security. Must use HTTP.

### Step 4: Shadow DOM Serialization via `page.evaluate()`

The key insight: **let the real browser do the work**.

```typescript
const result = await page.evaluate((tagName: string) => {
  const el = document.querySelector(tagName);
  if (!el || !el.shadowRoot) return null;

  // 1. Serialize adoptedStyleSheets → <style> tags
  const styleTags: string[] = [];
  for (const sheet of el.shadowRoot.adoptedStyleSheets) {
    const rules = Array.from(sheet.cssRules).map((r) => r.cssText).join('\n');
    if (rules) styleTags.push(rules);
  }

  // 2. Get shadow DOM HTML
  const shadowHtml = el.shadowRoot.innerHTML;

  // 3. Get light DOM slot assignments
  const slotMap: Record<string, string> = {};
  for (const child of el.childNodes) {
    if (child.nodeType === 1) {
      const slotName = (child as Element).getAttribute('slot') || 'default';
      slotMap[slotName] = (child as Element).outerHTML;
    } else if (child.nodeType === 3) {
      const text = child.textContent?.trim();
      if (text) slotMap['default'] = (slotMap['default'] || '') + text;
    }
  }

  return { shadowHtml, styleSheets: styleTags, slotMap };
}, tagName);
```

**Why `page.evaluate()`?** This runs JavaScript **inside the real browser**. We
get the ACTUAL DOM state, including:

- Correct shadow DOM rendering (no simulation gaps)
- Properly resolved `adoptedStyleSheets` (no manual conversion)
- Accurate slot assignments (both Element and Text nodes)
- No type incompatibilities (it's JavaScript, not TypeScript)

### Step 5: Post-Processing

After capturing the raw HTML from the browser:

1. **Insert style sheets** — prepend `<style>` tags from adoptedStyleSheets
   into shadow DOM HTML
2. **Replace `<slot>` with fallback content** — same regex-based approach as
   current code, but now the slot content comes from the real browser
3. **Sanitize** — apply existing `sanitizeSnapshot()` to strip `<script>`,
   `on*` handlers, `javascript:` URLs
4. **Wrap** — `<div class="snapshot-preview">themeCss<tagName>shadowHtml</tagName></div>`
5. **Replace `:host`** — same as current approach (regex replace to tagName)

### Step 10: Tests

Write integration test for the Playwright renderer:

```typescript
// packages/hub/__tests__/snapshot-playwright.test.ts

Deno.test('playwright: renders sl-button with variant', async () => {
  const result = await renderWithPlaywright({
    importSpec: '@shoelace-style/shoelace/dist/components/button/button.js',
    tagName: 'sl-button',
    demoAttrs: { variant: 'primary' },
    demoSlots: 'Click me',
  });
  assert(result.success, 'Render should succeed');
  assert(result.html.includes('sl-button'), 'Should contain tag name');
  assert(result.html.includes('Click me'), 'Should contain slot content');
  assert(!result.html.includes('<script'), 'Should not contain scripts');
});
```
