# LessJS v0.19.0 Phase 6 — SSG Resilient Rendering + Visual Overhaul

## 0. SOP 目标

修复 SSG 阶段 client-only 组件渲染崩溃（72 DSD errors → 0），并将整站视觉品质
从"通用文档模板"提升到"精密·克制·有深度"的品牌水准。

**范围限定**：v0.19.0 Phase 6。不涉及 v0.20+ 路线图，不涉及 ADR-0034 hermetic
migration（esm.sh → local server），不涉及新功能开发。

**核心原则**：

1. 不允许 SSG 构建产出破损 HTML（shadow DOM 为空的 `<template>` 标签）
2. client-only 组件在 SSG 阶段 **只输出裸标签**，不做 shadow DOM 渲染
3. Hub 预览必须在视觉上还原组件真实外观（iframe 隔离）
4. 视觉改动仅涉及 CSS 变量 + page-styles + UI 组件样式，不改变组件行为

**关联文档**：

- ADR-0035（本 SOP 的架构决策依据）
- ADR-0032（Playwright snapshot 渲染）
- ADR-0034（hermetic snapshots，后续 Phase）

## Step 1: SSG Client-Only 组件分类修复

**目标**：Shoelace 组件在 SSG 阶段被跳过，不再尝试实例化和渲染 shadow DOM。

### Step 1.1: 确认 Hub client-only 标签注入路径

**文件**：`packages/adapter-vite/src/entry-descriptor.ts`

**检查**：`buildSsrAdmissionPlan()` 中 `clientOnlyTags` 的生成逻辑。目前只从 route
islands 中收集。需要增加一条路径：从 Hub scanner 数据中读取 `client-only` 标签。

**实现**：

1. 在 SSG build 启动时，读取 `www/app/routes/registry/_hub-data.ts`
   （或直接读 Hub scanner 的 `WC_PACKAGES` 列表）
2. 提取所有 `compatibility: 'client-only'` 的 `tagName`
3. 注入到 `ssrAdmissionPlan.clientOnlyTags` 中

```ts
// 在 buildSsrAdmissionPlan() 末尾追加：
// Hub client-only tags — from registry data
try {
  const hubDataPath = resolve(cwd, 'www/app/routes/registry/_hub-data.ts');
  const hubData = await import(hubDataPath);
  // ... 提取 client-only tags
  for (const record of Object.values(hubData.records)) {
    if (record.compatibility === 'client-only') {
      for (const tag of record.tags) {
        if (!clientOnlyTags.includes(tag.tagName)) {
          clientOnlyTags.push(tag.tagName);
        }
      }
    }
  }
} catch { /* hub data not available — skip */ }
```

**文件**：`packages/adapter-vite/src/entry-renderer.ts`

**检查**：`__LESS_CLIENT_ONLY_TAGS__` 是否正确注入到 SSG bundle 的入口代码中。

## Step 2: renderDsd() 容错降级

**目标**：即使 `__LESS_CLIENT_ONLY_TAGS__` 未覆盖到某个组件（遗漏），`renderDsd()`
自身也有 try/catch 兜底，不会产出破损 HTML。

### Step 2.1: 修改 `renderDsd()` 的错误处理

**文件**：`packages/core/src/render-dsd.ts`

**当前行为**：实例化或 render() 抛错时，记录 error 到 `RenderOutput.errors`，但
`RenderOutput.html` 包含空的 `<template shadowrootmode="open"></template>`。

**修改为**：捕获到实例化/render 错误时，跳过 shadow DOM 输出，只返回裸标签：

```ts
// renderDsd() 内部，catch 块中：
catch (err) {
  // 记录错误
  errors.push({ phase: 'render', tagName, message: err.message, recoverable: true });

  // 降级：输出裸标签，不写 shadow DOM
  const attrs = Object.entries(props)
    .map(([k, v]) => v === true ? k : `${k}="${v}"`)
    .join(' ');
  const attrStr = attrs ? ` ${attrs}` : '';
  const lightDom = slotContent || '';

  return {
    html: `<${tagName}${attrStr}>${lightDom}</${tagName}>`,
    errors,
    metrics: { ... },
    hydrationHints: [],
  };
}
```

## Step 3: Hub Snapshot 数据重构 + iframe 预览

**目标**：组件详情页预览从 `unsafeHTML(dump)` 改为 `<iframe srcdoc="...">`，
Shoelace 组件完整渲染，CSS 隔离，数据瘦身。

### Step 3.1: 定义新的 snapshot 数据结构

**文件**：`packages/hub/src/schema.ts`

新增 `HubSnapshotMeta` 类型：

```ts
export interface HubSnapshotMeta {
  tagName: string;
  importSpec: string; // '@shoelace-style/shoelace'
  importUrl: string; // 'https://esm.sh/@shoelace-style/shoelace@2.20.1'
  demoAttrs: Record<string, string>;
  demoSlots: string;
  themeCssUrl?: string;
  compatibility: 'ssr-capable' | 'client-only';
}
```

### Step 3.2: 修改 HubTagRecord

**文件**：`packages/hub/src/schema.ts`

`HubTagRecord` 中 `ssrSnapshot?: string` 保留（向后兼容），新增 `snapshotMeta?: HubSnapshotMeta`。

scanner.ts 生成数据时同时填充两个字段：

- `ssrSnapshot`：保留给旧消费方（逐步废弃）
- `snapshotMeta`：新消费方使用

### Step 3.3: 修改组件详情页渲染

**文件**：`www/app/routes/registry/[package]/[component].ts`

替换 `unsafeHTML(sanitizeSnapshot(tag.ssrSnapshot))` 为 iframe 渲染逻辑：

```ts
// 如果有 snapshotMeta → 构建 srcdoc
if (tag.snapshotMeta) {
  const meta = tag.snapshotMeta;
  const attrs = Object.entries(meta.demoAttrs)
    .map(([k, v]) => v === '' ? k : `${k}="${v}"`)
    .join(' ');
  const srcdoc = `<!DOCTYPE html>
<html><head>
  <link rel="stylesheet" href="${meta.themeCssUrl || ''}">
  <script type="module" src="${meta.importUrl}"></script>
</head><body style="margin:0;padding:16px;font-family:system-ui">
  <${meta.tagName} ${attrs}>${meta.demoSlots}</${meta.tagName}>
</body></html>`;

  return html`
    <iframe
      srcdoc="${srcdoc}"
      style="width:100%;border:none;min-height:80px;background:#fff"
      @load="${this._resizeIframe}"
    ></iframe>
  `;
}
// Fallback: 旧 ssrSnapshot（兼容期）
if (tag.ssrSnapshot) {
  return html`
    <div style="width:100%">${unsafeHTML(sanitizeSnapshot(tag.ssrSnapshot))}</div>
  `;
}
```

### Step 3.4: iframe 自动高度适配

**文件**：`www/app/routes/registry/[package]/[component].ts`

添加 `_resizeIframe` 方法，在 iframe load 后读取内容高度并设置：

```ts
private _resizeIframe(e: Event) {
  const iframe = e.target as HTMLIFrameElement;
  try {
    const doc = iframe.contentDocument;
    if (doc?.body) {
      iframe.style.height = doc.body.scrollHeight + 32 + 'px';
    }
  } catch { /* cross-origin — skip */ }
}
```
