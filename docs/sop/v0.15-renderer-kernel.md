### 2.1 Create `render-errors.ts`

Extract from `render-dsd.ts`:

- `RenderError` interface
- `RenderOutput` interface
- `HydrationHint` interface
- `RenderInput` interface
- Error classification helper: `classifyRenderError(phase, tagName, error)`

File: `packages/core/src/render-errors.ts`

### 2.2 Create `render-instantiate.ts`

Extract from `render-dsd.ts`:

- Component instantiation logic (lines 122-139)
- Property injection loop with DANGEROUS_KEYS guard (lines 142-161)

File: `packages/core/src/render-instantiate.ts`

### 2.3 Create `render-serialize.ts`

Extract from `render-dsd.ts`:

- `camelToKebab()` function
- `serializeAttributes()` function
- `buildDsdTemplateAttrs()` function
- DSD template wrapping logic (pure-island vs dsd-static/dsd-interactive output)

File: `packages/core/src/render-serialize.ts`

### 2.4 Slim down `render-dsd.ts`

After extraction, `render-dsd.ts` becomes pipeline orchestration:

- Import from the 3 new modules
- `renderDsd()` calls: instantiate → render → nested → serialize → wrap
- `renderDSDByName()` unchanged
- `isLitTemplateResultHeuristic()` unchanged (internal helper)
