# SOP: v0.18.0 — Universal WC Engine

> Purpose: make LessJS a general-purpose WC SSR/SSG engine that can render
> any WC package with a CEM manifest, not just LessJS's own packages.
>
> Release type: **minor (new capability, backward compatible)**.

## Goal

1. **CEM manifest parser** — read standard `custom-elements.json` from any
   third-party WC package.
2. **Third-party WC SSR fallback strategies** — graceful degradation when
   packages don't support SSR natively.
3. **`less add` package discovery** — install, register, and render
   third-party WC packages.
4. **`less validate-manifest`** — validate any package's CEM manifest.

### Scope 1: CEM manifest parser

New module `packages/core/src/cem-parser.ts`:

```ts
interface CemManifest {
  schemaVersion: string;
  readme?: string;
  modules: CemModule[];
  declarations?: CemDeclaration[];
}

function parseCemManifest(json: unknown): CemManifest | ValidationError;
function cemToLessManifest(cem: CemManifest): LessPackageManifest;
```

The parser reads standard CEM format and converts to `LessPackageManifest`
with sensible defaults:

- No `less.ssr` field → default `ssr: true`
- No `less.dsd` field → default `dsd: true`
- No `less.hydrate` field → default to custom-element upgrade

### Scope 2: Third-party WC SSR fallback strategies

| Package capability             | Fallback strategy                           |
| ------------------------------ | ------------------------------------------- |
| CEM manifest, `less.ssr=true`  | Full SSR with `renderDSD()`                 |
| CEM manifest, `less.ssr=false` | Pure-island: client-only, no SSR HTML       |
| CEM manifest, no `less` field  | Conservative: pure-island + DSD detection   |
| No CEM manifest                | Legacy: `scanPackageIslands()` with warning |

### Scope 3: `less add @third-party/wc-button`

```sh
less add @shoelace-style/button
```

Steps:

1. Resolve package name → JSR or npm source
2. Look for `custom-elements.json` in package root
3. Parse CEM manifest → `LessPackageManifest`
4. Register in local `LessRegistry`
5. Generate import map entry
6. Print actionable next steps

### Scope 4: `less validate-manifest`

```sh
less validate-manifest ./custom-elements.json
```

Validates:

- CEM schema version
- Required fields present
- Custom element naming rules
- Module paths resolve correctly
- No duplicate tag names

### Scope 5: Integration tests

- Test CEM parser with Shoelace manifest fixture

## Key Risk

CEM manifest adoption varies widely across the WC ecosystem. Shoelace is the
primary validation target. Packages without CEM manifests will use the legacy
`scanPackageIslands()` path with a deprecation warning.
