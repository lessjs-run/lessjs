# SOP: v0.15.3 DSD Report + Release Gate

> Purpose: make SSG output observable and make the release gate repeatable.
>
> Release type: patch / release-readiness.

## Goal

v0.15.3 writes render diagnostics to build output and proves the renderer kernel
is stable enough to become the base for v0.16.

## Scope

### 1. Write `dsd-report.json`

File:

- `packages/adapter-vite/src/cli/ssg-render.ts`

Required behavior:

- Aggregate `RenderOutput.errors`, metrics, and hydration hints across rendered
  pages.
- Write `dsd-report.json` to the SSG output directory.
- Keep the report deterministic enough for tests.

### 2. Define build report type

File:

- `packages/core/src/types.ts`

Required fields:

- report version
- timestamp
- total pages
- total errors
- render errors
- metrics summary
- hydration hint summary

### 3. Add integration test

Suggested file:

- `packages/adapter-vite/__tests__/ssg-report.test.ts`

Required assertions:

- SSG build produces `dsd-report.json`.
- The report shape is valid.
- Error counts and metrics are not silently dropped.

### 4. Release gate

Required commands:

- `deno task fmt:check`
- `deno task lint`
- `deno task typecheck`
- `deno audit`
- `deno task test`
- `deno task build`
- `deno task test:e2e`

## Exit Criteria

- Renderer output is observable in SSG builds.
- CI/release can fail on render errors if project policy chooses to.
- v0.15.x can be tagged as the stable renderer-kernel foundation.
