# v0.24.2: Remove Old Component Model, Full JSX+Signal Migration

> **Released**: 2026-05-28
> **ADR**: ADR-0057
> **SOP**: v0.24.2

## Summary

v0.24.2 completes the migration started in v0.24.1 by removing the old `html` tagged template component model entirely. All components now use JSX + `static props` + Signal, with zero backward-compatibility overhead. Four production hotfixes were applied post-release to fix JSX infrastructure gaps.

## What Changed

### Removed

- `html` tagged template and all helpers (`classMap`, `when`, `choose`, `repeat`, `ref`, `unsafeHTML`)
- `@prop()` decorator
- `TemplateResult`, `PropertyOptions`, and related internal types from public API
- Template-specific rendering functions (`renderTemplateToString`, `applyRuntimeTemplateBindings`, etc.)

### Added

- JSX `IntrinsicElements` type declarations for TypeScript
- Signal auto-unwrap in `renderToString` (JSX expressions `{signal}` auto-resolve `.value`)
- `renderToDOM` integration in `_renderIntoShadowRoot` for proper event binding
- **`@lessjs/signals` as explicit dependency of `@lessjs/core`**: Core now imports `effect()` from signals for VNode signal tracking. Publish order adjusted: signals (5) тЖТ core (6).

### Fixed

#### Production Hotfixes (post-release)

| # | Bug | Root Cause | Fix |
|---|-----|-----------|-----|
| 1 | Island components render `[object Object]` | `build-client.ts` and `build-ssg.ts` internal `viteBuild()` calls use `configFile: false`, ignoring user's `vite.config.ts` esbuild JSX config. esbuild defaults to classic `React.createElement`, producing objects DsdElement can't read. | Added `esbuild: { jsx: 'automatic', jsxImportSource: '@lessjs/core' }` to both internal build configs. |
| 2 | SVG icons disappear after hydration | `renderToDOM` uses `document.createElement()` for all elements, placing SVG elements (circle, line, path, svg) in HTML namespace where browsers won't render them. | Added `SVG_TAGS` set (50+ elements) and `createElementForTag()` helper that routes SVG elements to `document.createElementNS()`. |
| 3 | Theme toggle doesn't switch / theme lost on navigation | `_renderIntoShadowRoot`'s VNode branch renders DOM but never subscribes to signals. TemplateResult path has `_subscribeTemplateSignals()`; VNode path had no equivalent. | Wrapped `render()` in an alien-signals `effect()` that auto-tracks signal accesses and re-renders DOM on changes. Disposed in `_disposeSignalSubscriptions()`. |
| 4 | `graph:check` fails: core imports signals without declaration | `dsd-element.ts` imports `effect` from `@lessjs/signals` but `packages/core/deno.json` didn't declare the dependency. | Added `@lessjs/signals` to core's imports; moved signals before core in CI publish order. |

#### Pre-release Fixes

- **VNode event binding**: `_renderIntoShadowRoot` previously used `renderToString` for VNode, which silently dropped `onClick`/`onInput` handlers. Now uses `renderToDOM` with `addEventListener`.
- **SSG package resolver `.tsx` subpath resolution**: `resolveVirtualLessPackageRelative` in `ssg-package-resolver.ts` blindly appended `.ts` to all relative imports, causing `.tsx` UI components to resolve to non-existent `.ts` files. Added `resolveSourcePathExtension` that consults `LESSJS_EXPORT_FILES`.
- **JSX DOM renderer signal child unwrap**: `renderToDOM` now auto-unwraps Signal values in children (matching `renderToString` behavior).
- **`www` stale `.js` search imports**: All route files and `_renderer.ts` updated to `.tsx`.
- **`@lessjs/runtime` missing `LessRenderer` export**: Added re-export.
- **Generated project build test alias drift**: Updated all UI component aliases to `.tsx` and added `jsx-runtime` subpath aliases.

## Migration Guide

### From `html` tagged templates to JSX

```tsx
// Before (v0.24.1, deprecated):
import { DsdElement, html } from '@lessjs/core';
class MyButton extends DsdElement {
  render() {
    return html`
      <button class="btn" @click="${this._handle}">Click</button>
    `;
  }
}

// After (v0.24.2):
import { DsdElement } from '@lessjs/core';
class MyButton extends DsdElement {
  render() {
    return <button className='btn' onClick={this._handle}>Click</button>;
  }
}
```

### From `@prop()` to `static props`

```tsx
// Before:
class MyComp extends DsdElement {
  @prop()
  count = 0;
}

// After:
class MyComp extends DsdElement {
  static props = { count: Number };
}
```

### From `classMap`/`when`/`repeat`

```tsx
// classMap({ active: isActive }) тЖТ className={isActive ? 'btn active' : 'btn'}
// when(cond, () => a, () => b) тЖТ {cond ? <A /> : <B />}
// repeat(items, (item) => x) тЖТ {items.map(item => <Item key={item.id} />)}
```

- **Build**: Vite/Rolldown transforms `.tsx` тЖТ LessJS VNode (not React)
