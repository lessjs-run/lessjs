# v0.24.2: Remove Old Component Model, Full JSX+Signal Migration

> **Released**: 2026-05-28
> **ADR**: ADR-0057
> **SOP**: v0.24.2

## Summary

v0.24.2 completes the migration started in v0.24.1 by removing the old `html` tagged template component model entirely. All components now use JSX + `static props` + Signal, with zero backward-compatibility overhead.

## What Changed

### Removed

- `html` tagged template and all helpers (`classMap`, `when`, `choose`, `repeat`, `ref`, `unsafeHTML`)
- `@prop()` decorator
- `TemplateResult`, `PropertyOptions`, and related internal types from public API
- Template-specific rendering functions (`renderTemplateToString`, `applyRuntimeTemplateBindings`, etc.)

### Added

- JSX `IntrinsicElements` type declarations for TypeScript
- Signal auto-unwrap in `renderToString` (JSX expressions `{signal}` auto-resolve `.value`)
- `renderToDOM` integration in `_renderIntoShadowRoot` for proper event binding

### Fixed

- **VNode event binding**: `_renderIntoShadowRoot` previously used `renderToString` for VNode, which silently dropped `onClick`/`onInput` handlers. Now uses `renderToDOM` with `addEventListener`.
- **SSG package resolver `.tsx` subpath resolution**: `resolveVirtualLessPackageRelative` in `ssg-package-resolver.ts` blindly appended `.ts` to all relative imports, causing `.tsx` UI components (`less-dialog`, `less-theme-toggle`, `less-layout`, etc.) to resolve to non-existent `.ts` files. Added `resolveSourcePathExtension` that consults `LESSJS_EXPORT_FILES` to preserve the exact source extension.
- **JSX DOM renderer signal child unwrap**: `renderToDOM` now auto-unwraps Signal values in children (matching `renderToString` behavior), preventing `[object Object]` from appearing in CSR when JSX expressions contain signals.
- **`www` stale `.js` search imports**: All route files and `_renderer.ts` still imported `less-search.js`, but the island file is `less-search.tsx`. Updated 6 import paths to `.tsx`.
- **`@lessjs/runtime` missing `LessRenderer` export**: Added `export type { LessRenderer }` so that `www/app/shared/section-renderer.ts` and `registry/_renderer.ts` can import the type from the authoring facade.
- **Generated project build test alias drift**: `packages/create/__tests__/cli.test.ts` local workspace aliases for `@lessjs/ui/*` components and `@lessjs/core/jsx-runtime` were still pointing to `.ts` files. Updated all UI component aliases to `.tsx` and added `jsx-runtime` / `jsx-dev-runtime` subpath aliases to prevent parent-alias ENOTDIR errors.

## Migration Guide

### From `html` tagged templates to JSX

```tsx
// Before (v0.24.1, deprecated):
import { DsdElement, html } from '@lessjs/core';
class MyButton extends DsdElement {
  render() {
    return html`
      <button class="btn" @click="${this._handle}">Click</button>
    `;
  }
}

// After (v0.24.2):
import { DsdElement } from '@lessjs/core';
class MyButton extends DsdElement {
  render() {
    return <button className='btn' onClick={this._handle}>Click</button>;
  }
}
```

### From `@prop()` to `static props`

```tsx
// Before:
class MyComp extends DsdElement {
  @prop()
  count = 0;
}

// After:
class MyComp extends DsdElement {
  static props = { count: Number };
}
```

### From `classMap`/`when`/`repeat`

```tsx
// classMap({ active: isActive }) → className={isActive ? 'btn active' : 'btn'}
// when(cond, () => a, () => b) → {cond ? <A /> : <B />}
// repeat(items, (item) => x) → {items.map(item => <Item key={item.id} />)}
```
