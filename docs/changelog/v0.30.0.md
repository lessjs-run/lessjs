# Changelog - v0.30.0

Release date: 2026-06-04

## Summary

v0.30.0 is the type safety cleanup release. Eliminates 3 `ban-types` suppressions, simplifies `render()` to VNode-only, removes `_SsrHTMLElementStub` and `happy-dom`, and deletes ~60 lines of dead code.

## Changes

### Phase 1: VNode.tag precise types

- Defined `ComponentFn` and `ComponentCtor` types replacing bare `Function` in VNode.tag.
- Added `isComponentCtor()` and `isComponentFn()` type guards.
- Updated all component call sites (`callComponent`, `renderToDom`, `event-hydration`) to use type guards.
- Eliminated 3 `ban-types` suppressions.

### Phase 3: render() → VNode | null

- Changed `render()` return type from `string | VNode` to `VNode | null`.
- Simplified `_renderIntoShadowRoot()` from 3 branches to 1.
- Simplified `_renderErrorFallback()` to always use `renderToDom()`.
- Changed `onRenderError()` return type to `VNode | null`.
- `renderDsd()` retains `typeof result === 'string'` path for backward compatibility.

### Phase 4: SSR stub removal + happy-dom cleanup

- Removed `_SsrHTMLElementStub` class (16 lines).
- Replaced with inline conditional base class + minimal `globalThis.HTMLElement` stub.
- Added SSR-safe DOM method overrides on `DsdElement`.
- Removed `happy-dom` from root `deno.json`.
- Deleted `dsd-element-hydration.test.ts` and `jsx-render-dom.test.ts`.

## Breaking Changes

- `render()` now returns `VNode | null` instead of `string | VNode`. All existing components already return JSX (VNode), so this is a zero-break change for the current codebase.
- `onRenderError()` now returns `VNode | null`.
- `_renderErrorFallback()` no longer uses `innerHTML` for string fallbacks.
