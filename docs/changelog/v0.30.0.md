# Changelog - v0.30.0

Release date: 2026-06-04

## Summary

v0.30.0 is the architecture contract freeze. It removes the remaining native
string-render compatibility path, makes `VNode | null` the only LessJS component
contract, adds an executable architecture gate, and aligns all 19 packages on
the 0.30.0 release line.

## Changes

### One public render contract

- `DsdComponent.render()` is now `VNode | null`.
- `DsdElement.render()` and current docs now describe `VNode | null` only.
- `renderDsd()` no longer accepts string returns from native components.
- Core no longer renders adapter-native template values through a fallback
  registry branch.
- The old `wrongTypeErrorHtml` compatibility helper was deleted.

### One renderer and trust boundary

- Native DSD output flows through the VNode-to-RenderNode serializer path.
- `rawHtml` was removed from the current API surface in favor of explicit
  `trustedHtml`.
- `innerHTML` is escaped as text by default; trusted HTML must be opted in.
- Current site usages were migrated to `trustedHtml` where the content pipeline
  already owns sanitization.

### Structured metadata and source-of-truth gates

- `route-scanner` now reads static route exports with the TypeScript AST.
- `@openelement/content` nav metadata extraction now uses TypeScript AST instead of
  source text splitting.
- `graph:check` validates package version consistency and internal
  `jsr:@openelement/*` ranges.
- `arch:check` byte-reads every tracked file and fails current-source regressions
  in render contract, trust boundary, duplicate helpers, production type escapes,
  TypeScript suppressions, and metadata parsing.

### Duplicate logic cleanup

- Hub CLI package record loading now uses `packages/hub/src/cli/shared.ts`.
- Hub snapshot placeholder rendering now uses
  `packages/hub/src/snapshot-placeholder.ts`.
- Adapter hydration event binding now uses `bindHydrateEvents()` from
  `@openelement/core`.
- `CompatibilityClassification` is canonical in `@openelement/core/types`.

### Adapter and docs cleanup

- `@openelement/adapter-vanilla` now registers style extraction only; it no longer
  exposes string render passthrough.
- The current architecture guide was rewritten for v0.30.0.
- The Lit migration guide was rewritten and cleaned of stale TemplateResult and
  string-render guidance.
- Current site guide snippets now use JSX/VNode examples and JSX event handlers.

### Release metadata

- All 19 packages were bumped to `0.30.0`.
- Internal `jsr:@openelement/*` ranges were bumped to `^0.30.0`.
- `www/app/data/version.ts` and Hub version constants were updated to 0.30.0.

## Breaking Changes

- Native LessJS components must return `VNode | null` from `render()`.
- `render(): string` is no longer accepted by core as a compatibility path.
- `rawHtml` is no longer the current trust-boundary API; use `trustedHtml` only
  for content that has already been sanitized or is otherwise trusted.
- `@openelement/adapter-vanilla` no longer exposes a string render passthrough.
