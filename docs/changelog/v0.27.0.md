# Changelog — v0.27.0

## Architecture

### ADR-0071: Single-Pass Render Nested DSD (a61c5ddb)

删除 `parse5` 依赖和 `renderNestedCustomElements`，将两遍树遍历（renderToString → parse5 重建 → renderNested）合并为一遍 VNode 遍历。`renderNestedDsd`（后改名 `renderDsdTree`）在遍历时内联渲染嵌套 Custom Elements。net -293 lines，-1 external dependency。

### ADR-0072: Unified Render API Surface (305cab7b)

将 `renderDsd` + `renderDsdByName` + `renderNestedDsd` 三个渲染函数合并为一个 `renderDsd(input, props?)`。`jsx`/`jsxDEV`/`jsxs`/`For`/`Show` 从根入口移除，保留在 `@lessjs/core/jsx-runtime` subpath。对标 React/Vue/SolidJS 的 SSR API 模式。

### ADR-0067: Ocean + Island Signal-Native Architecture (8145ca3c)

`data-signal`/`data-signal-html`/`data-signal-attr`/`data-signal-class` 标记 + `_hydrateSignals()` 替代脆弱的 `_walkAndBind`。CSR hydration 从位置匹配改为显式标记匹配。

### 统一 VNode 管线 (687526cf, 5b31b01b)

删除双渲染器 hack（`_layoutWorkaroundReRender`、`effectScope`），统一 SSR 和 CSR 的 VNode 处理路径。Signal identity 通过 `data-signal` 属性在 SSR HTML 中保留。

### 属性去品牌化 (687526cf)

`data-less-e` → `data-eid`，`__LESS_*` globals 移除。

## Bug Fixes

### SSR 渲染三 Bug

- **Sidebar 消失**：`less-layout` 未经过 `renderDsd()` — 修复：`__wrapAppShell` 改为 `renderDsd('less-layout')`
- **Search [object Object]**：`<dialog>` 中 signal computed 值被 `String()` 错误转换 — 改为 `<div>`
- **Search theme 不跟随**：`dialog::backdrop` 不继承 shadow DOM CSS 变量

### [object Object] 回归 (b097c9a9)

`__renderAppShell` 接收的 `routeNode`(VNode) 被字符串拼接转为 `[object Object]` — 修复：VNode 先经 `renderDsdTree()` 渲染

## Cleanup

### parse5 删除 (c6ed6a7f, a61c5ddb)

从 core deno.json、build-ssg.ts、create 模板、所有注释中彻底移除 parse5 引用。

### API 清理 (305cab7b, a750c4eb)

`renderDsdByName` 合并进 `renderDsd`；`jsx/jsxDEV/jsxs/For/Show` 移到 jsx-runtime subpath；`@lessjs/runtime` barrel 更新 subpath import。

### CI 修复 (1bbb1729, b021e8d9, 04ffdde0)

Lint 警告、格式化、测试期望值统一修复。`esm.sh` CDN 故障修复（marked 改为 npm 依赖）。

### Router 修复 (cea506a0)

`marked` 从 `esm.sh` 裸 URL 改为 `npm:marked@15.0.4` 依赖，消除 CDN 单点。

## Breaking Changes

- `renderDsdByName` → 请使用 `renderDsd('tag', props)`
- `renderNestedDsd` → 内部函数，不再导出
- `import { jsx } from '@lessjs/core'` → 改为 `import { jsx } from '@lessjs/core/jsx-runtime'`
- `parse5` 不再是 LessJS 的依赖
