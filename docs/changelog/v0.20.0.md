## Summary

Ocean-Island Architecture: decouple 9 DSD components from Lit and rebuild on a
zero-dependency `DsdElement` (native HTMLElement) base class. Lit is retained
only for `less-hero-ping` (Island component). Bundle size reduced by ~50%
(12KB ķł½?6KB gzip). All CSS moved from Lit `css\`\``to`CSSStyleSheet`, and
custom`--less-*` design tokens replaced with Open Props.

## Breaking Changes

- **Custom CSS variables renamed**: `--less-*` ķł½?Open Props equivalent
  (e.g., `--less-size-4` ķł½?`--size-4`, `--less-text-primary` ķł½?`--gray-9`).
  Consumers using `--less-*` custom properties directly will need to update.
- **`@openelement/adapter-lit` deprecated** for DSD components.
  Only `less-hero-ping` (Island) still uses it. New DSD components must
  extend `DsdElement` from `@openelement/core`.
- **`DsdLitElement` deprecated**. Use `DsdElement` for new DSD components.
- **`lessDesignTokens` removed** from `@openelement/ui`. Use
  `openPropsTokenSheet` from `@openelement/ui` instead.

## Migration Guide

### For component consumers (no changes needed)

```html
<!-- No change ķłź?the same HTML works -->
<open-button variant="primary">Click me</open-button>
```

### For component authors (migrating from Lit to DsdElement)

```ts
// v0.19 (Lit)
import { css, html } from 'lit';
import { DsdLitElement } from '@openelement/adapter-lit';
class MyButton extends DsdLitElement {
  static styles = css`
    .btn {
      color: var(--less-accent);
    }
  `;
  render() {
    return html`
      <button>...</button>
    `;
  }
}

// v0.20 (DsdElement)
import { DsdElement } from '@openelement/core';
const s = new CSSStyleSheet();
s.replaceSync(`.btn { color: var(--brand); }`);
class MyButton extends DsdElement {
  static styles = s;
  render() {
    return `<button part="control">...</button>`;
  }
}
```

## Post-Release Audit Fixes (2026-05-21)

### P0 Fix

- **openPropsTokenSheet missing `:host` wrapper** ķłź?CSS variables were bare
  declarations without a selector, making them invalid for both
  `CSSStyleSheet.replaceSync()` and SSR `<style>` output. Wrapped in `:host {}`.

### P1 Fixes

- **DsdElement: DSD path missing adoptedStyleSheets** ķłź?`_applyStyles()` extracted
  as shared method, DSD hydration now also applies adoptedStyleSheets to prevent
  style loss after innerHTML re-render.
- **DsdElement: `_hydrateEvents()` memory leak** ķłź?Previous AbortController not
  aborted before creating new one. Fixed by aborting on re-entry.
- **less-button: `_reRender()` replaceChildren bug** ķłź?`replaceChildren` moved
  light DOM children into shadow root, breaking slot projection. Removed manual
  DOM manipulation.
- **4 components: `_esc()` SSR-unsafe** ķłź?`document.createElement('div')` fails
  in SSR. Replaced with pure string regex in less-callout, less-step-card,
  less-dialog, less-input.
- **less-search: CSSStyleSheet SSR crash** ķłź?Module-level `new CSSStyleSheet()`
  converted to lazy `getOverlaySheet()` initialization.
- **less-search: missing openPropsTokenSheet** ķłź?Added to `static styles` array.
- **less-code-block, less-dialog: incorrect formAssociated** ķłź?Neither is a form
  element; removed `static override formAssociated = true`.
- **less-input: hardcoded error color** ķłź?`#e55` ķł½?`var(--error, #dc3545)`.
- **Homepage: missing search button** ķłź?`DocsHome.render()` didn't include
  `<less-search slot="header-actions">`. Non-homepage pages work via
  `_renderer.ts` injection, but homepage bypasses it. Added import and slot.

### Token System Cleanup

- **Deleted**: `design-tokens.ts` + `tokens/` directory (animation, effects,
  radius, spacing, typography) ķłź?superseded by `openPropsTokenSheet`.
- **Clean naming**: All `--less-*` CSS variables replaced with semantic names:
  `--text-primary`, `--bg-surface`, `--border`, `--brand`, etc.
- **www pages**: changelog/roadmap/contributing added `openPropsTokenSheet` +
  `pageSheet` to `static override styles`.
- **Updated**: `deno.json` (removed old exports), `index.ts` (cleaned exports),
