# Changelog - v0.28.5

Release date: 2026-06-02

## Summary

v0.28.5 is a post-publish consumer resolver patch for the v0.28.4 AppShell
release line.

After v0.28.4 published successfully, the post-publish consumer smoke test found
that generated SSG bundles could import bare `sanitize-html` from published
LessJS source. Local workspace builds passed because the root import map provided
that alias, but fresh JSR consumers do not inherit LessJS package import maps.

v0.28.5 removes that consumer-only resolver gap by making the sanitizer imports
explicit `npm:` specifiers in published source.

## Changes

- **Published dependency resolution**: `@lessjs/core`, `@lessjs/adapter-vite`,
  and `@lessjs/content` now import `sanitize-html` through explicit
  `npm:sanitize-html@^2.17.4` specifiers, with explicit Deno type hints.
- **Consumer smoke recovery**: fresh projects generated by
  `jsr:@lessjs/create@0.28.5` no longer need to know about the internal
  sanitizer dependency.
- **Package alignment**: all 19 workspace packages are aligned to `0.28.5` and
  all internal `@lessjs/*` ranges use `^0.28.5`.

## Known Issues

- v0.28.4 was published and should be treated as superseded by v0.28.5 for fresh
  JSR consumers.

## Breaking Changes

- None beyond the v0.28.4 cleanup line. This is a resolver correctness patch.
