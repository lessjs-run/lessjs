# Changelog - v0.28.2

Release date: 2026-06-02

## Summary

v0.28.2 closes the deprecated-code purge and hardening pass from SOP-003:
compat adapter exports are removed, DSD rendering moves to an options object,
head and HTML injection paths are sanitized, Hub snapshot imports move behind a
configurable CDN policy, and SSG no longer bundles Lit/React unless the user
opts in.

## Changes

- **Core adapter registry**: Removed the module-level `registerAdapter`,
  `getAdapter`, and `getRegisteredAdapters` compatibility helpers. Adapter
  packages now register through `getDefaultRegistry().register(...)`, while
  isolated callers can use `createAdapterRegistry()`.
- **Core DSD API**: Added `RenderDsdOptions` and converted internal callsites
  to object-form `renderDsd()` options. Legacy positional arguments remain
  accepted for existing consumers during the pre-1.0 window.
- **Core event hydration**: Expanded DOM event-name aliases for double-click,
  focus, mouse, and pointer events. Multi-handler VNodes now get one stable
  marker with every handler bound during hydration.
- **Core HTML safety**: `innerHTML` is escaped by default in string and DOM
  renderers. `rawHtml: true` is required for trusted HTML and still passes
  through sanitizer policy. `data-signal-html` updates are sanitized before DOM
  assignment.
- **Adapter Vite security**: `headExtras` and `inject.headFragments` now pass
  through `sanitize-html`. Structured stylesheet/script attributes reject event
  handler names, and unsafe URL schemes are blocked.
- **UI URL safety**: `less-layout` validates header, sidebar, GitHub, edit, and
  localized links against a safe URL-scheme allowlist.
- **Hub permissions and CDN policy**: `hub:scan` no longer uses `-A`. Snapshot
  imports resolve through `LESS_CDN_BASE`, defaulting npm packages to jsDelivr
  while preserving explicit self-hosted and esm.sh options.
- **SSG bundle size**: The default SSR `noExternal` list is reduced to LessJS
  workspace packages plus `alien-signals`. Lit and React stay external unless
  explicitly listed in `options.ssr.noExternal`.
- **Build context reset**: `LessBuildContext.reset()` now resets phase metadata
  without discarding user resolve aliases.
- **Generated client-only tags**: Hub client-only tag discovery no longer parses
  generated TypeScript with regex; it imports the generated module through a
  file URL.
- **Production logging cleanup**: Source/CLI `console.log` callsites were
  replaced with `console.info` or retained only in tests/docs/examples.

## Breaking Changes

- `registerAdapter`, `getAdapter`, and `getRegisteredAdapters` are no longer
  exported from `@openelement/core`. Use `getDefaultRegistry()` for the shared
  registry or `createAdapterRegistry()` for isolated registries.
