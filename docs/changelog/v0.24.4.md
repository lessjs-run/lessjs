# v0.24.4: API Naming Convention — Web Platform Style

> **Released**: 2026-05-29 | **SOP**: SOP-001 | **Previous**: v0.24.3

## Summary

v0.24.4 unifies the public API naming to follow Web Platform conventions:
`verbNoun` for functions, `PascalCase` for acronyms, no brand prefixes.
No backward compatibility — old names removed entirely.

## API Renames

### verbNoun Convention

| Old                      | New                            | Rationale                                    |
| ------------------------ | ------------------------------ | -------------------------------------------- |
| `island(tagName, Class)` | `defineIsland(tagName, Class)` | `define` + `Noun` expresses intent clearly   |
| `lessBind(host)`         | `bindEvents(host)`             | No brand prefix; `bind` + `Noun` is standard |

### PascalCase Acronym

| Old                 | New                 | Rationale       |
| ------------------- | ------------------- | --------------- |
| `renderDSD()`       | `renderDsd()`       | `Dsd` not `DSD` |
| `renderDSDStream()` | `renderDsdStream()` | `Dsd` not `DSD` |
| `renderDSDByName()` | `renderDsdByName()` | `Dsd` not `DSD` |
| `renderToDOM()`     | `renderToDom()`     | `Dom` not `DOM` |
| `getSSRProps()`     | `getSsrProps()`     | `Ssr` not `SSR` |

### Internal Cleanup

- `__lessBindDone` (DOM guard) → `__bindEventsDone`

No migration path. All old names removed. Update imports:

```diff

+ import { defineIsland, bindEvents, getSsrProps, renderDsd, renderToDom } from '@lessjs/runtime';
```
