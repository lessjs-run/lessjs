# v0.24.4: API Cleanup Line Endpoint

> **Released**: 2026-05-29 | **Previous**: v0.24.3

## Summary

v0.24.4 is the cleanup line endpoint: API naming unified, `less()` removed,
SSG robustness restored, `_dsdHydrated` eliminated. No backward compatibility
shipped — this is the clean baseline for v0.25.0.

## API Renames

### verbNoun Convention

| Old                      | New                            |
| ------------------------ | ------------------------------ |
| `island(tagName, Class)` | `defineIsland(tagName, Class)` |
| `lessBind(host)`         | `bindEvents(host)`             |

### PascalCase Acronym

| Old                 | New                 |
| ------------------- | ------------------- |
| `renderDSD()`       | `renderDsd()`       |
| `renderDSDStream()` | `renderDsdStream()` |
| `renderDSDByName()` | `renderDsdByName()` |
| `renderToDOM()`     | `renderToDom()`     |
| `getSSRProps()`     | `getSsrProps()`     |

## less() Public API Removal

- `less()` removed from `@lessjs/adapter-vite` — only `lessPipeline()` remains
- `lessCompat` wrapper removed from `build-pipeline.ts`
- `less-plugin.ts` marked `@internal`, imported via subpath by `@lessjs/app`
- Default export of `@lessjs/adapter-vite` is now `lessPipeline`
- All docs (PWA guide, API reference) updated to `lessPipeline()`

## SSG Robustness Fixes

### tagName Detection (Critical)

SOP-002 introduced `import()`-based tagName detection, which silently failed
for route modules depending on Vite virtual modules (`virtual:less-nav`).
All 55 page routes had `tagName: undefined` → SSG used synthetic names
(`index-index` instead of `docs-home`) → `customElements.get()` returned
`undefined` → DSD output was empty.

**Fix**: Reverted to regex-based source scanning (`readFile` + regex on
`export const tagName = '...'`). No module execution, no virtual module issues.

### SSG Smoke Test

- Removed `dist/` cache skip — test always rebuilds now
- DSD markers restored: 3 (polyfill JS only) → 5 (real `<template shadowrootmode>`)

### customElements Duplicate Registration

Added `if (!customElements.get(tagName))` guard to 4 island components
(`less-search`, `less-toc`, `scroll-reveal`, `less-term`) to prevent
SSR duplicate registration errors.

## Internal Cleanup

- `_dsdHydrated` field removed — unified `_renderOrHydrate()` path
- `__lessBindDone` → `__bindEventsDone`
- 5 `_dsdHydrated` test assertions replaced with behavior checks
- Unused lint directives cleaned from `prop.ts`
