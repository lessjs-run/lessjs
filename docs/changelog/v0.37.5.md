# v0.37.5 Changelog

## Summary

v0.37.5 establishes the protocol-first runtime architecture. It turns
`@openelement/protocols` into the replacement boundary for renderer, component,
runtime, route, island, cache, signal, and data contracts while positioning
Vite + Nitro as the default base engine for the next runtime proof.

## Changed

- Accepted the protocol-first Vite + Nitro architecture in ADR-0096.
- Accepted JSR publish as best-effort distribution telemetry in ADR-0097.
- Accepted the EntryDescriptor route manifest contract in ADR-0098.
- Updated roadmap, status, SOP, and NextVersion docs around the Web Components
  application framework identity.
- Kept shadow/DSD as the default render mode and light DOM as explicit opt-in.
- Added protocol group exports for cache, components, conformance, data,
  islands, renderer, routes, runtime, and signals.
- Added compatibility re-exports from existing public package surfaces where
  protocol types moved into `@openelement/protocols`.
- Updated generated project import maps so source packages can resolve protocol
  subpaths during local workspace builds and JSR-based consumer builds.
- Added the v0.37.6 Vite + Nitro runtime proof plan.

## Deferred

- Public API/product surface reset remains v0.38.x.
- Expanded UI interactive/form coverage remains a later UI product slice.
