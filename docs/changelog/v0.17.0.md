# v0.17.0 — Manifest-Native Pipeline

**Release date**: 2026-05-16

## Summary

Eliminates `PackageIslandMeta` and the backward-compat bridge, making the
entire build pipeline consume `LessPackageManifest` natively. adapter-vite now
reads manifest declarations directly instead of a legacy `islands` array.
This is a breaking change for any package that imported `PackageIslandMeta`
directly.

## Breaking Changes

- **`PackageIslandMeta` deleted** — use `LessPackageManifest.declarations`
  instead. The `tagName`, `modulePath`, and `strategy` fields are now
  `declaration.tagName`, `declaration.less.module`, and
  `declaration.less.hydrate`.
- **`packageIslandFromManifest()` deleted** — no bridge needed. Read
  `manifest.declarations` directly.
- **`@openelement/ui` no longer exports `islands`** — only `manifest` is exported.
  Consumers should derive island metadata from `manifest.declarations`.

## What's New

### Manifest-native adapter-vite pipeline

- adapter-vite reads `LessPackageManifest` directly from packages
- `scanPackageManifests()` replaces `scanPackageIslands()` — imports
  `manifest` export instead of `islands` array
- `IslandDecl` extended with `hydrate`, `ssr`, `dsd` fields from manifest
- `Phase1Meta.packageManifests` replaces `Phase1Meta.packageIslands`
- `Phase1Meta.packageIslandDecls` holds pre-computed `IslandDecl[]` from
  manifests

### Core cleanup

- `PackageIslandMeta` interface removed from `core/types.ts`
- `packageIslandFromManifest()` removed from `core/registry.ts`
- `PackageIslandMeta` and `packageIslandFromManifest` exports removed from
  `core/index.ts`
