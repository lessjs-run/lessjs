# Changelog - v0.30.1

Release date: 2026-06-04

## Summary

v0.30.1 is the post-freeze clean architecture sweep. It closes the openElement
rename, removes active LessJS compatibility names, keeps interactive island UI
on the VNode renderer path, and updates current docs and the www site to match
the v0.30 architecture contract.

## Changes

### openElement public contract

- `@openelement/app` now exposes `openElement()` as the Vite configuration
  facade.
- Vite plugin names, virtual module ids, generated entry labels, and current
  docs now use `open*` / `virtual:open-*`.
- UI package subpaths and custom elements now use `open-*`.
- Package and route metadata now use `openElement` as the structured metadata
  field.
- Protocol validators and public package manifest types were renamed to the
  openElement vocabulary.

### VNode-only dynamic island UI

- Active framework-authored interactive UI no longer uses `data-on-*` string
  event binding.
- `data-signal-html` was removed from the active island dynamic UI path.
- Signal-driven dynamic content now uses `data-signal-render` internally to
  connect signal changes to VNode rendering.
- Island prop binding was renamed from `bindEvents()` to `bindSsrProps()` to
  describe the remaining SSR-prop hydration behavior.

### Trust and renderer boundary

- `trustedHtml` remains only as an explicit non-interactive trust escape hatch
  for pre-sanitized content such as Markdown, MDX, syntax highlighting, and Hub
  snapshot output.
- Active dynamic UI must return JSX/VNodes instead of HTML strings.
- `innerHTML` remains audited by the architecture gate rather than being treated
  as a generic renderer feature.

### Build and metadata cleanup

- The SSG package resolver was renamed to the openElement vocabulary and aligned
  with the current 19-package export graph.
- Generated Hub data now lands in the current registry data directory.
- The old accidental `opc-doc/` material was removed from tracking.
- Current architecture docs now have a single source:
  `docs/arch/current-architecture.md`.

### Website and docs

- The www site was updated for the openElement identity, current `open-*`
  components, v0.30.1 version labels, and current architecture copy.
- Current guide, reference, ADR, SOP, status, and agent prompt docs were updated
  to avoid stale LessJS public-contract language.
- Historical ADRs, changelogs, and blog posts remain historical archives and are
  not rewritten solely for branding.

### Release metadata

- All 19 packages were bumped to `0.30.1`.
- Internal `jsr:@openelement/*` ranges were bumped to `^0.30.1`.
- The root `deno.lock` was regenerated and committed so CI cache keys and
  publish-workflow lockfile resets have a real tracked target.
- `www/app/data/version.ts`, Hub constants, package manifests, registry data,
  changelog, and release notes were aligned to v0.30.1.

## Breaking Changes

- `lessjs()` is no longer the current app facade; use `openElement()`.
- `lessPipeline`, `less-*` UI subpaths, `<less-*>` tags, `less:*` events, and
  `virtual:less-*` are no longer active public contracts.
- Manifest and route metadata use `openElement`, not `less`.
- Framework-authored dynamic island UI must use JSX/VNodes and JSX event
  handlers, not `data-on-*` method-name lookup or dynamic HTML strings.
