# v0.26.1 Changelog

## Breaking Changes

None.

## Critical Bug Fixes (P0)

- **G1: Effect memory leak** ‚Äî `_walkAndBind` passed `applyProps` without AbortSignal. All per-prop signal‚ÜíDOM effects from DSD hydration leaked permanently (never disposed on disconnect). Fixed by passing AbortSignal through the walk chain and aborting on disconnect.
- **G2: consumeContext dead copy** ‚Äî `consumeContext()` returned `signal(value)` ‚Äî a new independent signal disconnected from the provider. Provider updates never reached consumers. Fixed by returning the source signal from the central `contexts` Map.
- **B1: `Unexpected identifier 'as'`** ‚Äî DSD polyfill template literal contained TypeScript `as` assertion injected into inline `<script>` in HTML, causing JS parse error on every page load. Fixed by removing the `as` from the polyfill string.
- **B2: SignalContext subpath build failure** ‚Äî `@openelement/core/signal-context` resolved to `packages/core/src/index.ts/signal-context` (ENOTDIR) due to Rolldown path resolution. Fixed by importing from `@openelement/core` main entry.

## Framework Cleanup

- `effectScope` exposed from `@openelement/signals` ‚Äî one call replaces `_templateAbortController` + `_signalUnsubscribers` (~40 lines removed)
- `_walkAndBind` now handles signal-like text children (G3)
- `_layoutWorkaroundReRender()` replaces `effect(() => render())` ‚Äî one-time DOM replacement for Chromium DSD layout bug
- `consumeContext` API simplified ‚Äî `_host` parameter removed
- Double `@openelement/signals` import merged

## www Site Overhaul

- **Page styles**: Full rewrite of `pageStyles` ‚Äî all sizing uses `var(--size-N)` / `var(--font-size-N)`, all colors use semantic tokens, removed duplicate `.nav-row` rules, added `[data-theme]` light/dark contrast rules, improved typography spacing and readability
- **Nav links**: Enhanced header nav with `::after` underline animation, `aria-current="page"` active state indicator, hover background via `var(--bg-surface)`, all hard-coded `rgba` colors replaced with semantic tokens
- **Open Props migration**: `less-layout` CSS sheet ‚Äî all `var(--xxx, #fallback)` fallback values removed, all `rgba(124,111,245,0.XX)` replaced with `var(--border)` / `var(--bg-surface)`, all `--border-futuristic` removed, logo gradient uses `var(--brand)` ‚Üí `var(--brand-light)`
- **Header**: `rgba(9,11,17,0.88)` ‚Üí `var(--bg-base)`, `padding: 0 64px` ‚Üí `padding: 0 var(--size-16)`, `gap: 32px` ‚Üí `gap: var(--size-8)`
- **Mobile menu**: Button border, colors now use semantic tokens
- **Theme**: Dark mode is default; SignalContext (`THEME_CTX`) unifies theme state across components; CSS `[data-theme="light"]` overrides handle light-mode contrast
- **i18n**: Default locale is `en` (English); `lang-switch` button styled with semantic tokens; locale unified via `less-layout` attribute
