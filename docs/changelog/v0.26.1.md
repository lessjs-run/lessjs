# v0.26.1 Changelog

## Breaking Changes

None.

## Critical Bug Fixes (P0)

- **G1: Effect memory leak** — `_walkAndBind` passed `applyProps` without AbortSignal. All per-prop signal→DOM effects from DSD hydration leaked permanently (never disposed on disconnect). Fixed by passing AbortSignal through the walk chain and aborting on disconnect.
- **G2: consumeContext dead copy** — `consumeContext()` returned `signal(value)` — a new independent signal disconnected from the provider. Provider updates never reached consumers. Fixed by returning the source signal from the central `contexts` Map.
- **B1: `Unexpected identifier 'as'`** — DSD polyfill template literal contained TypeScript `as` assertion injected into inline `<script>` in HTML, causing JS parse error on every page load. Fixed by removing the `as` from the polyfill string.
- **B2: SignalContext subpath build failure** — `@lessjs/core/signal-context` resolved to `packages/core/src/index.ts/signal-context` (ENOTDIR) due to Rolldown path resolution. Fixed by importing from `@lessjs/core` main entry.

## Framework Cleanup

- `effectScope` exposed from `@lessjs/signals` — one call replaces `_templateAbortController` + `_signalUnsubscribers` (~40 lines removed)
- `_walkAndBind` now handles signal-like text children (G3)
- `_layoutWorkaroundReRender()` replaces `effect(() => render())` — one-time DOM replacement for Chromium DSD layout bug
- `consumeContext` API simplified — `_host` parameter removed
- Double `@lessjs/signals` import merged
