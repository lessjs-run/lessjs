# Changelog — v0.28.0

## Architecture

### v0.28 Architecture Contracts Stabilization (c9af56c8, #40)

`packages/core` 公开 API 表面稳定化：

- **`adapter-registry.ts`**：从模块级可变状态重构为工厂函数 `createAdapterRegistry(): AdapterRegistry`。Default registry 实例保留向后兼容导出（`registerAdapter` / `getAdapter` / `getRegisteredAdapters`），但子进程可创建独立 registry 实例提升可测试性。
- **`types.ts`**：`LessRenderer` 接口扩展，新增字段用于协议对齐。
- **`dsd-element.ts`**：
  - `_renderOrHydrate` 用 try/catch 包裹，失败时调用 `_renderErrorFallback`
  - 新增 `protected onRenderError(error: unknown): string | VNode` 钩子，子类可自定义降级 UI
  - `_bindEvents` 支持 `skipSignalHtmlDescendants` 选项，避免重复绑定 `data-signal-html` 内的标记
  - `_renderErrorFallback` 完整清理 effect disposers + event cleanups，再调 `onRenderError`
- **`vnode.ts`** / **`event-hydration.ts`** / **`render-dsd.ts`**：内部契约同步更新
- **`index.ts`**：re-export `createAdapterRegistry` / `AdapterRegistry` 类型

`packages/adapter-vite` 同步收口：

- **`build-manifest.ts`**：manifest 输出格式收紧
- **`entry-renderer.ts`**：`renderPageRoute` 路径简化

净结果：+547 / -255 行，含 128 行 dsd-element-hydration 新测试 + 54 行 event-hydration 新测试 + 32 行 adapter-registry 新测试。

## Theme System

### Full Open Props Migration (161ecb34, fada8a52)

`packages/ui/src/open-props-tokens.ts` 扩展 7 个 Open Props 颜色阶（indigo / green / red / orange / blue / teal / cyan），全部带 dark mode 反演。`www/app/` 30+ 文件改写：

- `var(--bg-*)` → `var(--gray-*)` 直接引用
- `var(--brand)` / `var(--brand-*)` → `var(--indigo-5)` / `var(--indigo-*)`
- `var(--success/error/warning/info)` → `var(--green/red/orange/blue-6)`
- `var(--text-*)` → `var(--gray-*)`
- `var(--border)` → `var(--gray-3)`
- `var(--overlay)` → `color-mix(...)`
- 所有 hardcoded `rgba(...)` → `color-mix()` / semantic tokens

净结果：0 custom semantic tokens，0 hardcoded colors（除 hero 装饰 + `--compat-*` 源定义保留 6 个 hex）。

### adoptedStyleSheets Theme 切换重算 (a2f81743)

切换 light / dark 时强制 `adoptedStyleSheets` 重新求值，修复 shadow DOM 短暂继承旧主题 token 的闪烁。`:host()` CSS 不再 stale。

### Theme Gates Stabilization (34e936cb, #39)

`less-layout` / `less-theme-toggle` / `home-console` / `www/app/routes/index/index.tsx` 的主题切换路径加固：

- `data-theme` 状态机收紧：dark / light 切换时序确定
- `less-theme-toggle` button 状态语义修正（`aria-pressed` 同步）
- `home-console` signal binding 路径调整，避免主题切换时的中间态闪烁
- 新增 `www/e2e/theme-system.spec.ts` 主题切换 E2E 验证

### Hardcoded Color 三轮清扫 (fada8a52, 6a6e02af, 57f8af79)

- `registry/[package]/[component]`：25 个 hex → CSS vars + `color-mix()`
- feature grid cards：改用 semantic tokens，跟随主题
- dark mode hero bg：gray 阶反演， `--gray-0/1` 替代 `--gray-1/2`（gray scale 是反的）

## Signal-Native Islands

### Signal-Driven Registry (e4cd8d01)

`www/app/routes/registry/index.tsx` 重写为 `registerSignal()` + `computed()` 驱动：

- 0 inline 箭头处理器（`onClick` / `onChange` / `onInput`）
- 0 手动状态字段（`_query` / `_tierFilter` / `_sortBy` / `_filtered`）
- 0 非 signal DOM 写入
- 改用 `data-on-click` / `data-on-change` / `data-on-input` 标记 + `dataset.filter` 读取

`reactive-showcase.tsx` 同步切到同模式（`onInput={(e) => ...}` → `data-on-input` + 命名 handler）。

### less-toc 整体删除 (d56c75a5)

`www/app/islands/less-toc.tsx` 217 行整文件删除。`scroll-reveal` island 改写为 signal-driven：`scrollY` → `signal` → `data-visible` 属性 → CSS `[data-visible]` 选择器。**0 手动 DOM 操作**。

### Islands 手动 DOM 归零

合并 Open Props 迁移后，`www/app/islands/` 内不再有 `addEventListener` / `innerHTML=` / `document.createElement` 写法。

## CI & Tooling

### setup-deno-workspace Composite Action (c9af56c8)

新增 `.github/actions/setup-deno-workspace/action.yml` 复合 action（17 行）：

```yaml
- 安装 Deno
- 恢复 node_modules 缓存（key: ${{ runner.os }}-deno-${{ hashFiles('deno.lock') }}）
- 跑 `deno install --node-modules-dir`
```

`test.yml`（13 jobs）、`publish-jsr.yml`、`hub-ci.yml`、`deploy-api.yml` 全部改用该 action。`hub-ci.yml` 顺手把 `deno run -A packages/hub/src/cli/*.ts` 替换为 `deno task hub:*`（与 root `deno.json` 的 task 对齐）。

净节省：CI 配置 169 行 → 56 行（-67%），setup 步骤重复消除。

### Fmt 忽略列表扩展 (34e936cb)

`deno.json` 的 `fmt.exclude` 与 `fmt:check` 忽略列表新增两个目录：

- `www/app/data/`（构建生成的数据文件 `_generated-*.ts`）
- `docs/conversation/`（中文校审 / 校对稿，含 UTF-8 编码长文件名，fmt 处理有副作用）

净结果：deno fmt 范围从 771 个文件收窄到 263 个（按 lint 范围对齐）。
