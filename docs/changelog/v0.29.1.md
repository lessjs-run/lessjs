# Changelog - v0.29.1

Release date: 2026-06-03

## Summary

v0.29.1 completes the renderer IR cleanup. All compatibility shortcuts kept in
v0.29.0 are removed. The renderer now has one attribute serializer, one async
render path, and one public rendering API.

## Changes

### Phase 1: zero-risk fixes

- Fixed `_hyrateExistingDom` typo → `_hydrateExistingDom`.
- Removed hardcoded version `'0.20.0'` in `hub/scanner.ts`.
- Added dev-mode warning to `trustRenderHtml` (trust boundary, not a sanitizer).

### Phase 2: unified attribute serialization

- Created single `serializeAttrs(tag, props)` replacing `serializeAttributes`
  and `serializeElementAttrs`.
- `dsdHostNode.attrs` changed from pre-serialised `string` to
  `Record<string, unknown>`, flowing through the same serializer as JSX elements.
- `camelToKebab` moved from `render-serialize.ts` into `render-ir.ts`.

### Phase 3: file merge (7 files deleted)

- **Deleted**: `jsx-render-string.ts`, `custom-element.ts`, `api.ts`,
  `prop-types.ts`, `render-errors.ts`, `render-instantiate.ts`,
  `render-serialize.ts`.
- **Merged into**: `render-ir.ts`, `render-dsd.ts`, `types.ts`, `island.ts`,
  `prop.ts`.
- `core/src` reduced from 33 files to 26 (-21%).
- Removed duplicate `isValidCustomElementName` in `registry.ts`; now uses
  `tag-utils.ts:isValidTagName`.

### Phase 4: API cleanup

- **Deleted `renderDsdByName`**: superseded by `renderDsd(tagName, { props })`.
- Updated all test callers and re-exports.

### Phase 5: single async render path

- Created `renderToNode(node, eventContext): Promise<RenderNode>` — one function
  merging the old `renderToStaticNode` / `renderToDsdNode` dual-track.
- Inlined `renderSpecialVNodeSync` / `Async` and `renderChildrenFromProps` /
  `Async` helpers into the single traversal.
- **Deleted `renderToString`** — use `renderDsdTree` with `await` instead.
- Deleted dead code `_resolveRenderOutput` from `dsd-element.ts`.
- `renderDsdTree` is now the **only** public rendering API.

## Public API

- `renderDsdTree(node)` → `Promise<string>` (unchanged, was already async).
- `renderDsd(input, options)` → `Promise<RenderOutput>` (unchanged).
- `renderToString` — **removed**; use `renderDsdTree` instead.

## Breaking Changes

- `renderToString` is completely removed. Callers must switch to `await renderDsdTree(node)`.
- `renderDsdByName` is removed. Callers must use `renderDsd(tagName, { props })`.
- `@openelement/core/api` subpath is removed. `LessApiContext` is available from `@openelement/core`.
- `@openelement/core/render-serialize`, `@openelement/core/render-errors`,
  `@openelement/core/render-instantiate` subpaths are removed. Their content is now
  internal to core.
- Component authors calling `renderToString` inside `render()` must restructure
  to avoid sync rendering inside the synchronous `render()` contract.

## Migration

```ts
// Before
import { renderToString } from '@openelement/core';
const html = renderToString(vnode);

// After
import { renderDsdTree } from '@openelement/core';
const html = await renderDsdTree(vnode);
```
