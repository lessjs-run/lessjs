# v0.21.5 - 2026-05-24

## Summary

v0.21.5 is the hardening release that wraps up the v0.21.x line. All 11 SOP gates
pass, the Core API is classified, DSD conformance is proven, the Adapter
architecture is cleaned, and Hub trust gates are strengthened. v0.22.0 Edge
Full-Stack is unblocked.

## Changes

### Core API Tightening (SOP-001 ~ SOP-003)

- `RenderError` now includes stable `code: RenderErrorCode` (6 variants:
  `LESS_RENDER_INSTANTIATE_FAILED`, `LESS_RENDER_INVALID_OUTPUT`,
  `LESS_RENDER_RENDER_FAILED`, `LESS_RENDER_NESTED_FAILED`,
  `LESS_RENDER_STYLE_FAILED`, `LESS_RENDER_SERIALIZE_FAILED`) and
  `severity: 'error' | 'warning'` for downstream gate consumption.
- `classifyError()` auto-derives code from phase + message pattern.
- `DsdComponent.render()` type tightened to `string | TemplateResult | unknown`,
  TemplateResult-first.
- `RenderErrorCode` and `RenderErrorSeverity` types exported in public types.
- Public API surface classified into 4 levels: Stable (8 items), Stable Data
  Contracts (5), Experimental (6), Internal (4), Deprecated (1).
  Documented in `docs/reference/core-api-surface.md`.

### DSD Conformance Evidence (SOP-004 ~ SOP-006)

- New `packages/core/__tests__/dsd-conformance.test.ts`: 3 tests covering
  `shadowrootmode`, `shadowrootdelegatesfocus`, `shadowrootslotassignment`,
  and host/template structure.
- New `packages/core/__tests__/core-api-surface.test.ts`: verifies core stable
  exports are functions with correct types.
- `dsd:check-report` gate: 112 known Shoelace SSR boundary diagnostics,
  0 unknown error types, non-recoverable threshold ≤ 12 (current: 8).
- Reference docs: `template-reactive-contract.md`, `web-component-compatibility.md`
  in `docs/reference/`.

### Adapter Architecture Cleanup (SOP-007 ~ SOP-008)

- `route-scanner.ts`: new `readRouteTagName()` statically reads
  `export const tagName = 'xxx'` from route source files, eliminating runtime
  module export probing.
- `entry-descriptor.ts`: `PageRouteDecl.tagName` changed from optional to
  required, with `fileToTagName()` fallback.
- `entry-renderer.ts`: generated SSR code no longer contains
  `module.tagName || 'fallback'` runtime probes — all tagName resolution is
  compile-time. Eliminates IMPORT_IS_UNDEFINED build warnings and prevents
  docs code-block misidentification.
- SSG report test updated to include `code` and `severity` in RenderError fixture.

### Hub Trust Gate Hardening (SOP-009)

- `manifestHash` now validated as 64-char lowercase hex (`/^[a-f0-9]{64}$/`).
- Submission artifacts must have valid non-empty `path`, `contentType`, and
  `content` fields.
- `hub-submit` passes real custom-elements-manifest content to
  `buildPackageRecord()`, generating cryptographic hashes based on actual CEM data.
- New validation tests: missing manifestHash, empty artifact content.
- `hub-submit` dry-run test now uses temp directory with real
  `custom-elements.json` fixture.
