# v0.21.0 - 2026-05-22

## Summary

v0.21.0 ships the Hydration + ISR contract defined by the v0.21 SOP: explicit
island strategies, client-only SSR exclusion, ISR metadata primitives, and API
route production context.

## Changes

- **Hydration**: Replaced legacy strategy vocabulary with `client:load`,
  `client:idle`, `client:visible`, and `client:only`.
- **Client runtime**: Regenerated island client entry around the four v0.21
  strategies, with `client:only` scheduled as client-only idle work.
- **SSR admission**: `client:only` islands now resolve to `ssr: false`,
  `dsd: false`, and are excluded from the SSR bundle.
- **Island manifest**: Build output can emit per-page island manifests with
  chunk URLs, strategies, and layer metadata.
- **DSD report**: `dsd-report.json` now includes a v0.21 hydration strategy
  summary and client-only exclusion evidence.
- **ISR**: Added core ISR cache primitives and SSG `isr-manifest.json` emission
  for routes exporting positive `revalidate` values.
- **API routes**: Added `LessApiContext` / `LessApiHandler`; generated routes
  now support either default-exported Hono apps or direct LessJS handlers.
- **Docs**: Updated SOP, README, roadmap, and www guide pages for v0.21.

## Known Issues

- Provider-specific ISR persistence, cache locking, and CDN invalidation remain
  adapter responsibilities.
- `client:*` directive parsing is represented through island metadata in this
  line; syntax sugar can be added later without changing the core contract.

## Breaking Changes

- `eager` and `lazy` are no longer valid island strategy names. Use `load` and
  `idle`.
- Public manifest validation now accepts only `load`, `idle`, `visible`, and
  `only` for `less.hydrate`.
