# v0.21.0 - 2026-05-23

## Summary

v0.21.0 ships Reactive DSD: native `html` templates, safe interpolation,
Signal-driven `DsdElement` updates, and Response-compatible streaming DSD.

## Changes

### Reactive DSD Runtime

- `DsdElement.render()` now supports `string | TemplateResult`.
- Existing static `render(): string` components keep the old CSR and DSD behavior.
- Components returning `html` templates render in CSR fallback and SSR `renderDSD()`.
- SSR `renderDSD()` now preserves internal runtime markers for `TemplateResult`
  event/property bindings so Declarative Shadow DOM upgrade can attach behavior.
- Template event bindings such as `@click=${handler}` attach after CSR render and
  DSD upgrade without serializing functions into HTML.
- Template property bindings such as `.value=${value}` are applied on the client
  without becoming HTML attributes.
- Signal-like values with `.value` and `.subscribe()` are collected from templates.
- Signal writes schedule microtask-batched component-local rerenders.
- Signal subscriptions and template event listeners are cleaned up on disconnect.

### Safe Templates

- Added `packages/core/src/template.ts`.
- Added public `html`, `unsafeHTML`, `isTemplateResult`, `isUnsafeHTML`,
  `isSignalLike`, and `renderTemplateToString` exports from `@lessjs/core`.
- Text interpolation escapes HTML by default.
- Attribute interpolation escapes quote and tag breakout vectors.
- URL-sensitive attributes such as `href` and `src` reject dangerous protocols.
- `unsafeHTML()` is the explicit trust boundary for raw HTML insertion.
- Nested templates and arrays are rendered recursively.
- Event handlers and property values are excluded from SSR HTML output.

### Streaming DSD

- Added `renderDSDStream()` to `@lessjs/core`.
- Added `createRenderDSDStreamMetrics()` and stream option/types for chunk and
  error observation.
- Stream output emits shell first, component chunks next, and footer last.
- Stream chunks are encoded as `Uint8Array` so `new Response(stream)` works in
  Deno, Workers-style runtimes, and browser Web Streams.
- `onChunk` still exposes the rendered HTML string and structured `RenderOutput`
  for diagnostics.

### UI And Website Adoption

- `less-theme-toggle` now uses Reactive DSD directly: `html` templates,
  component-local Signals, safe `unsafeHTML()` icon insertion, and template
  `@click` binding.
- Removed the theme toggle's old manual `hydrateEvents` and imperative DOM
  update path in favor of the v0.21 runtime contract.
- Added `@lessjs/signals/framework` as the side-effect-light public subpath for
  framework-level Signal primitives.

### Architecture Boundary

- No DOM diff API was added.
- No virtual DOM, compiler, JSX transform, Vite, Hono, Node, or npm runtime
  dependency was introduced into `@lessjs/core`.
- Complex UI remains an Island responsibility; Reactive DSD is for small
  DSD-native Web Component interactions.

### SOP And Strategy Docs

- Rewrote `docs/sop/v0.21.0` as five detailed SOPs:
  SOP-001 `DsdElement + Signals`, SOP-002 `Safe Templates`,
  SOP-003 `Streaming DSD`, SOP-004 `Integration Depth + DX`,
  SOP-005 `Verification + Release Gate`.
- Removed the old optional DOM diff SOP from v0.21 scope.
- SOP-004 delivered: fine-grained `data-b` markers, `_patchBindings` reactive
  DOM patching (no full innerHTML replacement after initial render),
  single-import DX (`import { DsdElement, html, signal } from '@lessjs/core'`).
- `@lessjs/core` now re-exports `signal`, `computed`, `effect` from
  `@lessjs/signals/framework`.
- Rewrote `docs/sop/v0.22.0` around Edge Full-Stack.
- Updated roadmap/status/README strategy text.

## Breaking Changes

- `renderDSDStream()` returns a byte stream (`ReadableStream<Uint8Array>`) rather
  than a string stream so it can be passed directly to `new Response(stream)`.
  Consumers that need text chunks should use `onChunk`.
- UI tests and website components now assert the Signal-based theme state rather
  than the removed `_isLight` implementation detail.

## Known Issues

- Reactive DSD currently uses microtask-batched component-local rerendering for
  Signal changes; it intentionally does not include a general DOM diff engine.
- Fine-grained keyed list reconciliation, focus-preserving subtree diffing, and
  complex stateful surfaces remain Island responsibilities.
