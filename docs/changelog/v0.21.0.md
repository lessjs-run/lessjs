# v0.21.0 - 2026-05-22

## Summary

v0.21.0 ships the Hydration + ISR contract: explicit island strategies
(`client:load/idle/visible/only`), client-only SSR exclusion with placeholder
stubs, ISR cache primitives, API route production context, and demo pages for
all four strategies. All 10 SOPs completed (PLANNED → IMPLEMENTED).

## Changes

### Hydration Contract (SOP-001)

- Replaced legacy `eager`/`lazy` with `client:load`/`client:idle`/`client:visible`/`client:only`.
- Added `HydrationStrategy` and `StrategySource` types to `@lessjs/core`.
- `strategySource` tracking: `directive` | `island-options` | `manifest` | `default`.

### Island Manifest Pipeline (SOP-002)

- ClientIslandEntry now carries `strategySource`.
- `ManifestDecision` includes `strategySource` for build report diagnostics.
- Per-island strategy counts in `dsd-report.json` (`hydrationStrategySummary`).

### Client Runtime Strategies (SOP-003)

- `client:load` → immediate import.
- `client:idle` → `requestIdleCallback` with RAF/setTimeout fallback chain.
- `client:visible` → `IntersectionObserver` (200px rootMargin) + `MutationObserver` element discovery.
- `client:only` → immediate import (no DSD/SSR expected).
- SSR guard: strategy scheduling skipped during server rendering (IntersectionObserver etc. not called in Deno).
- Module dedup via `__map[tag] = null` after first load.
- Error handling: failed imports log a concise warning per module.

### Client-Only SSR Exclusion (SOP-004)

- `client:only` → `ssr: false`, `dsd: false`, excluded from SSR bundle.
- Stub plugin preserves original tag name with `data-less-client-only="true"` attribute.
- Conflict detection: same tag in both SSR and `client:only` throws build error.
- `clientOnlyExcluded` count in build report.

### ISR Cache Layer (SOP-005)

- Core primitives: `IsrCache`, `IsrCacheState` (miss/hit/stale/error), `IsrRouteConfig`.
- `MemoryIsrCache` in-memory implementation for tests/dev.
- `createIsrCacheKey` for route key generation (path + locale + params).
- `RouteEntry.revalidate` field for route-level ISR configuration.
- `IsrRouteRecord` in `DsdBuildReport.isrRoutes`.

### API Route Production Parity (SOP-006)

- `LessApiContext` / `LessApiHandler` types.
- `createLessApiContext()` factory function.
- Generated API routes support default-exported Hono apps or direct LessJS handlers.
- Public type does not expose Vite internals.

### WWW Demo And Docs (SOP-007)

- 4 demo island components: `demo-load`, `demo-idle`, `demo-visible`, `demo-only`.
- `/guide/islands` route page with live examples and strategy descriptions.
- Updated: `/guide/api`, `/guide/ssg`, `/engine/islands-deep`, `/roadmap`, `/index`.

### Hub Hydration Metadata (SOP-008)

- `HubTagRecord.defaultHydrationStrategy` field in hub schema.

### Release Verification (SOP-009)

- `IMPORT_IS_UNDEFINED` warnings suppressed in Rollup `onwarn` (revalidate handled correctly via typeof check).
- All 10 SOPs verified against implementation.

### Non-Goals & Deferred (SOP-010)

- No generic request-time SSR. No auth/ORM/Signals rendering. No publisher accounts.

## Breaking Changes

- `eager` and `lazy` are no longer valid island strategy names. Use `load` and `idle`.
- Public manifest validation now accepts only `load`, `idle`, `visible`, `only` for `less.hydrate`.
- Island `only` strategy now loads immediately (was deferred to idle in earlier dev builds).

## Known Issues

- Provider-specific ISR persistence, cache locking, and CDN invalidation remain adapter responsibilities.
- `client:*` directive HTML attribute syntax (`<my-counter client:visible>`) is parsed through island metadata; syntax sugar in route templates deferred to v0.22.
