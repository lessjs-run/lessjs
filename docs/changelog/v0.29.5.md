# Changelog - v0.29.5

Release date: 2026-06-04

## Summary

v0.29.5 is the type safety and SSR cleanup release. Eliminates `ban-types` suppressions, replaces symbol-keyed prop storage with type-safe WeakMap, deletes the Happy DOM dependency, and simplifies the SSR stub class.

## Changes

### VNode.tag precise types

- Defined `ComponentFn` and `ComponentCtor` types replacing bare `Function` in VNode.tag.
- Added `isComponentCtor()` and `isComponentFn()` type guards.
- Updated all component call sites (`callComponent`, `renderToDom`, `event-hydration`) to use type guards.
- Eliminated 3 `ban-types` suppressions.

### prop.ts WeakMap refactor

- Replaced 5 `Symbol.for()` declarations with 5 `WeakMap` instances.
- Changed function signatures from `Record<string, unknown>` to `DsdElement`.
- Eliminated 21 `as unknown as` type escapes in prop.ts and dsd-element.ts.
- Deleted 4 symbol-keyed accessor types.

### SSR stub cleanup

- Replaced named `_SsrHTMLElementStub` class with anonymous inline class containing 6 stub methods.
- Simplified `globalThis.HTMLElement` assignment.
- Removed happy-dom from root `deno.json`.
- Deleted `dsd-element-hydration.test.ts` and `jsx-render-dom.test.ts`.

### Other fixes

- Fixed `adoptedStyleSheets` type escape.
- Fixed `Show` and `For` function props types for `ComponentFn` compatibility.
- Replaced `Function` with `object` in WeakMap generic for lint cleanup.

## What Did Not Change

- `render()` still returns `string | VNode` — not narrowed in this release.
- No public API surface changes.

## Verification

- 1297 tests passed, 0 failed.
- `deno task fmt:check` passed (760 files).
- `deno task lint` passed (278 files).
- `deno task build` passed with DSD verification.
- All 19 packages bumped to `0.29.5`.
