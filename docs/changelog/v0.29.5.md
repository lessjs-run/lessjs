# Changelog - v0.29.5

Release date: 2026-06-04

## Summary

v0.29.5 is the type safety and SSR cleanup release. Eliminates `ban-types` suppressions, tightens component types, removes the SSR stub class, deletes the Happy DOM dependency, and replaces symbol-keyed prop storage with type-safe WeakMap.

## Changes

### Phase 1: VNode.tag precise types

- Defined `ComponentFn` and `ComponentCtor` types replacing bare `Function` in VNode.tag.
- Added `isComponentCtor()` and `isComponentFn()` type guards.
- Updated all component call sites (`callComponent`, `renderToDom`, `event-hydration`) to use type guards.
- Eliminated 3 `ban-types` suppressions.

### Phase 2: prop.ts WeakMap refactor

- Replaced 5 `Symbol.for()` declarations with 5 `WeakMap` instances.
- Changed function signatures from `Record<string, unknown>` to `DsdElement`.
- Eliminated 21 `as unknown as` type escapes in prop.ts and dsd-element.ts.
- Deleted 4 symbol-keyed accessor types (`_PropSignalsAccessor`, etc.).
- Restructured `createPropSignal()` inline for self-contained signal creation.

### Phase 3: render() → VNode | null

- Changed `render()` return type from `string | VNode` to `VNode | null`.
- Simplified `_renderIntoShadowRoot()` from 3 branches to 1.
- Changed `onRenderError()` return type to `VNode | null`.
- Simplified `_renderErrorFallback()` to always use `renderToDom()`.

### Phase 4: SSR stub removal + happy-dom cleanup

- Removed `_SsrHTMLElementStub` class (16 lines).
- Replaced with conditional base class + minimal `globalThis.HTMLElement` stub.
- Added SSR-safe DOM method overrides (`hasAttribute`, `getAttribute`, `setAttribute`, `removeAttribute`).
- Removed `happy-dom` from root `deno.json`.
- Deleted `dsd-element-hydration.test.ts` and `jsx-render-dom.test.ts`.

### Phase 5: Remaining escapes

- Fixed `adoptedStyleSheets` type escape.

## Breaking Changes

- `render()` now returns `VNode | null`. All existing components use JSX; string returns are still handled for backward compatibility.
- `onRenderError()` returns `VNode | null`.
- `_renderErrorFallback()` no longer uses `innerHTML` for string fallbacks.
- `happy-dom` removed from dependencies.

- `deno task lint` passed (278 files).
