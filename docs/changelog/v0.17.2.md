# v0.17.2 — SSR Filtering + dsd-report Manifest Decisions

**Release date**: 2026-05-16

## Summary

Implements the two deferred v0.17.0 SOP items: SSR-aware island filtering
(`less.ssr: false`) and manifest-driven render decisions in `dsd-report.json`.

## Changes

- **SSR Filtering**: Islands with `less.ssr: false` are excluded from SSR
  `customElements.define()` registration. They render as empty custom element
  tags in SSR HTML and get upgraded client-side via the client entry.
- **ManifestDecision type**: New `ManifestDecision` interface in `@lessjs/core`
  records per-island render path (`ssr+client` | `client-only`) derived from
  manifest flags (ssr, dsd, hydrate).
- **dsd-report.json**: New `manifestDecisions` field populated from
  `ctx.phase1.packageIslandDecls` + `ctx.phase1.packageManifests`.
  Empty array when no package manifests are configured.
- **Tests**: 6 new tests (4 in entry-renderer, 2 in ssg-report).
  Total: 528 unit + 92 e2e passing.

## Breaking Changes

None. This is a patch release — feature completion, no API change.
`manifestDecisions` is an optional field in `DsdBuildReport`.

## Known Issues

- 3 JSR `unanalyzable-dynamic-import` warnings in adapter-vite (expected,
  runtime-only deps, not blocking publish)
