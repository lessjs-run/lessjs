# Changelog - v0.28.6

Release date: 2026-06-03

## Summary

v0.28.6 is the build, release, metadata, and trust-boundary cleanup line before
the renderer IR work.

The release removes the remaining hand-maintained release-task gap, makes
generated metadata boundaries explicit, and stops treating `rawHtml` as a core
sanitizer feature. Core now escapes untrusted text by default and treats
`rawHtml` as a caller-owned trusted HTML boundary.

## Changes

- **Graph-driven release tasks**: root `typecheck`, `publish:dry-run`, and
  `publish` now run through `tools/run-package-graph-task.ts`, which discovers
  all 19 workspace packages and executes package work in dependency order.
- **Workspace-local typecheck proof**: graph typecheck validates exported
  package entrypoints from the repository root so local source is checked
  instead of already-published JSR ranges.
- **Hub metadata boundary**: Hub client-only discovery now uses a shared
  module loader. Missing generated Hub data is optional, but an existing broken
  `hub-data.ts` fails SSG with `HUB_DATA_IMPORT_ERROR` instead of silently
  dropping client-only tags.
- **Island metadata AST scan**: local `export const less = { ... }` metadata is
  extracted with TypeScript AST instead of source regex. Dynamic metadata is
  rejected instead of guessed.
- **Optional package stubs**: optional adapter/generated-data stubs are owned by
  `optional-package-stubs.ts` and cover adapter, content, sitemap, and i18n
  imports.
- **Core trust boundary**: `@lessjs/core` no longer owns a sanitizer dependency
  for JSX raw HTML. `rawHtml` is explicit trusted HTML; normal text and
  `innerHTML` remain escaped by default.
- **Adapter registry cleanup**: resetting the default adapter no longer clears
  named adapters. `clear()` remains the explicit full reset.

## Tests Added

- Hub client-only loader tests for missing, valid, and broken generated data.
- Optional package stub coverage tests.
- AST island metadata rejection tests.
- Raw trusted HTML boundary tests.
- Adapter registry default-reset behavior test.

## Breaking Changes

- `rawHtml` is no longer described or tested as sanitizer-backed behavior.
  Callers must sanitize untrusted HTML before opting into `rawHtml`.
- Dynamic island `less` metadata is rejected at scan time. Use a static object
  literal for framework metadata.

## Verification

- Focused adapter-vite/core tests passed locally.
- `deno task typecheck` passed with graph-driven 19-package coverage.
- Full release gates are recorded in the v0.29.0 release note because this
  branch carries v0.28.6 directly into the v0.29.0 renderer line.
