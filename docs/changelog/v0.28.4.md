# Changelog - v0.28.4

Release date: 2026-06-02

## Summary

v0.28.4 is the AppShell protocol and cleanup release. It makes the application
shell explicit configuration, removes remaining compatibility residue from the
render and adapter surfaces, hardens renderer HTML sanitization, and aligns local
release gates with the 19-package workspace graph.

## Changes

- **AppShell protocol**: Added `appShell` and `layouts` framework options for
  default, no-shell, custom-shell, and route-selected layout rendering.
- **Generated entry**: SSG/SSR entry generation no longer unconditionally
  imports `@lessjs/ui/less-layout`; it imports only the configured shells from
  the descriptor.
- **Route layouts**: Route modules can export `meta.layout` to select a named
  layout or `false` for bare rendering.
- **Layout props**: Shell props are merged with generated context:
  `currentPath`, `locale`, `locales`, `navItems`, `headerNav`, `home`, and
  `routeMeta`.
- **Alias priority**: The SSG package resolver now lets exact user aliases win
  before resolving `@lessjs/*`.
- **Neutral default layout**: `less-layout` no longer hard-codes logo text,
  GitHub URL, or footer text. The LessJS docs site configures its own branding.
- **`renderDsd()` contract**: Removed legacy positional arguments. Public calls
  now use `renderDsd(input, options)`.
- **Sanitizer**: Replaced duplicated regex raw-HTML sanitizers with one
  parser-backed `sanitize-html` policy in core.
- **Hub data boundary**: SSG Hub client-only discovery imports generated data as
  structured module data instead of regex-parsing TypeScript text.
- **Adapter diagnostics**: Optional SSR adapter import failures now distinguish
  missing packages from broken packages or missing exports.
- **Adapter-Lit cleanup**: Removed deprecated `DsdLitElement` /
  `WithDsdHydration` exports from the public adapter-lit package surface.
- **DSD gate**: Native non-recoverable DSD render error tolerance now defaults
  to `0`.
- **Deterministic blog data**: Blog posts without frontmatter dates now derive
  dates from `YYYY-MM-DD-*` filenames before falling back to the current date.
- **Release tasks**: Root `publish`, `publish:dry-run`, and `typecheck` cover
  all 19 workspace packages.

## Package Bumps

All workspace packages are aligned to `0.28.4`:

`@lessjs/core`, `@lessjs/rpc`, `@lessjs/ui`, `@lessjs/adapter-lit`,
`@lessjs/adapter-vanilla`, `@lessjs/adapter-react`, `@lessjs/adapter-vite`,
`@lessjs/create`, `@lessjs/signals`, `@lessjs/content`, `@lessjs/i18n`,
`@lessjs/app`, `@lessjs/hub`, `@lessjs/style-sheet`, `@lessjs/compat-check`,
`@lessjs/cem`, `@lessjs/protocols`, `@lessjs/runtime`, and `@lessjs/router`.

## Breaking Changes

- `renderDsd()` no longer accepts positional component class, props, sourceInfo,
  DSD options, collector, nesting depth, or hooks arguments.
- `@lessjs/adapter-lit` no longer exports deprecated `DsdLitElement`,
  `WithDsdHydration`, or the `./dsd-hydration` public subpath.
