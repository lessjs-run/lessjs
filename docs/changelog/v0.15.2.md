# v0.15.2 — RenderOutput + RenderHooks

> Release type: minor-compatible internal protocol hardening

## Summary

v0.15.2 makes the renderer kernel return structured data instead of a raw HTML
string. This is required before package manifests can declare SSR behavior in
v0.16.

## Changes

### `renderDsd()` → `Promise<RenderOutput>`

- `renderDsd()` now returns `Promise<RenderOutput>` instead of `Promise<string>`.
- The returned object includes `html`, `errors`, `metrics`, and `hydrationHints`.
- All callers updated to destructure `out.html` from the result.

### `RenderHooks` pipeline integration

- New `RenderHooks` type: `beforeRender(input)`, `afterRender(output)`, `onError(error)`.
- `beforeRender` fires before component instantiation.
- `afterRender` fires after serialization with the full `RenderOutput`.
- `onError` fires for classified render errors (instantiate, render, style phases).
- Hooks are optional — pipeline behavior unchanged when omitted.

### Error propagation

- `renderNestedCustomElements()` now propagates child errors and hydration hints
  up to the parent `RenderOutput`.
- Structured `RenderError` objects are collected throughout the pipeline via
  `classifyError()` from `render-errors.ts`.

### Callers updated

- `packages/adapter-vite/src/entry-renderer.ts` — `__ssr()` destructure, `renderRoute()` destructure
- `packages/core/src/render-nested.ts` — returns `RenderOutput`, propagates child errors/hints
- `packages/core/__tests__/render-dsd.test.ts` — all assertions use `output.html`
- `renderDSDByName()` — returns `Promise<RenderOutput>`

## New exports

- `RenderHooks` type exported from `@lessjs/core`

## Test coverage

- New test file: `packages/core/__tests__/render-hooks.test.ts`
  - beforeRender fires before instantiation
  - beforeRender receives correct RenderInput
  - afterRender receives full RenderOutput
  - afterRender receives errors from failed render/instantiation
  - onError fires for instantiation errors
  - onError fires for render() errors
  - onError fires for wrong return type
  - Pipeline works with no hooks / empty hooks object
  - Pipeline behavior unchanged when hooks are undefined
  - RenderOutput shape verified (html, errors, metrics, hydrationHints)

## Exit Criteria

- [x] `RenderOutput` is the stable renderer return contract.
- [x] `RenderHooks` is real pipeline behavior, not only a type definition.
- [x] v0.16 manifest work can rely on structured SSR output.
