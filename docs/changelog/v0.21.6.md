# v0.21.6 - 2026-05-24

## Summary

v0.21.6 is a targeted hotfix for standalone JSR consumers. v0.21.5 hardened the
framework internals, but the user-facing path
`deno run -A jsr:@lessjs/create@0.21.5 my-blog` still failed because the
published create CLI queried outdated JSR metadata URLs. This release fixes that
remote create path and keeps the v0.21.x line stable before v0.22.0 work begins.

## Changes

### Create CLI JSR Metadata Fix

- `@lessjs/create` now fetches package metadata from
  `https://jsr.io/@lessjs/<pkg>/meta.json`.
- Remote create mode now reads the actual JSR `latest` field instead of the old
  `latestVersion`/array fallback shape.
- All LessJS packages are versioned as `0.21.6` to keep the monorepo release
  line consistent and ensure generated projects resolve the JSR consumer-build
  fixes instead of immutable `0.21.5` packages.

### JSR Consumer Build Line

- v0.21.6 keeps the previous JSR consumer-build fixes from the dev line:
  adapter-vite alias generation works outside the monorepo, the create template
  avoids hardcoded JSR URL aliases, and Windows/Linux generated-project coverage
  is documented in `docs/sop/v0.21.6/`.

## Verification

```sh
deno fmt --check packages/create/cli.ts packages/create/deno.json
deno test --allow-read --allow-write --allow-env --allow-net --allow-run packages/create/__tests__/cli.test.ts
deno task typecheck
cd packages/create && deno publish --dry-run --allow-dirty
```

Observed result:

- create CLI tests: 14 passed
- typecheck: passed
- publish dry-run: `@lessjs/create@0.21.6` success

## Publishing Note

JSR packages are immutable. The fix is already in git on `dev`, but users running
`jsr:@lessjs/create@0.21.5` will still hit the old `/meta` URL. Publish
`@lessjs/create@0.21.6` from a JSR-authenticated environment, then validate:

```sh
deno run -A jsr:@lessjs/create@0.21.6 test-blog
cd test-blog
deno task build
```

All `@lessjs/*@0.21.6` packages must be published before this remote consumer
test can pass consistently.
