# v0.32.0 - 2026-06-05

## Summary

v0.32.0 turns the JSX-first Application API into a documented and tested app
lifecycle contract.

## Changes

- **Application API**: Added `redirect()`, `notFound()`,
  `OpenElementRedirect`, `OpenElementNotFound`, and lifecycle type guards to
  `@openelement/app`.
- **Page Lifecycle**: `definePage()` render context now receives structured
  `route` and `meta` data, and pages may provide `error(ctx)` fallback rendering
  through the same VNode contract.
- **Generated Entry**: SSR/Hono entry generation now passes route/meta context
  into pages and translates lifecycle redirect/not-found controls at the
  framework boundary.
- **SSG Runtime Contract**: `renderRoute()` catches redirect/not-found controls
  and returns structured output; `routeInfo` now exposes file path, revalidate,
  rendering intent, and streaming intent.
- **AutoWorkflow**: Added mandatory project workflow governance, GitHub issue
  and PR templates, refreshed AI agent prompts, `docs/next/v0.32.0/`, and
  `deno task workflow:check`.
- **Docs and Website**: Updated Status, Roadmap, Architecture, SOP, API
  reference, Getting Started, and homepage content for the v0.32 lifecycle line.
- **Release Metadata**: Aligned all 19 `@openelement/*` packages and internal
  JSR ranges to `0.32.0`.

## Known Issues

- The docs build still reports advisory HTML-size warnings for the generated
  Getting Started page; this is non-blocking and should be handled as docs-site
  content compression or page-splitting work.

## Breaking Changes

- `@openelement/app` now treats redirect and not-found as public lifecycle
  controls. Code that catches all thrown values from `load()` must rethrow
  `OpenElementRedirect` and `OpenElementNotFound` if it wants framework routing
  semantics to apply.
