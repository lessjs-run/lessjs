# v0.14.11 — 2026-05-16

## Summary

Fix 5 deferred audit items from v0.14.10 code review + JSR publish dependency fix.

## Changes

- **G1 (adapter-vite)**: Use single quotes in `app.get()` route paths for consistent formatting
- **G5 (content)**: Replace fragile `extractMeta()` regex with char-by-char parser supporting single/double/backtick strings, hyphenated keys, string arrays, booleans, numbers
- **G10 (core/adapter-vite/content/i18n)**: Break circular dependency — extract `build-types.ts` and `virtual-ids.ts` to `@openelement/core`; adapter-vite re-exports for backward compatibility; content and i18n import directly from core
- **G11 (www)**: Deduplicate term command logic into `www/app/shared/term-commands.ts`; both Hono route and Cloudflare Function call `executeTermCommand()`
- **G12 (lint)**: Remove `no-explicit-any` from global `deno.json` lint exclude; add per-file `// deno-lint-ignore-file no-explicit-any` to `signals/polyfill.ts` and `signals/engine.ts`; add per-line ignores for unavoidable `any` casts
- **JSR fix**: Bump all cross-package `jsr:` version constraints from `^0.14.8` to `^0.14.11` (0.14.9 lacks `./build-types` and `./virtual-ids` exports); remove stale `@openelement/adapter-vite` import map entries from content and i18n; revert sitemap to dynamic import to break publish-time circular dependency

## Known Issues

- 3 JSR `unanalyzable-dynamic-import` warnings in adapter-vite (expected for runtime-path-dependent imports, not blocking)

## Breaking Changes

- None (backward-compatible re-exports in adapter-vite)
