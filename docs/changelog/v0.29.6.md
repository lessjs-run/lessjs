# Changelog - v0.29.6

Release date: 2026-06-04

## Summary

v0.29.6 closes architecture debt across the entire v0.29.x cleanup arc. Eliminates 20+ type escapes, deletes 40+ lines of duplicate code, centralizes hub constants, and winds down the `render(): string` path.

## Changes

### Type escape elimination

- **island.ts**: Constructor metadata stamping replaced with WeakMap via `getIslandMeta()`.
- **signal-context.ts**: DOM symbol-keyed signal storage replaced with WeakMap. Uses `WritableSignal<T>` for mutable values.
- **render-dsd.ts**: `DsdComponentConstructor` type replaces 3 inline casts.
- **vnode.ts**: `RenderFn` type added to `VNode.children`, eliminating 4 render function casts.
- **compat-check**: Production `as any` replaced with precise structural type.
- **adapter-lit, adapter-vanilla**: Styles property cast simplified to `{ styles?: unknown }`.

### render() → VNode | null

- `DsdElement.render()` returns `VNode | null` (was `string | VNode`).
- `_renderIntoShadowRoot` simplified from 3 branches to 1.
- `_renderErrorFallback` simplified from 2 branches to 1.
- `onRenderError()` returns `VNode | null`.
- `renderDsd()` retains `typeof result === 'string'` compat path with deprecation warning.

### Code deduplication

- `escapeAttr` / `htmlEscape` duplicated in hub and adapter-vite → imported from `@lessjs/core`.
- Removed unused `LessBuildContext as LessBuildContextClass` alias in `@lessjs/app`.
- Removed unused `StyleSheetLike` import in render-dsd.ts.

### Hub centralization

- Created `packages/hub/src/constants.ts` with centralized `VALIDATOR_VERSION`, `LESSJS_UI_VERSION`, `SHOELACE_VERSION`, and Shoelace cache paths.
- Updated `scanner.ts` to use constants instead of hardcoded versions.

### Other

- Updated `signal-context.ts` to use `WritableSignal<T>` for mutable signal values.
- Fixed `callComponent()` children type to include `RenderFn`.

## Breaking Changes

- `render()` now returns `VNode | null`. Components returning strings receive a deprecation warning. SSR path still handles strings for backward compatibility.
