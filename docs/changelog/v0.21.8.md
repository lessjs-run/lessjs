# v0.21.8 - 2026-05-24

## Summary

v0.21.8 is a second JSR consumer-build hotfix for generated project SSG. v0.21.7
fixed `@lessjs/core` resolution inside the virtual SSG bundle, but package
island imports such as `@lessjs/ui/less-card` still failed when the generated
project consumed LessJS from JSR.

## Changes

### Adapter Vite SSG Package Resolver

- Replaced the core-only JSR resolver with a LessJS package resolver for
  `@lessjs/core` and `@lessjs/ui`.
- `@lessjs/ui/*` subpath exports now resolve to JSR source files during the SSR
  bundle build.
- Relative imports inside virtual LessJS package modules are resolved back into
  the same virtual package namespace, so imports such as
  `./open-props-tokens.js` work from `@lessjs/ui/less-card`.
- The resolver version follows the current adapter-vite JSR package version,
  with a `0.21.8` fallback.

### Release Version Alignment

- All LessJS packages are aligned to `0.21.8` because JSR packages are immutable
  and `@lessjs/adapter-vite@0.21.7` was already published before this UI package
  resolver fix.
- Internal package references and `deno.lock` were updated to the `0.21.8`
  release line.
