# Changelog - v0.29.0

Release date: 2026-06-03

## Summary

v0.29.0 is the structured renderer IR release.

The renderer now has an internal `RenderNode` model with a single serializer for
text, element, fragment, trusted HTML, and DSD host output. Public APIs remain
stable: `renderToString()` still returns a string, `renderDsdTree()` still
returns a string promise, and `renderDsd()` still returns `RenderOutput`.

## Changes

- **Internal renderer IR**: added `packages/core/src/render-ir.ts` with
  `text`, `element`, `fragment`, `trusted-html`, and `dsd-host` node types.
- **Single serializer**: HTML escaping and trusted HTML output now flow through
  `serializeRenderNode()`.
- **Thin compatibility entrypoints**: `jsx-render-string.ts` now re-exports the
  IR-backed render entrypoints instead of owning a second traversal body.
- **Trusted HTML node**: `rawHtml` is represented as explicit
  `trusted-html` IR instead of a magic sanitizer branch.
- **DSD host node**: `wrapDsdOutput()` now constructs a `dsd-host` IR node and
  serializes it through the same serializer used by normal JSX output.
- **Renderer tests**: added direct IR tests for escaping, trusted HTML, VNode
  conversion, DSD host serialization, and fragment ordering.

## Public API

No public call shape changes are required in this release.

- `renderToString(node)` remains synchronous.
- `renderDsdTree(node)` remains async and returns HTML.
- `renderDsd(input, options)` remains object-form and returns `RenderOutput`.

## Breaking Changes

- Internal renderer behavior now treats trusted HTML as a first-class node.
  Tests or integrations that expected sanitizer mutation from `rawHtml` must
  sanitize before passing trusted content into LessJS.
