# v0.15.0 — 2026-05-16

## Summary

Renderer Kernel Protocol — replace RenderAdapter with RendererProtocol, add
structured output types, error taxonomy, DSD build reports, and renderDSD
module split. All legacy backward-compatible code removed.

## Changes

- **Protocol**: Replace `RenderAdapter` with `RendererProtocol` (adds required
  `name` field). `RenderAdapter` is fully removed — no aliases, no wrappers.
- **Types**: Add `RenderOutput`, `RenderInput`, `RenderError`, `HydrationHint`
  to `@lessjs/core`
- **Module split**: `render-dsd.ts` (360 lines) → `render-dsd.ts` +
  `render-errors.ts` + `render-instantiate.ts` + `render-serialize.ts`
- **DSD report**: SSG builds emit `dsd-report.json` with aggregate render metrics
- **Adapter registry**: Support named adapters via `getRegisteredAdapters()`
- **customElementRegistry**: Type simplified from `boolean | string` to
  `boolean` (per WHATWG spec — the attribute has no value)
- **create CLI**: Fix JSR remote version resolution; update scaffold to v0.15.0
- **ADR-0025**: Renderer Protocol technical specification

## Known Issues

- 3 JSR `unanalyzable-dynamic-import` warnings in adapter-vite (unchanged from
  v0.14.11)

## Breaking Changes

- `RenderAdapter` removed entirely. Use `RendererProtocol` (requires `name:
  string`).
- `DsdOptions.customElementRegistry` type changed from `boolean | string` to
  `boolean`. String values are no longer accepted.
- `camelToKebab` re-export removed from `render-dsd.ts`. Import directly from
  `@lessjs/core` or `@lessjs/core/render-serialize`.
