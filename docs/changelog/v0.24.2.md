# v0.24.2 - 2026-05-29

## Summary

v0.24.2 ships the JSX+Signal component model (ADR-0057) and removes the old
`html` tagged template API entirely. All components now use JSX + `static props`

- Signal, with zero backward-compatibility overhead. Four infrastructure fixes
  were required for end-to-end JSX pipeline correctness.

## Breaking Changes

- `html` tagged template and all helpers removed: `html`, `classMap`, `when`,
  `choose`, `repeat`, `ref`, `unsafeHTML`
- `@prop()` decorator removed — use `static props = { name: Type }` instead
- `TemplateResult`, `PropertyOptions`, and related internal types removed
- `renderTemplateToString` removed — use `renderToString()` for VNode
- All UI and island components: `.ts` → `.tsx`
- `@lessjs/signals` is now a dependency of `@lessjs/core`

## Added

- **jsx-runtime**: `jsx()`, `jsxs()`, `jsxDEV()`, `Fragment` — React-compatible
  JSX transform interface
- **VNode**: 5-field frozen interface (`tag`, `props`, `children`, `key`, `ref`)
- **renderToString**: VNode → HTML string for SSR
- **renderToDOM**: VNode → DOM nodes for CSR, `addEventListener` + SVG namespace
- **static props**: `static props = { name: Type }` replacing `@prop()` decorator
- **Signal auto-unwrap**: `valueOf()` + `Symbol.toPrimitive` for JSX `{}` auto-unwrap
- **effect() in VNode tracking**: `_renderIntoShadowRoot()` wraps render() in `effect()`
- JSX `IntrinsicElements` TypeScript declarations

## Fixed

### JSX Infrastructure

1. **Island `[object Object]`**: `build-client.ts`/`build-ssg.ts` internal
   `viteBuild()` configs lacked `esbuild` JSX config — esbuild defaulted to
   React `createElement`, producing objects DsdElement couldn't read.
2. **SVG icons disappearing**: `renderToDOM` used `document.createElement()` for
   all elements — SVG needs `createElementNS()`.
3. **Theme toggle not switching**: `_renderIntoShadowRoot` VNode branch never
   subscribed to signals — no re-render on state change.
4. **graph:check fail**: `@lessjs/signals` import in `dsd-element.ts` not declared
   in `core/deno.json`.

### Pre-release Fixes

- VNode event binding: `renderToDOM` wires `onClick` via `addEventListener`
- SSG package resolver `.tsx` subpath resolution
- JSX DOM renderer signal child unwrap
- Stale `.js` search imports → `.tsx`
- `@lessjs/runtime` missing `LessRenderer` export
- Generated project build test alias drift

## Migration

- `html` tagged template → JSX syntax
- `@prop()` → `static props = { name: Type }`
- `@click` → `onClick` (native `addEventListener`)
- `classMap`/`when`/`choose`/`repeat` → JSX ternary, `&&`, `Array.map()`
- Add `jsx: "react-jsx"` + `jsxImportSource: "@lessjs/core"` to `deno.json`
- Add `esbuild: { jsx: 'automatic', jsxImportSource: '@lessjs/core' }` to `vite.config.ts`
