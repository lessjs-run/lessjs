# v0.19.0 — Registry Hub + Component Browser

> Release Date: 2026-05-17
> Branch: dev → main
> SOP: `docs/sop/v0.19.0-platform-hub.md`, `docs/sop/v0.19.0-component-browser.md`

## Summary

The first public-facing Registry Hub with package search, compatibility evidence,
CLI-driven submission pipeline, component detail pages with rendered previews, and
the `less add` CLI. This is the most significant feature release since v0.18.0.

## Phase 1: Hub MVP

### New Package: `@lessjs/hub`

- `src/schema.ts` — HubPackageRecord, HubIndex, HubSubmission types + validators
- `src/builder.ts` — Build Hub records from validation/build artifacts
- `src/indexer.ts` — Build lightweight search indices
- `src/snapshot.ts` — Snapshot management (encode/decode)
- `src/submitter.ts` — Submission bundler + GitHub PR logic
- `src/scanner.ts` — Scan node_modules for WC packages, generate records with CEM data
- `src/snapshot-renderer.ts` — SSR + Happy DOM snapshot rendering
- `src/cli/hub-submit.ts` — `less hub submit` CLI
- `src/cli/validate.ts` — `deno task hub:validate` record validator
- `src/cli/check-index.ts` — `deno task hub:check-index` drift detector
- `src/cli/less-add.ts` — `less add <package>` CLI
- `src/cli/render-happy.ts` — Happy DOM subprocess renderer

### Hub CI

- `.github/workflows/hub-ci.yml` — Validate submissions, check scoped paths,
  conditional auto-merge

### Registry UI (www)

- `/registry/` — Package search + filter + sort
- `/registry/:package` — Package detail (compatibility, install guidance, components)
- `/registry/:package/:component` — Component detail (rendered preview, API reference,
  usage snippet, install command)

### Fixture Data

- Shoelace (39 components, client-only)
- @lessjs/ui (8 components, SSR-capable)
- Media Chrome (6 components, client-only)

## Phase 2: Component Browser

### `less add <package>` CLI

- Auto-detect JSR/npm/local source
- Component count from hub-index or CEM
- `--apply` flag updates deno.json imports
- `--verbose` shows details

### Component Detail Pages

- Per-component SSG pages with `getStaticPaths()`
- Breadcrumb navigation (Registry → Package → Component)
- Rendered preview (SSR or Happy DOM snapshot)
- Usage code snippet
- Compatibility badge
- API Reference from CEM (Attributes, Events, Slots tables)
- Install guidance (`less add <package>`)
- Related components in same package

### Snapshot Rendering

- `renderSnapshotLit()` for SSR-capable Lit components (@lessjs/ui)
- `renderSnapshotWithHappyDom()` via subprocess for npm packages (Shoelace, Media Chrome)
- Demo attributes for visible previews (sl-alert: open, sl-button: variant=primary, etc.)
- CEM data extraction: attributes, events, slots from custom-elements.json
- manifestHash computed from CEM content (SHA-256)
- **adoptedStyleSheets serialization**: Lit's constructable stylesheets are now
  converted to inline `<style>` tags before snapshot serialization. Fixes blank
  previews for `sl-card`, `sl-button`, and all other Shoelace components that
  rely on `adoptedStyleSheets` for shadow DOM styling.

## Phase 6: SSG Resilient Rendering + Visual Overhaul (2026-05-19)

> SOP: `docs/sop/v0.19.0-phase6-ssg-visual-overhaul.md`
> ADR: `docs/adr/0035-ssg-resilient-rendering-visual-overhaul.md`

### A: SSG Resilient Rendering

**A1: Hub client-only tags in SSG admission plan**

- `buildSsrAdmissionPlan()` in `entry-descriptor.ts` now accepts `hubClientOnlyTags`
  parameter — tags from Hub registry data (Shoelace, Media Chrome) are classified as
  client-only without requiring CEM manifests
- `discoverHubClientOnlyTags()` in `index.ts` reads `_hub-data-full.ts` at build time
  and extracts client-only tag names via regex parsing
- Same logic added to `build-ssg.ts` for SSG builds
- `__LESS_CLIENT_ONLY_TAGS__` runtime global now includes Hub registry tags
- **Result**: Shoelace components are no longer instantiated during SSG, eliminating
  72 DSD render errors across 51 pages

**A2: renderDSD() bare-tag fallback**

- `instantiationErrorHtml()` in `render-errors.ts` now outputs bare custom element tag
  (`<sl-input></sl-input>`) instead of broken `<template shadowrootmode>` with error
  comments
- `renderDSD()` in `render-dsd.ts` catches render() errors and returns bare-tag output
  with early return — no broken shadow DOM in SSG output
- **Result**: Browser will upgrade the component when JS loads — correct progressive
  enhancement

**A3: Hub snapshot iframe isolation**

- `HubSnapshotMeta` type added to `packages/hub/src/schema.ts` — structured metadata
  (tagName, importSpec, importUrl, demoAttrs, demoSlots, themeCssUrl, compatibility)
- `HubTagRecord` gains optional `snapshotMeta` field
- Component detail page (`[component].ts`) renders `<iframe srcdoc="...">` when
  `snapshotMeta` is available — CSS isolation, no host pollution
- `_buildSrcdoc()` method constructs self-contained HTML for iframe
- `_resizeIframe()` method auto-adjusts iframe height after load
- Scanner (`scanner.ts`) generates `snapshotMeta` for all tags via `buildSnapshotMeta()`
- Fallback to legacy `ssrSnapshot` for backward compatibility

### B: Visual Overhaul

**B1: CSS tokens + page-styles**

- `effects.ts`: Explicit 3-tier shadow system replacing OpenProps variable references
  (`--less-shadow-sm/md/lg` with direct rgba values)
- `spacing.ts`: Extended spacing tokens (`--less-size-12`, `--less-size-16`);
  border-radius increased (4/6/8px)
- `page-styles.ts`: Complete visual overhaul

  - H2: 1rem → 1.25rem, weight 500 → 600

  - Tables: striped rows + `th` background

**B2: UI component upgrades**

   passing them to `<less-layout>`.

2. **Search only on docs pages**: `<less-search slot="header-actions">` was only injected
   by `guide/_renderer.ts` and `engine/_renderer.ts`. Added search injection to
   `index/_renderer.ts` and `registry/_renderer.ts` so all pages have the search trigger.

   `snapshotMeta` was added to the scanner, so component detail pages fell through to
   `unsafeHTML(sanitizeSnapshot())` which dumped raw shadow DOM HTML. Fixed by
   regenerating hub data with `deno task hub:scan`.

4. **Hub Shoelace duplicate/failing renders**: The iframe `srcdoc` approach requires
   valid CDN URLs in `snapshotMeta.importUrl`, but `@lessjs/ui` components had local
   file paths (`packages/ui/src/less-button.ts`). Fixed `buildSnapshotMeta()` to use
   `https://esm.sh/jsr/` CDN URLs for all non-npm packages.

5. **v0.19.1 tag removed**: User requested staying at v0.19.0. Deleted both local and
   remote `v0.19.1` tags.

## Phase 6 Visual Overhaul + Six-Defect Fix (2026-05-19)

### Bug Fixes

1. **Homepage SearchButton duplicate rendering**: `DocsHome` extended `LitElement`
   instead of `DsdLitElement`, causing SSG DSD output + client Lit re-render to produce
   two `<less-search>` elements. Fixed by changing base class to `DsdLitElement` from
   `@lessjs/adapter-lit`.

   `<less-search>`. Created `www/app/routes/blog/_renderer.ts` following the same

3. **Registry Hub fake i18n**: Registry hardcodes English-only content but declared
   `.locales="${['en', 'zh']}"`, showing a language switcher that leads nowhere. Fixed
   by setting `.locales="${['en']}"` to hide the misleading switcher.

   sections in sidebar (Quick Start, Core, Production, Principles, etc.) instead of
   only their own relevant sections. Added `filterRegistryNav()` and `filterBlogNav()`
   to `www/app/utils/nav-filter.ts` and applied them in all registry and blog route files.

5. **Homepage CTA visual quality**: Replaced the full-width purple gradient CTA block
   with a warm-gray footer (`#F1EFE8`) featuring a 2px brand-color accent line, three-column
   navigation links, and a small dark terminal window. This avoids visual duplication with
   the Hero section and creates a proper "quiet landing" rhythm.

6. **Sidebar too far left on desktop**: Added `max-width: 1400px; margin: 0 auto;` to
   `.layout-body` in `less-layout`, centering the sidebar+content area within the viewport.

## Phase 7: www Full-Site v7 Redesign (2026-05-19)

> Team: PM(许清楚) → Architect(高见远) → Engineer(寇豆码) → QA(严过关)
> Docs: `docs/conversation/www-redesign/`

### What Changed

The v6 homepage-only redesign was replaced with a full-site v7 overhaul. Core complaint:
multi-framework section had only code snippets, no real interactive content; only the
homepage was touched while all other pages remained plain.

### New Design Token System

- **Brand color spectrum**: `--less-brand-light/pale/dark/deep/glow` (5 tokens)
- **Semantic spacing**: `--less-space-1` through `--less-space-16`
- **Border radius**: `--less-radius-xs` through `--less-radius-2xl` (6 levels)
- **Animation**: `--less-duration-micro/fast/normal/slow` + `--less-easing-default`
- **Brand shadows**: `--less-shadow-brand-sm/md/lg` + `--less-shadow-glow`
- **Typography**: `--less-font-size-display` (3.25rem) + `--less-font-size-h1`

### New Components

- **`less-showcase-panel`**: Multi-framework tab container with live island (left) +
  source code (right) side-by-side, using the 3 existing `shoelace-showcase`,
  `react-showcase`, and `media-chrome-showcase` islands
- **`less-callout`**: Callout component (info/warning/danger/tip) using `label`
  attribute (not `title` — avoids HTMLElement.title conflict). Replaces raw `.callout`
  CSS classes on getting-started, positioning, and roadmap pages
- **`less-step-card`**: Step card component for Quick Start section on homepage

### Homepage Rewrite

- **Hero**: Display-level typography, enhanced gradient glow, brand-shadow buttons,
  all hardcoded colors → design tokens
- **Multi-framework**: Real interactive showcase panel replacing v6's code-only tabs
- **Quick Start**: Replaced inline HTML with `less-step-card` components

### Page-Level Visual Upgrade

- `page-styles.ts`: New `.section-divider` style (brand-color gradient hairline)
- `registry/index.ts`: Search input glow effect + card brand-shadow upgrade
- `guide/getting-started.ts`, `guide/positioning.ts`, `roadmap.ts`: `.callout` →
  `less-callout` component

### Code Convention Fixes

- All new components extend `DsdLitElement` (not `LitElement`) for DSD pipeline
  compatibility, consistent with all other `@lessjs/ui` components
- `private _prop + requestUpdate()` pattern (Rolldown doesn't correctly transpile
  decorators)
- New module subpath exports registered in both `packages/ui/deno.json` (for JSR) and
  root `deno.json` imports (for local dev/Vite)

### Technical Debt Recorded

- **Workspace alias explosion** (`docs/conversation/www-redesign/www-redesign-alias-debt.md`):
  `generateWorkspaceAliases()` generates 71 aliases from all 15 workspace packages;
  ~50 are never used by www. Recommendations documented for future refactoring.
