# v0.26.0: Framework Decoupling

> **Released**: 2026-05-29 | **Previous**: v0.25.0 | **ADR**: ADR-0061

## Summary

v0.26.0 removes Vite virtual module coupling from 60 route files, extracts
island transform core logic to `@lessjs/core`, introduces `dev:fast` zero-bundler
dev server, and fixes a critical SSG rendering bug in SSR stubs.

## Virtual Modules → Framework Exports

| Before                   | After                       |
| ------------------------ | --------------------------- |
| `virtual:less-nav`       | `@lessjs/content/nav`       |
| `virtual:less-blog-data` | `@lessjs/content/blog-data` |
| `virtual:less-i18n-data` | `@lessjs/i18n/data`         |

- `@lessjs/content`: new `./nav-data` and `./blog-data` exports
- `@lessjs/i18n`: new `./data` export
- Generated `_generated-*.ts` files written by buildStart hooks
- SSG bundle keeps virtual modules internally

## Island Transform Extraction

- `@lessjs/core/island-transform.ts`: `transformIslandSource()` — pure function
- `@lessjs/adapter-vite/island-transform`: Vite plugin reduced to thin wrapper
- 18 tests pass (6 core + 12 adapter)

## Dev Server

```bash
deno task dev:fast  # ~100ms cold start, zero Vite
```

- Deno.serve + Hono, `www/app/dev-server.ts`
- Static asset middleware, CORS, health check
- Existing Vite HMR (`deno task dev`) preserved

## SSG Fixes

- Adapter stubs (DsdReactElement/DsdLitElement/DsdVanillaElement): class → real class stub
- `react-showcase.ts`: `typeof WithDsdHydration === 'function'` guard
- `WithDsdHydration` export added to adapter-react SSR stub

### Post-Release Cleanup (SOP-006)

- Removed dead `VIRTUAL_PAGE_DATA_ID` from protocols (zero handler)
- `renderEnd_timeFallback` → `renderEndTimeFallback` (camelCase)
- New `ssr-data-stubs.ts` — single source for empty data stubs
- `HydrateEventDescriptor` deprecation updated to reflect actual v0.27 target
- `DsdElement.static client` and `DANGEROUS_KEYS` marked `@internal`
- `island-transform` tagName logic unified with `route-scanner` (separator→hyphen)
