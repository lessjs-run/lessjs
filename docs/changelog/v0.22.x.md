# v0.22.x Changelog

> Architecture Integrity — [ADR-0049](../adr/ADR-0049-architecture-debt-first-roadmap-reset.md)

## v0.22.0

### Changed

- **Consumer Surface Cleanup** ([SOP-001](../sop/v0.22.0/SOP-001-consumer-surface-cleanup.md)): Generated projects now import only `@deno/vite-plugin` + `@lessjs/app` + `@lessjs/core` + `@lessjs/ui` (4 imports, down from 18). Framework transitives (parse5, entities, hono) resolved by ADR-0047 pre-resolution.
- **Package Boundary Repair** ([SOP-002](../sop/v0.22.0/SOP-002-package-boundary-repair.md)): `build-types.ts` and `virtual-ids.ts` moved from `@lessjs/core` to `@lessjs/adapter-vite`. StyleSheet de-duplicated (canonical: `@lessjs/style-sheet`, core re-exports). UI escape helpers extracted to `shared/escape.ts`.
- **Adapter Decomposition** ([SOP-003](../sop/v0.22.0/SOP-003-adapter-vite-decomposition.md)): `@lessjs/adapter-vite` index.ts split from 841→62 lines into 5 modules: `less-plugin.ts`, `head-injection.ts`, `subpath-resolver.ts`, `optional-package-stubs.ts`, `phase-context.ts`.
- **Signals: alien-signals default engine** ([SOP-004](../sop/v0.22.0/SOP-004-signals-schema-validation-hardening.md)): `alien-signals` (1.6KB, Vue 3.6 core algorithm) is now the sole signal engine. TC39 polyfill (`polyfill.ts`, 413 lines) removed. `framework.ts` simplified from 139→20 lines.
- **Signals: sugar layer removed**: `islandEffect`, `channel`, `themeSignal`, `batch`, `untracked`, `isNativeSignal` — all dead code, zero consumers. `Channel`/`ChannelHandler` types removed. `@lessjs/signals` package: 903→319 lines (-65%).
- **Schema Unification**: Hub `CompatibilityTier` now imports from `@lessjs/core` (single source of truth).
- **Hub Scanner Fix**: `records.push(null!)` replaced with typed pre-allocation.

### Removed

- TC39 Signal polyfill (`packages/signals/src/polyfill.ts`, 413 lines)
- Sugar layer (`packages/signals/src/sugar.ts`, 188 lines)
- `@lessjs/core` no longer exports build contracts (`build-types`, `virtual-ids`)
- `@lessjs/ui` no longer has duplicate `_esc`/`_escAttr` implementations per component
- `@lessjs/style-sheet` no longer has duplicated implementation in core

### Upgrade Notes

- Consumer `deno.json` imports reduced from 18→4. Existing projects: remove `parse5`, `entities`, `hono`, `@lessjs/signals/framework`, `@lessjs/core/navigation`, `@lessjs/ui/` from imports.
- `build-types` and `virtual-ids` imports should use `@lessjs/adapter-vite/build-types` and `@lessjs/adapter-vite/virtual-ids`.
- `signal()`/`computed()`/`effect()` API unchanged. Underlying engine is now alien-signals — no behavior change for consumers.
- `islandEffect`, `channel`, `themeSignal`, `batch` are removed. These had zero consumers in LessJS workspace and zero JSR consumers (0.22.0 not yet published).

### Dead Code Audit (v0.22.x)

- `entry-renderer.ts:19` — TODO: shared types extraction (P2, v0.23)
- `route-scanner.ts:326` — TODO: AST replace regex (P3, low priority)

## Breaking Changes

- `Channel`, `ChannelHandler` types removed from `@lessjs/signals`
- `islandEffect`, `channel()`, `themeSignal`, `batch()`, `untracked()`, `isNativeSignal()` removed
- `build-types` and `virtual-ids` moved from `@lessjs/core` to `@lessjs/adapter-vite`
