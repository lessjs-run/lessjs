# v0.15.3 — DSD Report + Release Gate

> **Release type**: patch / release-readiness
> **Date**: 2026-05-16

## Summary

v0.15.3 makes SSG output observable via `dsd-report.json` and establishes a repeatable release gate. The renderer kernel is now stable enough to serve as the foundation for v0.16.

## Changes

### Scope 1: `dsd-report.json` in SSG output

- **`packages/adapter-vite/src/cli/ssg-render.ts`**: After SSG rendering, aggregates per-page `RenderOutput` data (errors, hydration hints, metrics) and writes `dsd-report.json` to the output directory. Report is deterministic and machine-readable.
- **`packages/adapter-vite/src/cli/ssg-render.ts`**: `SsrBundle.renderRoute()` now returns `SsgPageOutput` (structured type with `html`, `errors`, `hydrationHints`, `componentCount`, `renderTimeMs`) instead of bare `string`. Backward compatibility maintained via `collectPageOutput()` which handles both old string returns and new structured returns.
- **`packages/adapter-vite/src/entry-renderer.ts`**: Generated `renderRoute()` function now returns structured output `{ html, errors, hydrationHints, componentCount, renderTimeMs }` derived from `renderDSDByName()`'s `RenderOutput`.

### Scope 2: Build report type definitions

- **`packages/core/src/types.ts`**: New types:
  - `DsdPageDiagnostics` — per-page render diagnostics
  - `DsdMetricsSummary` — aggregated metrics across all pages
  - `DsdHydrationHintSummary` — aggregated hydration hint counts
  - `DsdBuildReport` — full report written to `dsd-report.json`
- **`packages/core/src/index.ts`**: Exported new types.

### Scope 3: Integration test

- **`packages/adapter-vite/__tests__/ssg-report.test.ts`**: New test file with 3 tests (9 steps):
  - `dsd-report.json` is produced after `ssgRender()`
  - Report shape is valid with all required fields
  - Diagnostics collected from `renderRoute()` output
  - Error counts and metrics are not silently dropped
  - Backward compat with string-only `renderRoute` returns

## Exit Criteria

- ✅ Renderer output is observable in SSG builds (`dsd-report.json`)
- ✅ CI/release can fail on render errors if project policy chooses to
- ✅ v0.15.x can be tagged as the stable renderer-kernel foundation

## Breaking Changes

- `SsrBundle.renderRoute()` return type changed from `Promise<string>` to `Promise<SsgPageOutput>`. This is an internal API used by the SSG pipeline only. The `collectPageOutput()` helper provides backward compatibility for string returns.
