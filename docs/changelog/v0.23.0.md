# v0.23.0 - 2026-05-27

## Summary

v0.23.0 ships the Layered Package Architecture line from ADR-0050: shared
contracts move to protocols, component authoring moves to runtime, app remains a
configuration facade, and package graph reality becomes a release gate.

## Changes

- **Protocols ownership**: `@lessjs/protocols` is the canonical owner for shared
  build context contracts and virtual module ids consumed by `adapter-vite`,
  `content`, and `i18n`.
- **Runtime facade**: `@lessjs/runtime` is the authoring facade for generated
  components, exporting core runtime APIs, `StyleSheet`, and signal helpers
  without turning `@lessjs/core` into a convenience barrel.
- **App facade cleanup**: `@lessjs/app` remains the `lessjs()` configuration
  facade and no longer imports `@lessjs/adapter-vite/build-context`.
- **Generated project contract**: `@lessjs/create` templates use
  `@lessjs/runtime`, and generated import maps declare their direct runtime,
  app, UI, and Vite imports.
- **Adapter contract cleanup**: stale adapter-owned exports for old
  `build-types` and `virtual-ids` ownership paths were removed.
- **Package graph gate**: `deno task graph:check` now validates cycles, publish
  order, unified versions, workflow coverage, and source-level direct
  `@lessjs/*` import declarations per package.
- **Publish workflow**: the JSR publish workflow now includes all 18 packages,
  including `@lessjs/runtime`, and publishes dependency owners before packages
  that import them.
- **Architecture docs**: README, status, roadmap, SOP v0.23.0, `docs/arch/`,
  and this changelog align on v0.23.0 as the active implemented architecture
  line.
- **WWW 0.23.0 surface**: the documentation site now presents the 0.23.0
  architecture line as an artifact-first product surface, with homepage,
  architecture, roadmap, and changelog pages aligned to the layered package
  architecture decision.
- **WWW i18n/search/theme controls**: the updated 0.23.0 `www` pages keep the
  shared language switcher, command-search island, and theme toggle wired
  through `less-layout` instead of treating the redesigned pages as isolated
  static surfaces.
- **CI E2E browser install**: the SOP E2E gate now installs Playwright Chromium
  through the same Deno npm specifier used by `deno task test:e2e`, avoiding
  mismatched browser cache directories on GitHub Actions.
- **WWW _renderer.ts DRY**: 8 identical `_renderer.ts` files have been replaced
  with a single shared factory at `www/app/shared/section-renderer.ts`. The
  `'LESS IS MORE'` magic-string footer injection has been removed in favor of
  `less-layout`'s native `edit-url` attribute.
- **WWW mobile-menu.js removal**: the external `www/public/mobile-menu.js`
  script (42 lines) has been merged into `less-layout.ts` via a
  `_setupBackdropClose()` handler, eliminating a shadow-DOM encapsulation leak
  through `composedPath()`.
- **Theme toggle robustness**: `less-theme-toggle` now defers `update()` during
  DSD hydration via `queueMicrotask`, ensuring `_bindTemplateRuntime` attaches
  `@click` event handlers after the shadow DOM is fully settled.
- **Theme consistency**: `theme-init.js` defaults to `dark`, matching
  `less-theme-toggle`'s signal default and preventing FOUC on first load.
- **I18n navigation fix**: `less-layout._loadContent()` now syncs the `locale`
  attribute from the fetched page, correcting language content after
  client-side navigation.
- **Search island fix**: 9 section renderers now import `less-search.js`,
  restoring search functionality on non-homepage routes.
- **Publish workflow**: the JSR publish workflow has been renamed to
  `publish-jsr.yml` to fix a GitHub Actions name-parsing issue.

### v0.23.0 P0 Bug Fixes (2026-05-27)

- **DSD hydration event binding fix** (P0, SOP-007): The root cause was NOT in
  `dsd-element.ts` — `render-dsd.ts` already uses `renderTemplateToString(result,
  {runtimeMarkers: true})`. The issue was that `section-renderer.ts` and
  `registry/_renderer.ts` bypassed the standard SSG pipeline by injecting
  hand-written SEARCH_DSD strings without markers. Fix: `less-layout` now renders
  `<less-search>` directly in its shadow DOM. The SSG's nested component
  rendering pipeline (`renderNestedCustomElements` → `renderDSD`) handles it
  automatically with correct runtime markers. Removed all manual SEARCH_DSD
  injection code (+11/-64 lines). See SOP-007.
- **Search universal placement**: `less-layout` renders `<less-search>` by
  default (controlled by `no-search` attribute). All pages now get search
  automatically — no need to manually slot it. Removed `<less-search>` slot
  elements from index, changelog, roadmap, 404 pages.
- **Duplicate search on engine/architecture**: removed inline `<less-search>`
  element and import.
- **Imperative workaround removal**: `less-search.ts` no longer uses
  imperative `addEventListener`. The `@click` binding in `render()` now works
  through the standard DSD hydration pipeline.
- **Island theme tokens**: Added `openPropsTokenSheet` to shoelace-showcase,
  reactive-showcase, less-term, and less-toc islands so `data-theme` switching
  properly updates CSS custom properties in their shadow DOMs.
- **Global theme CSS fix**: `vite.config.ts` now maps `:host { }` selectors from
  `openPropsTokenSheet` to `:root, :host { }` so CSS custom properties are
  available to regular DOM elements outside shadow trees. Previously, `:host`
  rules were silently ignored in the global context.
- **contributing.ts cleanup**: Removed duplicate `pageStyles` application
  (`pageSheet` + `routeSheet` both contained `pageStyles`). Merged into
  single `routeSheet`.
- **404.ts dead code**: Removed unused `pageSheet` import and instantiation.
- **nav-filter.ts**: Added `@deprecated` JSDoc to legacy aliases
  (`filterFrameworkNav`, `filterEngineNav`, `filterRegistryNav`).
- **Home nav restoration**: `{ href: '/', label: 'Home' }` restored to
  `headerNav` after CF deployment verification test.
- **SOP-007**: Added standardized procedure for DSD hydration event binding
  verification and cleanup.
- **v0.23.0 audit**: Full framework reliability and cleanliness audit
  completed (2 P0 / 8 P1 / 7 P2 findings, weighted score 6.0/10).

## Breaking Changes

- Shared build contracts are no longer exported from adapter-owned paths.
  Use `@lessjs/protocols/build-types` and
  `@lessjs/protocols/virtual-ids`.
- Ordinary component authoring examples and generated projects should use
  `@lessjs/runtime` instead of treating `@lessjs/core` as the product-level DX
  barrel.
- Removed wrong-owner root aliases and stale adapter-vite exports for old
  contract locations.
