# v0.16.0 — WC Package Protocol

**Release date**: 2026-05-16

## Summary

Introduces the CEM-compatible `LessPackageManifest` and local `LessRegistry`,
turning the 3-field `PackageIslandMeta` into a structured, tool-consumable
package protocol. LessJS can now describe WC packages as data instead of only
executing package code.

## What's new

### Manifest type system

- `LessPackageManifest` — top-level package descriptor (20+ fields)
- `LessDeclaration` — custom element declaration with CEM-compatible metadata
- `LessElementExtensions` — SSR/DSD/hydration declarations per component
- `LessPackageExtensions` — package-level LessJS configuration
- CEM-compatible subtypes: `LessAttribute`, `LessMember`, `LessEvent`,
  `LessSlot`, `LessCssProperty`, `LessCssPart`, `LessExport`, `LessModule`
- `ValidationResult` / `ValidationError` / `ValidationWarning` — structured
  validation output
- `RegistryIndex` / `RegistryIndexEntry` — aggregated registry index

### Local registry + validation

New `packages/core/src/registry.ts` with 6 public APIs:

- `registerManifest(manifest)` — register a package manifest
- `getByTagName(tagName)` — look up a declaration by tag name
- `getAllManifests()` — get all registered manifests
- `validateManifest(manifest)` — validate with 7 rejection rules
- `generateIndex()` — generate aggregated registry index
- `clearRegistry()` — clear all manifests (for testing)

Validation rejects: missing required fields, invalid custom element names,
unsafe module paths, invalid layer/hydrate strategy, duplicate tag names
within a package, unresolved export references, and registry-level tag
conflicts (as warnings).

### Backward compatibility adapter

- `packageIslandFromManifest(manifest)` — converts `LessPackageManifest` to
  legacy `PackageIslandMeta[]` format
- `PackageIslandMeta` marked as `@deprecated` (removal target: v0.18+)

### @openelement/ui manifest

- New `packages/ui/src/manifest.ts` with full CEM-compatible manifest for
  all 7 UI components
- `islands` array in `index.ts` now generated from manifest via
  `packageIslandFromManifest()`
- Backward-compatible: `islands` output matches v0.15.x exactly

### Roadmap documentation

- New `docs/roadmap/ROADMAP.md` — global 6-phase roadmap
- New `docs/roadmap/v0.16.md` — patch-level detail for v0.16
- Updated `www/app/routes/roadmap.ts` — public roadmap page reflects
  6-phase vision with v0.15 as Done and v0.16 as Next
