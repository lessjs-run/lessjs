# v0.37.6 Changelog

## Summary

v0.37.6 proves the Vite + Nitro runtime path prepared by the
protocol-first architecture. It mounts the openElement universal request
handler in real Nitro Node and Cloudflare Workers output while preserving
openElement ownership of application, route, render, island, and cache intent
semantics.

## Changed

- Added a real Nitro proof fixture under `fixtures/nitro-proof/`.
- Extended the Nitro proof harness to build and inspect `node-server` and
  `cloudflare-module` output.
- Verified the mounted openElement Web `Response` path through Nitro output.
- Verified static routes keep the zero-framework-JS default.
- Verified explicit islands and client-only behavior reference only the
  expected client assets.
- Verified public asset serving, `load()` data, layout composition, redirects,
  not-found, error handling, and API routes through the Nitro-mounted handler.
- Mapped `/isr` cache intent to Nitro route rules while keeping openElement as
  the semantic owner of the cache contract.
- Made the Chromium E2E release gate single-worker by default to avoid local
  Windows worker-spawn instability and match CI stability expectations.
- Updated v0.37.6 status, roadmap, NextVersion, proof plan, test matrix, and
  release checklist evidence.
