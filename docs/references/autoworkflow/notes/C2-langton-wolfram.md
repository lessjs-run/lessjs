# C2. Langton 与 Wolfram：分类、涌现与计算

> Langton, C. G. "Self-Reproduction in Cellular Automata." _Physica D_, 1984.
> Wolfram, S. _A New Kind of Science._ Wolfram Media, 2002.
> （Wolfram 不是传统同行评议期刊——但他对 CA 的分类工作是领域标准）

## 两条路线

von Neumann 之后，细胞自动机研究分裂成两条路：

### 路线 A：简化自复制（Langton）

von Neumann 的 29 状态机太复杂。有没有更简单的自复制系统？

Langton 找到了一个**环形自复制机**：

```
┌────────────────────────────┐
│ 一个简单的环形结构，只依赖 8 状态 │
│ 环上的数据既是"指令"又是"身体"   │
│ 环向外延伸 → 形成另一个环      │
└────────────────────────────┘
```

关键发现：**自复制不需要 29 种状态。8 种就够了。简单规则可以产生自复制行为。**

这对我们的启示：**Cell 的类型不需要很多。低风险（bump/changelog/fmt）三类可能就覆盖 90% 的机械操作。**

### 路线 B：分类与计算普遍性（Wolfram）

Wolfram 不关心自复制。他关心：**给定一组细胞自动机规则，能产生多复杂的宏观行为？**

## Wolfram 的四类分类

这是 Wolfram 最重要的贡献：

| 类别        | 行为     | 例子                    | 最终状态               |
| ----------- | -------- | ----------------------- | ---------------------- |
| Class 1     | 死寂     | 所有细胞变成同一状态    | 均匀静止               |
| Class 2     | 周期     | 重复模式                | 稳定的振荡             |
| Class 3     | 混沌     | 没有明显模式            | 伪随机噪声             |
| **Class 4** | **复杂** | **局部结构 + 长程交互** | **不可预测的持续演化** |

Class 4 是最有趣的：在"有序"和"混沌"的**边缘**，系统产生持续的、不可归约的复杂行为。

**Rule 110**（一种 Class 4 的一维细胞自动机）被证明是**图灵完备的**——它能做任何可计算的事情。

## 计算等价原理 (Principle of Computational Equivalence)

Wolfram 的核心主张：

> 几乎所有不是明显简单的系统，计算能力都是一样的——图灵完备。

一个蚂蚁群落、一场飓风、人脑、Rule 110——它们的计算能力完全等价。不是因为它们相似，是因为它们都能达到计算的极限。

对 AI 的意义：

> 如果 Rule 110 这么简单的规则都能达到图灵完备，
> 那么把 AI + State Machine + Git + Harness 拼起来，
> 几乎一定会产生比单独组件复杂得多的涌现行为。

**不是设计出来的。是涌现出来的。**

## 对 openElement 的启示

### 1. 我们追求 Class 4 行为

Class 1 = 每次发布都一潭死水。\
Class 3 = 系统疯狂改代码，不可预测。\
Class 4 = **持续演化，有模式，但不可归约为简单周期。**

这恰好是我们想要的：一个不断更新版本、文档、代码的系统，但不是随机乱改——有 Governance 文档约束方向，有 Harness 约束质量。

### 2. 复杂性来自交互，不是来自设计

Wolfram 最深刻的洞见：**Class 4 不需要复杂规则。Rule 110 只有 8 条规则。**

我们的 Cell 类型不需要很多。规则不需要复杂。复杂行为从 Cell × State Machine × DAG × Harness 的交互中涌现。

### 3. 不可预测 ≠ 不可控制

Class 4 的长程行为不可预测——这是真的。但这不意味着失控。

Harness 是 guardrail。不管涌现出什么复杂行为，fmt/lint/typecheck/test 门前一视同仁。**你可以不可预测，但你不能不合规。**
