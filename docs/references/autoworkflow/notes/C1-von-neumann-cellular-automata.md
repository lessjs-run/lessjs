# C1. von Neumann 细胞自动机：自复制系统的起源

> von Neumann, J. _Theory of Self-Reproducing Automata._ Edited by A. W. Burks. University of Illinois Press, 1966.
> （遗作，von Neumann 1952-1957 年间的研究，死后由 Burks 整理出版）

## 来处

1940 年代末，von Neumann 思考一个问题：

> "机器能自己复制自己吗？"

这不仅仅是哲学问题。他想要一个**严格的数学证明**——到底什么条件下一个系统可以自我复制。

## 细胞自动机的定义

von Neumann 发明了细胞自动机（Cellular Automaton）作为思考工具：

```
一个无限网格。每个格子是一个"细胞"。
每个细胞有有限个状态（比如 0 或 1）。
每个细胞的下一状态只取决于自己和邻居的当前状态。

规则是局部的、统一的、确定的。
但宏观行为可以极其复杂。
```

关键约束：

- **局部性**：每个细胞只看到邻居。没有全局视野。
- **均匀性**：所有细胞遵守同一组规则。没有特殊细胞。
- **确定性**：给定状态 + 规则 → 下一状态唯一确定。没有随机。

## 29 状态自复制机

von Neumann 设计了一个 29 状态、约 20 万个细胞的构造，在数学上证明了：

> 存在一个细胞自动机配置，可以从空白空间中**完整地复制自己**，包括复制指令。

三个部分：

```
┌──────────────┐
│ 构造臂        │ ← 读取指令带 → 按指令操作
├──────────────┤
│ 指令带        │ ← "怎么造一个我"的完整描述
├──────────────┤
│ 复制机        │ ← 把指令带复制一份到子代
└──────────────┘
```

关键洞察：**指令带必须同时包含"构造指令"和"复制指令"。** 如果只包含构造指令，子代就会丢失复制能力——第一代能造子代，子代不能造孙代。

这恰好对应我们架构中的问题：

```
Cell 能执行代码 → 但 Cell 能不能创建新的 Cell？
                                ↓
                 必须能。否则演化到一半就停了。
                 Cell 的"指令带"必须是自包含的。
```

## 对 openElement 的直接意义

### 1. 细胞 = 局部规则

von Neumann 的细胞不知道自己在造一个完整的机器。它只知道"我邻居是这个状态，所以我下一状态应该是那个"。

我们的 Cell 也应该是局部的：

```
Cell-01（bump version）不需要知道 Cell-08（发布到 JSR）的存在。
它只需要知道："我完成了，SOP 任务 #1 打勾"。

宏观的版本发布行为，从局部 cells 的简单规则中涌现。
```

### 2. 通用构造器 vs 通用计算机

von Neumann 区分了：

| 概念                               | 能力               |
| ---------------------------------- | ------------------ |
| 通用计算机 (Universal Computer)    | 给定程序，算出结果 |
| 通用构造器 (Universal Constructor) | 给定蓝图，造出实物 |

我们做的是通用演化器，不是通用编程器。OpenHands 是"通用计算机"——给任务写代码。我们要的是"通用构造器"——给蓝图自己演化。

### 3. 自描述的递归

von Neumann 的自复制机有一个悖论：

> 指令带必须描述"包含指令带在内的整个机器"。

这就是著名的**自指涉悖论**——描述不能比被描述的东西更小。

Goedel, Turing, von Neumann — 三个人从不同角度撞上了同一个问题：系统如何谈论自己。

我们面对的是同一个问题：**Governance 文档（系统自描述）能不能完整描述系统的演化规则？**

如果不行，就永远有一部分规则在人脑子里，AI 读不到。
