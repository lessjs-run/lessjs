# v0.29.4 - Architecture Debt Closure

Release date: 2026-06-03\
Changelog: [v0.29.4](../changelog/v0.29.4.md)\
SOP: [SOP-010](../sop/v0.29.4/SOP-010-architecture-debt-closure.md)\
Supersedes: [v0.29.3](./v0.29.3.md)

v0.29.4 closes the last two architecture debts before JamFullStack work.

## TL;DR

The rendering pipeline is now architecturally complete:

- `callComponent()` filters prototype pollution keys (matching SSR's existing guard).
- `renderDsd()` accepts `lightDom` at the IR level — no more fragile string
  manipulation on rendered HTML.
- `mergeDsdHostHtmlWithLightDom` is deleted.

## What Changed

### callComponent prototype pollution filter

```ts
// Before: props assigned directly, no filter
for (const [key, value] of Object.entries(props)) {
  (instance as Record<string, unknown>)[key] = value;
}

// After: dangerous keys skipped
import { DANGEROUS_KEYS } from './security.ts';
for (const [key, value] of Object.entries(props)) {
  if (DANGEROUS_KEYS.has(key)) continue;
  (instance as Record<string, unknown>)[key] = value;
}
```

### renderDsd lightDom at IR level

```ts
// Before: renderDsd returns HTML string, caller does string manipulation
const dsdResult = await renderDsd(tagName, { props });
return mergeDsdHostHtmlWithLightDom(dsdResult.html, tagName, childNodes);

// After: lightDom passed to renderDsd, handled at IR level
const dsdResult = await renderDsd(tagName, { props, lightDom: childNodes });
return trustedHtmlNode(dsdResult.html);
```

## What Did Not Change

- No public API changes.
- `renderDsd()` signature is backward-compatible.
- No rendering behavior changes.
