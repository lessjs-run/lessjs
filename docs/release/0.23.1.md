# LessJS v0.23.1 — Patch: SSG Package Resolver Runtime Fix

**2026-05-27** | v0.23.1 | Patch | SOP-016

v0.23.1 is a targeted patch that fixes the v0.23.0 CI failure cascade.

## What Broke

v0.23.0's push to `main` triggered 3 CI workflow failures:

| Workflow                        | Root Cause                                                |
| ------------------------------- | --------------------------------------------------------- |
| Lint & Format                   | `deno fmt` not run on ADR-0052 and ADR README before push |
| SOP Gate v0.21.x                | Same — `fmt:check` failed                                 |
| Publish to JSR (consumer-smoke) | `@lessjs/runtime` not in SSG package resolver whitelist   |

## What We Fixed

### 1. `deno fmt` (14 files)

Ran `deno fmt` on 12 docs files and 1 e2e test file that were not formatted.
This restores the Lint & Format and SOP Gate checkpoints.

### 2. SSG Package Resolver Runtime Entry (P0)

**Root cause**:
`packages/adapter-vite/src/ssg-package-resolver.ts` maintains a whitelist
(`DEFAULT_LESSJS_PACKAGES`) of `@lessjs/*` packages that the SSG build can
resolve during consumer project builds. `runtime` was missing from this list.

When a consumer project (created via `@lessjs/create`) ran SSG/SSR build:

```
import { DsdElement, html, signal, StyleSheet } from '@lessjs/runtime';
→ createLessJsrPackageResolverPlugin returns null (not in whitelist)
→ Rolldown falls back to Node.js resolution
→ Unresolved import → build fails
```

**Fix**: Added `'runtime'` to both `DEFAULT_LESSJS_PACKAGES` and
`LESSJS_EXPORT_FILES` in `ssg-package-resolver.ts`.

### 3. Unified Version Bump (17 packages → 0.23.1)

The SSG resolver uses the adapter-vite's own version to resolve all `@lessjs/*`
packages on JSR. All packages must therefore share the same version number.

Bumped all 17 packages from `0.23.0` to `0.23.1`.

## Maintenance Rule Added

> When adding a new `@lessjs/*` package, you MUST also add it to:
>
> 1. `DEFAULT_LESSJS_PACKAGES` in `packages/adapter-vite/src/ssg-package-resolver.ts`
> 2. `LESSJS_EXPORT_FILES` in the same file
> 3. `PKG_DIR_MAP` in `packages/create/cli.ts` (if used in project templates)
