# v0.29.1 - Core Simplification

Release date: 2026-06-03\
Changelog: [v0.29.1](../changelog/v0.29.1.md)\
ADR: [ADR-0078](../adr/ADR-0078-core-simplification-and-merge.md)\
SOP: [SOP-007](../sop/v0.29.1/SOP-007-core-simplification.md)\
Supersedes: [v0.29.0](./v0.29.0.md)

v0.29.0 introduced a structured renderer IR. v0.29.1 removes the compatibility
shortcuts kept in v0.29.0. The renderer now has **one attribute serializer**,
**one async render path**, and **one public rendering API**.

## TL;DR

LessJS core is leaner and more consistent:

- `core/src` drops from 33 files to 26 (-21%).
- The old sync/async dual-track renderer is merged into a single `renderToNode`.
- `renderToString` is removed — use `await renderDsdTree(node)` instead.
- `renderDsdByName` is removed.
- Attribute serialization is unified into one `serializeAttrs(tag, props)`.
- Seven thin or single-consumer source files are merged into their parent modules.
- A typo, a hardcoded version, and dead code are cleaned up.

## What Changed

### Single Render Path

The sync `renderToStaticNode` and async `renderToDsdNode` were two near-identical
VNode traversals (~100 lines of duplication). They are now one ~90-line async
function:

```ts
// Before: two paths, two functions
renderToString(node); // sync, no DSD
renderDsdTree(node); // async, with DSD

// After: one path
renderDsdTree(node); // async, handles everything
```

### Unified Attribute Serialization

```ts
// Before: two serializers with different camel-case rules
serializeAttributes(props); // used camelToKebab (DSD host)
serializeElementAttrs(props); // used className/htmlFor mapping (JSX)

// After: one serializer
serializeAttrs(tag, props); // handles both, decides kebab-case by tag name
```

### File Merge

| Removed                 | Merged into                      | Reason                                        |
| ----------------------- | -------------------------------- | --------------------------------------------- |
| `jsx-render-string.ts`  | `render-ir.ts`                   | Pure re-export (16 lines)                     |
| `custom-element.ts`     | `island.ts`                      | Single helper function                        |
| `api.ts`                | `types.ts`                       | Single interface                              |
| `prop-types.ts`         | `prop.ts`                        | Tight coupling, type-only                     |
| `render-errors.ts`      | `render-dsd.ts`                  | Single consumer                               |
| `render-instantiate.ts` | `render-dsd.ts`                  | Single consumer                               |
| `render-serialize.ts`   | `render-ir.ts` + `render-dsd.ts` | Attributes unified, remainder single-consumer |

### Quality Fixes

- `_hyrateExistingDom` → `_hydrateExistingDom` (typo).
- `hub/scanner.ts` hardcoded `'0.20.0'` → `'0.29.0'`.
- `trustRenderHtml` now warns once in dev mode about its trust-boundary semantics.
- Dead `_resolveRenderOutput` method removed from `dsd-element.ts`.
- Dead `renderErrorHtml` function removed from `render-dsd.ts`.
- Duplicate `isValidCustomElementName` in `registry.ts` replaced with
  `isValidTagName` from `tag-utils.ts`.

## Breaking Changes

### `renderToString` removed

```ts
// Before
import { renderToString } from '@openelement/core';
const html = renderToString(vnode);

// After
import { renderDsdTree } from '@openelement/core';
const html = await renderDsdTree(vnode);
```

### `renderDsdByName` removed

```ts
// Before
await renderDsdByName('my-tag', { count: 1 });

// After
await renderDsd('my-tag', { props: { count: 1 } });
```

### Subpath removals

- `@openelement/core/api` — `LessApiContext` is now in `@openelement/core`.
- `@openelement/core/render-serialize` — removed.
- `@openelement/core/render-errors` — removed.
- `@openelement/core/render-instantiate` — removed.

## What Did Not Change

- `renderDsd()` signature and return shape.
- `renderDsdTree()` signature (was already async).
- `DsdElement` base class API.
- AppShell protocol and config options.
- All 19 packages remain in the monorepo.
