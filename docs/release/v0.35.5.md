# v0.35.5 Release Note: Complete Configuration Deduplication & Standardization

Issue: Configuration deduplication across workspace\
CHANGELOG: [CHANGELOG.md](../../CHANGELOG.md)

v0.35.5 addresses all redundancy issues in the Deno workspace `deno.json` configurations,
consolidating duplicated settings into a unified template system with automated verification.

## Completed This Release

### 1. Subpath Deduplication (35 removed)

- Removed redundant subpath declarations from root `deno.json`
- Leverages Deno workspace automatic exports resolution
- Reduced root imports from 80 to 18 lines

### 2. Publish Configuration Standardization

- All 19 packages now use standard publish config
- Handled exceptions for `@openelement/hub` (mod.ts entry) and `@openelement/create` (cli.ts entry)
- Single source of truth in `tools/config-templates.ts`

### 3. Include/Exclude Configuration Standardization

- Unified `include: ["src"]` and `exclude: ["node_modules", "dist"]`
- Applied to 10 packages that were missing these configs
- Consistent across entire workspace

### 4. Tasks Configuration Standardization

- Unified `build` task: `deno check src/index.ts`
- Applied to 6 packages that were missing standard tasks
- Handled package-specific exceptions (core, hub, create, adapters)

### 5. Configuration Management Tools

- `tools/config-templates.ts`: Central configuration definitions
- `tools/verify-package-configs.ts`: Validation and auto-fix capability
- `deno task verify:configs`: Verify configuration compliance
- `deno task verify:configs:fix`: Auto-fix non-compliant configs

### 6. autoflow Integration

- New invariant: `I-PACKAGE-CONFIG-STANDARD`
- 9 invariants checked (was 8)
- Config verification wired into pre-commit hook and CI

### 7. Dependency Fix

- Added missing `sanitize-html` and `@types/sanitize-html` to root `deno.json`
- Fixes CI failures in fresh environments with `nodeModulesDir: "manual"`

## Benefits

- Single source of truth for all package configurations
- Automatic validation prevents configuration drift
- Easy to maintain and update standards
- Handles exceptions gracefully
- Reduced maintenance burden (from 35+ duplicates to 1 template)

## Enforcement

- **pre-commit**: 8 steps now include `verify:configs`
- **pre-push**: 5 gates unchanged
- **CI sop-gate.yml**: New `config-validate` job
- **autoflow**: `I-PACKAGE-CONFIG-STANDARD` invariant (error severity)

19 packages at 0.35.4 (no version bump — this is a tooling/config-only release).
