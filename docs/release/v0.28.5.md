# v0.28.5 - Consumer Resolver Patch

Release date: 2026-06-02\
Changelog: [v0.28.5](../changelog/v0.28.5.md)\
Supersedes: [v0.28.4](./v0.28.4.md)

v0.28.5 is a small release-line repair after the v0.28.4 AppShell Protocol
release.

The v0.28.4 package publish completed, but the post-publish consumer smoke test
failed in a fresh generated project. The failure was not in AppShell behavior:
the generated SSR bundle attempted to import bare `sanitize-html` from published
LessJS source. That worked inside the monorepo because the root import map had a
bare alias, but a fresh JSR consumer does not inherit package-local import maps.

## What Changed

- Published `sanitize-html` imports are now explicit
  `npm:sanitize-html@^2.17.4` specifiers.
- `@lessjs/core`, `@lessjs/adapter-vite`, and `@lessjs/content` no longer depend
  on a consumer project providing a `sanitize-html` import-map alias.
- All 19 packages are bumped together to `0.28.5` so the JSR package set remains
  coherent.

## Scope

This release does not change the AppShell API shipped in v0.28.4. It preserves:

- `appShell: 'default' | false | { tagName, import, props }`
- route-selected layouts via route `meta.layout`
- strict object-form `renderDsd(input, options)`
- parser-backed sanitizer behavior for raw HTML paths

## Verification Target

The required release evidence is:

- local full test suite
- package graph check
- publish dry-run over all 19 packages
- `dev` CI success
- `main` publish workflow success, including post-publish consumer smoke

This release note intentionally does not claim live publication until the main
workflow completes.
