# v0.37.4 Release Note

Hygiene + Pure CSS UI Foundation + Architecture Decoupling

## Highlights

- **CSS UI Foundation**: daisyUI class set as pure CSS via `CSSStyleSheet`,
  DSD shadow DOM compatible, Open Props token powered.
- **Router decoupled from UI**: `@openelement/ui` now 3 deps (was 5).
- **11 deprecated shells removed**: adapter-vite cleanup complete.
- **Core logger unified**: consistent scoped logging across core packages.
- **1600 tests, 0 failures**: +113 new tests across ssg, router, protocols.
- **autoflow:gate**: unified gate controller with impact surface analysis
  and LLM directives — single pre-commit entry point replaces 12 individual
  tasks.
- **Full www daisy adoption**: 36 files converted, zero legacy CSS.

## Packages

All 20 `@openelement/*` workspace packages are version-aligned at **0.37.4**.
Release closure still requires JSR publish recovery to make all 20 package
versions visible on JSR and pass post-publish consumer smoke.

Recovery status as of 2026-06-11: JSR package-level metadata currently shows
3/20 packages at `0.37.4`; version-level `_meta.json` may appear earlier and is
used only to stop hung `deno publish` processes after JSR accepts an immutable
version. Release closure still requires package-level `meta.json` to expose
`0.37.4` for all 20 packages and post-publish consumer smoke to pass against a
fresh `jsr:@openelement/create@0.37.4` consumer.

## Migration Notes

- If importing SSG APIs from `@openelement/adapter-vite`, migrate to
  `@openelement/ssg`.
- If using `@openelement/router` via `@openelement/ui`, add a direct
  dependency on `@openelement/router`.
