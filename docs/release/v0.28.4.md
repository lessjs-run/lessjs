## TL;DR

v0.28.4 makes LessJS application shells explicit. The framework no longer treats
`@openelement/ui/less-layout` as an unconditional outer wrapper for every SSG app.
Apps can now choose the default shell, no shell, a custom shell, or named
route-level layouts through `lessjs()` configuration.

This release also finishes the v0.28 cleanup track: stricter `renderDsd()`,
parser-backed raw HTML sanitization, structured Hub data loading, better adapter
diagnostics, zero native DSD error tolerance, and 19-package local release task
coverage.

## AppShell Is Now A Framework Protocol

Default behavior is still available:

```ts
lessjs({
  appShell: 'default',
});
```

Bare route rendering is now explicit:

```ts
lessjs({
  appShell: false,
});
```

Custom shells are configured directly:

```ts
lessjs({
  appShell: {
    tagName: 'site-layout',
    import: './app/components/site-layout.tsx',
    props: {
      siteName: 'Field Notes',
      footerText: 'Built with LessJS',
    },
  },
});
```

Route-level layouts are selected through route metadata:

```ts
export const meta = {
  layout: 'post',
};
```

```ts
lessjs({
  layouts: {
    default: {
      tagName: 'site-layout',
      import: './app/components/site-layout.tsx',
    },
    post: {
      tagName: 'post-layout',
      import: './app/components/post-layout.tsx',
    },
    bare: false,
  },
});
```

## Cleanup Highlights

- `renderDsd()` now uses only the object options contract.
- `rawHtml` in core renderers is sanitized through one parser-backed policy.
- Hub client-only admission consumes generated data as module data, not regex
  matches on TypeScript source text.
- Optional SSR adapter failures are visible when a package is present but broken.
- Native DSD non-recoverable errors must be zero unless a caller explicitly
  overrides the investigation threshold.
- `@openelement/adapter-lit` no longer exposes deprecated hydration base classes.
- Blog data generation is deterministic for `YYYY-MM-DD-*` filenames without
  frontmatter dates.
- Local release tasks cover the full 19-package workspace graph.

## Breaking Changes

`renderDsd()` positional arguments were removed:

```ts
// Before
await renderDsd('my-card', MyCard, { title: 'Hello' });

// After
await renderDsd('my-card', {
  componentClass: MyCard,
  props: { title: 'Hello' },
});
```

`@openelement/adapter-lit` no longer exports `DsdLitElement`, `WithDsdHydration`, or
`./dsd-hydration`. Use `DsdElement` from `@openelement/core` for new DSD components.
