# v0.32.0 Release Note: App Lifecycle Contract

Changelog: [docs/changelog/v0.32.0.md](../changelog/v0.32.0.md)\
SOP: [docs/sop/v0.32.0/README.md](../sop/v0.32.0/README.md)\
NextVersion package: [docs/next/v0.32.0/README.md](../next/v0.32.0/README.md)\
ADR: [ADR-0085](../adr/ADR-0085-app-lifecycle-contract.md)

v0.32.0 makes openElement's application lifecycle explicit. v0.31 gave app
authors `definePage()` and `defineIsland()`; v0.32 defines what happens around
those pages: params, load context, route metadata, page render context,
redirect, not-found, and page error fallback.

## Highlights

- `@openelement/app` now exports `redirect()` and `notFound()` lifecycle helpers.
- `definePage()` render functions receive `route` and `meta` context.
- `definePage({ error })` renders fallback UI through the same VNode pipeline.
- Generated SSR/Hono entries translate redirect and not-found controls at the
  framework boundary.
- SSG `renderRoute()` returns structured redirect and not-found output.
- `routeInfo` carries file path, revalidate, rendering intent, and streaming
  intent for v0.33.
- AutoWorkflow is now a repository-level gate through
  `deno task workflow:check`.

## Example

```tsx
import { definePage, notFound, redirect } from '@openelement/app';

export default definePage({
  title: 'Post',
  meta: { section: 'blog' },
  async load({ params, route }) {
    if (!params.slug) notFound();
    if (params.slug === 'old-post') redirect('/posts/new-post', 301);
    return { slug: params.slug, source: route.filePath };
  },
  render({ data, route, meta }) {
    return (
      <article data-section={String(meta.section)}>
        <h1>{data.slug}</h1>
        <p>Rendered from {route.filePath}</p>
      </article>
    );
  },
  error({ error }) {
    return <main>{String(error)}</main>;
  },
});
```
