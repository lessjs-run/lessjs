# v0.38.0 Release Note

Product Surface Reset

## Highlights

- openElement now has a documented v0.38 product surface that separates
  first-run product packages from advanced, internal, and archived surfaces.
- The retained first-run surface is `@openelement/create`, `@openelement/app`,
  `@openelement/app/vite`, `@openelement/runtime`, advanced
  `@openelement/core`, `@openelement/ui`, and `@openelement/protocols`.
- No new top-level package names were introduced. Candidates such as
  `@openelement/elements`, `@openelement/protocol`, and
  `@openelement/framework` remain deferred until an ADR and migration plan
  approve them.
- `@openelement/create` starter projects now hide protocol subpath aliases from
  new projects while keeping the proven `@openelement/app/vite` build path.
- `@openelement/rpc` is archived/advanced for this line, and ordinary API docs
  now steer users toward Hono plus platform `Request`/`Response` boundaries.
- `@openelement/ssg`, `@openelement/cem`, and
  `@openelement/compat-check` remain in the package graph as internal
  build/tooling surfaces instead of first-run product APIs.
- `@openelement/hub` remains archived for the v1 product map until a future
  roadmap entry and ADR restart that product scope.

## Migration Notes

- Existing projects can keep protocol subpath aliases if they use them. New
  projects should prefer the generated starter surface.
- Direct `@openelement/adapter-vite` build wiring should be treated as an
  advanced reference; ordinary projects should start from
  `@openelement/app/vite`.
- Direct `@openelement/ssg` usage is now documented as internal build
  infrastructure. The public app story runs through `@openelement/app` and
  `@openelement/app/vite`.
- Direct `@openelement/rpc` usage is archived/advanced and is not the v1
  first-run API story.

## Release Truth

- `dev` non-JSR CI must pass for the release-bump commit before merge or sync
  to `main`.
- `main` non-JSR CI must pass before tag and GitHub release publication.
- Local or CI JSR publish should be attempted as best-effort telemetry, but
  JSR publish success and registry visibility do not block version exit.
