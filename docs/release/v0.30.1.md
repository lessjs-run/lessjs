# v0.30.1 - Clean Architecture Sweep

Release date: 2026-06-04\
Changelog: [v0.30.1](../changelog/v0.30.1.md)\
ADR: [ADR-0081](../adr/ADR-0081-vnode-event-unification.md)\
SOP: [v0.30.1](../sop/v0.30.1/SOP-013-islands-fix-openelement-rename.md)\
Supersedes: [v0.30.0](./v0.30.0.md)

v0.30.1 is the cleanup release after the v0.30 architecture freeze. It does not
add a second renderer or preserve compatibility aliases. Its purpose is to make
the active repository match the frozen contract: one public openElement name,
one renderer path, one structured metadata vocabulary, and one gate-proven
current documentation story.

## TL;DR

- `openElement()` is the current app facade.
- UI tags and package subpaths use `open-*`.
- Dynamic island UI returns VNodes and uses JSX handlers.
- `data-on-*` and dynamic `data-signal-html` are not restored.
- `trustedHtml` remains only for pre-sanitized, non-interactive content.
- Structured metadata uses `openElement`.
- All 19 packages are `0.30.1`.
- `deno.lock` is tracked and aligned to the v0.30.1 workspace.

## Migration

Update the Vite app facade:

```diff
- import { lessjs } from '@openelement/app';
+ import { openElement } from '@openelement/app';

  export default defineConfig({
-   plugins: [lessjs()],
+   plugins: [openElement()],
  });
```

Update UI imports and tags:

```diff
- import '@openelement/ui/less-theme-toggle';
+ import '@openelement/ui/open-theme-toggle';

- <less-theme-toggle client:idle />
+ <open-theme-toggle client:idle />
```

Update metadata:

```diff
- export const less = { ssr: false, hydrate: 'idle' };
+ export const openElement = { ssr: false, hydrate: 'idle' };
```

Move interactive dynamic UI to JSX:

```diff
- <button data-on-click="_close">Close</button>
+ <button onClick={() => this.close()}>Close</button>
```

For trusted non-interactive HTML, keep the trust boundary explicit:

```tsx
<article innerHTML={compiledMarkdownHtml} trustedHtml />;
```

Do not use `trustedHtml` for live filters, buttons, links with framework
handlers, or island UI that should be represented as VNodes.

## Architecture Notes

The important v0.30.1 decision is that `data-signal-render` is an internal
bridge from signals to VNode rendering, not a new public directive language. The
public model remains simple: component state is represented by signals, render
functions return VNodes, and JSX event props bind handlers.

The release also removes stale current documentation that still presented old
LessJS v0.27 architecture as current. Historical ADRs, changelogs, and blog
posts remain available as archives, but current product docs now point at
`docs/arch/current-architecture.md`.
