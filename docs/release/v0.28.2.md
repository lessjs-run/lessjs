# v0.28.2 - Deprecated Purge + Hardening

Release date: 2026-06-02

Changelog: [v0.28.2](../changelog/v0.28.2.md)

## TL;DR

v0.28.2 is the cleanup release that removes the last deprecated adapter
compatibility helpers and hardens HTML, head injection, Hub scanning, CDN
loading, and SSR bundling before the v0.28.3 closure.

## What Changed

- Adapter registration is registry-first. The shared path is now
  `getDefaultRegistry().register(adapter)`, and isolated code uses
  `createAdapterRegistry()`.
- `renderDsd()` accepts a structured options object across the internal render
  pipeline, making the pre-1.0 render contract easier to freeze.
- Raw HTML is explicit and sanitized. `innerHTML` escapes by default; trusted
  HTML uses `rawHtml: true`.
- `headExtras` and `headFragments` are sanitized, structured injected assets
  validate URLs and attributes, and `less-layout` rejects unsafe link schemes.
- Hub snapshots default to jsDelivr via `LESS_CDN_BASE`, while `hub:scan` runs
  without all-permission `-A`.
- SSR builds stop bundling Lit and React by default, reducing the vanilla SSR
  bundle path and letting adapter-specific consumers opt in when needed.

## Migration

```ts
// Before
import { registerAdapter } from '@lessjs/core';
registerAdapter(adapter);

// After
import { getDefaultRegistry } from '@lessjs/core';
getDefaultRegistry().register(adapter);
```

For isolated tests or subprocess rendering:

```ts
import { createAdapterRegistry } from '@lessjs/core';

const registry = createAdapterRegistry();
registry.register(adapter);
```

## Package Version

coherent patch set after both SOP-003 and SOP-004 gates are satisfied.
