v0.24.0 is the DX Leap release. It transforms LessJS from "architecture 8.5,
DX 5" to "architecture 8.5, DX 8". Three ADRs drive the changes.

## Release Note

v0.24.0 delivers three major DX improvements for component authors:

1. **Write templates like a framework, not string concatenation.**
   `classMap()`, `when()`, `choose()`, `repeat()`, and `ref()` make
   `html\`\`` expressive enough for real-world UIs.

2. **Properties in 1 line.** `@prop() count = 0` replaces 10+ lines of
   boilerplate (getter, setter, `attributeChangedCallback`,
   `observedAttributes`, manual signal). Signal changes auto-update the DOM.

3. **Errors that help, not hide.** `ErrorBoundary` catches child render failures
   and shows fallback UI. All errors carry structured `code`, `severity`,
   `phase`, and `recoverable` metadata.

All features are available through a single import:

```ts
import { classMap, DsdElement, ErrorBoundary, html, prop, signal } from '@openelement/runtime';
```

## What Shipped

### SOP-009: html Template Strengthening (ADR-0051)

Five template helpers + template caching added to `packages/core/src/template.ts`
(from ~309 â†’ ~500 lines):

| Helper                          | Purpose                                                                |
| ------------------------------- | ---------------------------------------------------------------------- |
| `classMap(obj)`                 | Dynamic CSS classes (`{ active: true, disabled: false }` â†’ `"active"`) |
| `when(cond, t, f?)`             | Conditional rendering (eager + lazy)                                   |
| `choose(key, cases, fallback?)` | Multi-way conditionals                                                 |
| `repeat(items, keyFn, fn)`      | Keyed iteration with identity tracking                                 |
| `ref(callback)`                 | DOM reference directive (replaces `querySelector`)                     |

Template caching via `WeakMap<TemplateStringsArray, ParsedTemplate>` avoids
re-parsing static parts on re-render.

Type narrowing: `AttrValue`, `ContentValue`, `EventValue` replace the broad
`TemplateValue` union.

All helpers re-exported through `@openelement/runtime`.

### SOP-010: Signal-DOM Deep Integration (ADR-0052)

`@prop()` decorator reduces property declaration from 10+ lines across 4
locations to **1 line:**

```ts
count = 0;
disabled = false;
ariaLabel = '';
```

Features:

- Signal-backed accessors (via internal `PropSignal`)
- Microtask-batched DOM updates (via `ReactiveHost`)
- Attributeâ†”property sync with type conversion (String/Number/Boolean)
- `reflect` option writes property changes back to HTML attribute
- Auto `observedAttributes` registration
- Lifecycle cleanup via `disposeProps()` in `disconnectedCallback()`

New file: `packages/core/src/prop.ts`

### SOP-011: Unified Error Handling (ADR-0053)

Four-layer error architecture:

| Layer | Component       | Location                                                                              |
| ----- | --------------- | ------------------------------------------------------------------------------------- |
| 1     | Typed hierarchy | `errors.ts` â€” `LessError` â†’ `RenderError`/`NavigationError`/`BuildError` + subclasses |
| 2     | ErrorBoundary   | `error-boundary.ts` â€” catch child render failures, display fallback UI                |
| 3     | Propagation     | `SsrErrorContext` â€” SSR error accumulation + merge                                    |
| 4     | Telemetry       | `setErrorTelemetryHook()` / `reportError()` â€” global observer                         |

All errors carry: `code` (ErrorCode enum), `severity`, `phase`, `recoverable`, `cause`.
