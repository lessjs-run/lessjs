# v0.33.0 Release Note: AI-Readable API Foundation

CHANGELOG: [CHANGELOG.md](../../CHANGELOG.md)\
SOP: [docs/sop/v0.33.0/README.md](../sop/v0.33.0/README.md)\
NextVersion package: [docs/next/v0.33.0/README.md](../next/v0.33.0/README.md)\
ADR: [ADR-0086](../adr/ADR-0086-ai-readable-architecture-and-autoflow2-roadmap.md)

v0.33.0 makes the openElement Application API structured, single-path, and
machine-readable. v0.31 introduced `definePage()`; v0.32 added lifecycle
controls like `redirect()` and `notFound()`; v0.33 removes the shortcut forms,
introduces explicit `head`, `route`, and `renderIntent` intents, and gives
islands a named metadata helper.

This is the foundation for AutoFlow2. A tool (or AI assistant) can now inspect a
page or island and answer "what route shape does this page expect?", "what head
data does it contribute?", "is this island SSR-capable?", and "what rendering
mode does this page intend?" — all from structured declarations, not inference.

## Highlights

- `definePage()` requires canonical object-form with `route`, `head`,
  `renderIntent`, `load`, `render`, and `error` fields.
- `page.head` exposes `title`, `description`, `meta`, and
  `dangerouslyHeadFragments` — raw HTML only enters through the named boundary.
- `page.renderIntent` replaces top-level `rendering`, `streaming`, and
  `revalidate` shortcuts.
- `defineIslandConfig()` replaces object-literal island metadata
  (`export const openElement = { ... }`).
- App-level island options now expose `ssr?: boolean`.
- Old v0.31-v0.32 function-form pages and object-literal island metadata are
  rejected at runtime and scanner time.

## Migration Guide

### Pages

Before (v0.31):

```tsx
import { definePage } from '@openelement/app';

export default definePage({
  title: 'My Page',
  rendering: 'static',
  render: ({ data }) => <main>{data}</main>,
});
```

After (v0.33):

```tsx
import { definePage } from '@openelement/app';

export default definePage({
  route: { path: '/my-page' },
  head: { title: 'My Page' },
  renderIntent: { mode: 'static' },
  render: ({ data }) => <main>{data}</main>,
});
```

Fenced raw head content:

```tsx
head: {
  title: 'Title',
  dangerouslyHeadFragments: ['<script src="/analytics.js"></script>'],
}
```

### Islands

Before (v0.31):

```tsx
export const openElement = { ssr: true, hydrate: 'visible' };
```

After (v0.33):

```tsx
export const openElement = defineIslandConfig({ ssr: true, hydrate: 'visible' });
```
