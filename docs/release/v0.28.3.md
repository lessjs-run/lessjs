## TL;DR

v0.28.0 shipped "Contracts & Tokens" and left **47 audit findings** across hygiene, security, and cleanup. The **v0.28.1 → v0.28.3 arc** closes them in three focused releases: **hygiene first** (no code), **then deprecated purge + security hardening** (zero-net API), **then closure with MDX** (only feature add). v0.28.3 lands as **one coherent `0.28.3` package line** — all 19 packages aligned, no intermediate half-states. The pre-freeze runway is now clean for **v0.29.0-rc.1**.

## The Story

v0.28.0 merged on 2026-06-01 with stable architecture contracts, Open Props migration, and signal-driven islands. A blind 10-category scan afterward surfaced 47 findings (8 P0, 13 P1, 12 P2, 14 P3). Rather than pack all of them into one release, we split them by risk:

| Release | Role                                  | Risk profile                                                                   | Code changes |
| ------- | ------------------------------------- | ------------------------------------------------------------------------------ | ------------ |
| v0.28.1 | Hygiene convergence                   | Zero — docs/`.gitignore`/version anchors only                                  | 0 lines      |
| v0.28.2 | Deprecated purge + security hardening | Breaking change to internal adapter API, with `getDefaultRegistry()` migration | ~800 lines   |
| v0.28.3 | Closure + MDX feature                 | New public API (`@openelement/content/mdx`, `lessMdx()` plugin)                | ~2,500 lines |

The three SOPs were authored as plans first, then converted to execution records as the work landed. v0.28.2 and v0.28.3 ship as **one patch line** (`0.28.3`) because the lockfile and package alignment had to land together.

## v0.28.2 — Deprecated Purge + Hardening (SOP-003)

**Day 2-4. Internal API cleanup, security closure, SSR bundle reduction.**

### What Changed

#### Adapter Registry Purge

The module-level `registerAdapter` / `getAdapter` / `getRegisteredAdapters` exports from `@openelement/core` are **removed**. The shared path is now `getDefaultRegistry()`, and isolated code uses `createAdapterRegistry()`.

```ts
// Before
import { registerAdapter } from '@openelement/core';
registerAdapter(adapter);

// After
import { getDefaultRegistry } from '@openelement/core';
getDefaultRegistry().register(adapter);

// For isolated tests / subprocess rendering
import { createAdapterRegistry } from '@openelement/core';
const registry = createAdapterRegistry();
registry.register(adapter);
```

This is a **technical breaking change** but the migration is mechanical. No external consumer was using the module-level helpers — the v0.28.0 release note already declared them as "default registry for backward compatibility."

#### DSD Render Contract

`renderDsd()` now takes a structured `RenderDsdOptions` object across the internal render pipeline. Legacy positional handling is preserved during the pre-1.0 window so existing callers don't break beyond the explicit adapter-helper removal.

```ts
export interface RenderDsdOptions {
  mode?: RenderMode;
  ssrAdmissionPlan?: SsrAdmissionPlan;
  isHydrating?: boolean;
  isClient?: boolean;
  ctx?: LessRenderContext;
  adapter?: LessAdapter;
  experimental?: { signalTracking?: boolean };
}
export function renderDsd(vnode: VNode, opts: RenderDsdOptions = {}): string;
```

Stream rendering, JSX rendering, generated SSR entry code, and tests are all updated.

#### HTML, Signal HTML, and Head Safety

- **`innerHTML` escapes by default.** Trusted HTML uses explicit `rawHtml: true` and still routes through `sanitize-html`.
- **`data-signal-html` updates are sanitized** before DOM assignment.
- **`headExtras` and `headFragments` are sanitized** with structured `sanitize-html` policy. Rejects `on*` attribute names.
- **`less-layout` URL safety**: validates `href` / `src` against an allowlist (`http:`, `https:`, `mailto:`, `tel:`, `sms:`) — rejects `javascript:`, `data:`, `vbscript:`, `file:`.

#### Event Hydration

- **Expanded DOM event aliases**: `dblclick`, `focusin`, `focusout`, `mouseenter`, `mouseleave`, `pointerover`, `pointerout`, `pointerenter`, `pointerleave`, `pointerdown`, `pointermove`, `pointerup`, `pointercancel`.
- **Multi-handler binding preserved** on the same VNode during hydration (was a bug — `serializeEventMarkers` was only serializing the first event handler).

#### Hub and Build Cleanup

- **CDN resolution via `LESS_CDN_BASE`**. Default is `jsDelivr`. Other supported: `unpkg`, `esm.sh`, `self-hosted`. Privacy and SLA are the rationale.
- **`hub:scan` runs with least privilege**: `--allow-read --allow-write=. --allow-net=esm.sh,registry.npmjs.org --allow-env=DENO_DIR` instead of `-A`.
- **SSR `noExternal` reduced to default**: only LessJS workspace packages + `alien-signals`. Lit and React stay external unless explicitly opted in. This drops the vanilla SSR bundle significantly.
- **Generated TypeScript regex scanning for client-only tags** replaced with a dynamic `import()` of the generated module URL — no more regex parsing at runtime.
- **`LessBuildContext.reset()`** preserves user resolve aliases when phase metadata resets.

#### Rewrite Passes

- Removed v0.27/v0.26 implementation-anchor comments from touched runtime, build, and README files.
- Production `console.log` callsites rewritten to `console.info` or left only in tests, docs, examples, and generated fixtures.

### Impact

- Technical breaking change on adapter registry; migration is one-line for any user
- ~800 lines of code change
- 5 P1 security findings closed
- 1 P1 perf finding (SSR bundle) closed

Plus a new public export: `@openelement/protocol/validators`.

### MDX Through DSD

**MDX is build-time only.** No runtime MDX parser. No React dependency. No second HTML renderer. It enters the same VNode and `renderDsd()` pipeline as `.tsx` routes.

```ts
import { compileMdx } from '@openelement/content/mdx';

const mod = await compileMdx(source, { filepath: 'post.mdx' });
```

```ts
// vite.config.ts
import { lessMdx } from '@openelement/adapter-vite/plugin-mdx';
export default { plugins: [lessMdx()] };
```

Key design decisions (full ADR at [ADR-0072](../adr/ADR-0072-mdx-in-lessjs.md)):

- **Parser**: `@mdx-js/mdx@3.1.1` (de-facto standard, active maintenance, ESM-native)
- **`jsxImportSource: '@openelement/core'`** — uses LessJS's JSX runtime, not React's
- **No provider** — LessJS's context model is component-scoped, not tree-scoped
- **No plugin system** — v1.0+ consideration
- **Build-time only** — no runtime MDX parsing

Frontmatter uses the same `@openelement/content` manifest format as `.md` blog posts. Island embedding works as expected:

```mdx
---
title: My MDX Post
client:load: true
---

import { Counter } from '../islands/counter.tsx';

# Hello MDX

<Counter client:idle />
```

The docs site has a small example at `www/content/mdx/example.mdx` and a guide page at `/guide/mdx`.

## Migration

### From v0.28.0 to v0.28.3

#### Adapter registry (required for any consumer using the old module-level helpers)

```diff
- import { registerAdapter, getAdapter } from '@openelement/core';
- registerAdapter(myAdapter);
+ import { getDefaultRegistry } from '@openelement/core';
+ getDefaultRegistry().register(myAdapter);
```

For isolated tests or subprocess rendering, use the factory:

```ts
import { createAdapterRegistry } from '@openelement/core';
const registry = createAdapterRegistry();
registry.register(myAdapter);
```

#### `innerHTML` (required if you relied on the implicit trust)

```diff
- host.innerHTML = userContent; // unsafe in v0.28.0
+ host.innerHTML = userContent; // safe by default in v0.28.3 (escaped)
+ host.innerHTML = trustedHtml; // use rawHtml: true or pass through sanitize-html explicitly
```

#### `renderDsd` (optional — old positional form still works during pre-1.0 window)

```ts
// New (recommended)
renderDsd(vnode, { mode: 'ssr', isHydrating: false });

// Old (still works, deprecated, will be removed at v1.0.0)
renderDsd(vnode, 'ssr', undefined, false, false, ctx, adapter);
```

#### MDX (new — opt-in)

Add `.mdx` content to your `www/content/` directory and include the plugin in your Vite config. No changes to existing `.tsx` routes.

```ts
// vite.config.ts
import { lessMdx } from '@openelement/adapter-vite/plugin-mdx';
export default { plugins: [lessMdx()] };
```

### What Did Not Change

- DSD rendering pipeline (single render path preserved)
- `DsdElement` lifecycle
- Signal integration
- File-based routing
- Hub protocol / manifest schema
- JSX runtime API
