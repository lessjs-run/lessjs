# v0.28.3 - Cleanup Closure + MDX

Release date: 2026-06-02

Changelog: [v0.28.3](../changelog/v0.28.3.md)

## TL;DR

v0.28.3 turns the v0.28 cleanup work into a real package release: the deferred
test floors are in place, dependency drift is removed, MDX compiles through the
existing LessJS DSD path, and every package is aligned to `0.28.3`.

## What Changed

### Test Floor

Router, runtime, and protocols now have first-class unit coverage instead of
being deferred to a later v1.0 patch train. The new tests cover route pattern
translation, locale path helpers, runtime facade exports, JSX runtime behavior,
manifest validation, version negotiation, and protocol failure cases.

### MDX Through DSD

MDX is build-time only. `@mdx-js/mdx` produces JSX configured for
`@lessjs/core`, and the result enters the same VNode and `renderDsd()` pipeline
as `.tsx` routes. There is no second HTML renderer, no runtime MDX parser, and
no React provider dependency.

```ts
import { compileMdx } from '@lessjs/content/mdx';

const mod = await compileMdx(source, { filepath: 'post.mdx' });
```

```ts
import { lessMdx } from '@lessjs/adapter-vite/plugin-mdx';

export default {
  plugins: [lessMdx()],
};
```

### Dependency Closure

The lockfile was regenerated after stale dependency state was removed:

- `playwright` resolves to `1.59.1`
- `playwright-core` resolves to `1.59.1`
- `@mdx-js/mdx` and `@mdx-js/rollup` resolve to `3.1.1`

### Package Alignment

All 19 packages are published as one coherent `0.28.3` line, with internal JSR
imports moved to `^0.28.3`.

## Migration

Existing `.tsx` routes and DSD components do not need changes. To use MDX,
add `.mdx` content and include the adapter plugin, or use `compileMdx()`
directly from `@lessjs/content/mdx` in custom content workflows.

The only compatibility-sensitive migration is inherited from v0.28.2:
module-level adapter helpers were removed from `@lessjs/core`; use
`getDefaultRegistry()` or `createAdapterRegistry()`.
