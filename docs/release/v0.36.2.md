# v0.36.2 Release Note

## Summary

v0.36.2 is the SSG bridge migration patch. It moves the Vite-free static render
engine and HTML post-processing helpers into `@openelement/ssg`, while
`@openelement/adapter-vite` continues to own Vite bundle construction and plugin
orchestration.

## Changes

- `@openelement/ssg` now exports `ssgRender()`, `resolveDynamicRoutePath()`,
  SSG render types, and postprocess helpers.
- `adapter-vite` delegates the Phase 3 SSG render step to `@openelement/ssg`.
- Existing adapter-vite SSG imports remain compatible through re-exports and a
  thin wrapper.
- ISR manifest output now uses `createIsrCacheKey()` and records `path`,
  `revalidate`, `cacheKey`, and `params`.
- `FileIsrCache` has real file-system persistence coverage.
- `renderDsdStream()` is verified through Web `Response` consumption.
- v0.36.2 AutoFlow evidence records `totalCellsAttempted: 1`,
  `totalCellsMerged: 1`, and `firstPassRate: 1`.

## Scope Boundary

This release does not move Vite SSR bundle construction, route scanning, virtual
entry generation, generated data resolution, or plugin orchestration out of
adapter-vite. Complete migration and Server/Data/UI/starter/Hub product closure
remain v0.37.0 work.
