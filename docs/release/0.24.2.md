# v0.24.2: JSX+Signal Component Model & Remove Old Template API

> **Released**: 2026-05-28
> **ADR**: ADR-0057
> **Previous**: v0.23.0

## Summary

v0.24.2 introduces the JSX+Signal component model per ADR-0057 and removes the old `html` tagged template API entirely. All components now use JSX + `static props` + Signal, with zero backward-compatibility overhead. Four infrastructure fixes were required to get the full JSX pipeline (build â†’ render â†’ reactivity â†’ publish) working end-to-end.

## What Changed

### Removed

- `html` tagged template and all helpers (`classMap`, `when`, `choose`, `repeat`, `ref`, `unsafeHTML`)
- `@prop()` decorator
- `TemplateResult`, `PropertyOptions`, and related internal types from public API
- Template-specific rendering functions (`renderTemplateToString`, `applyRuntimeTemplateBindings`, etc.)

### Added

- JSX `IntrinsicElements` type declarations for TypeScript
- Signal auto-unwrap in `renderToString` (JSX expressions `{signal}` auto-resolve `.value`)
- `renderToDom` integration in `_renderIntoShadowRoot` for proper event binding
- **`@openelement/signal` as explicit dependency of `@openelement/core`**: Core now imports `effect()` from signals for VNode signal tracking. Publish order adjusted: signals (5) â†’ core (6).

### Fixed

#### JSX Infrastructure Fixes

| # | Bug                                                           | Root Cause                                                                                                                                                                                                                                  | Fix                                                                                                                                                            |
| - | ------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1 | Island components render `[object Object]`                    | `build-client.ts` and `build-ssg.ts` internal `viteBuild()` calls use `configFile: false`, ignoring user's `vite.config.ts` esbuild JSX config. esbuild defaults to classic `React.createElement`, producing objects DsdElement can't read. | Added `esbuild: { jsx: 'automatic', jsxImportSource: '@openelement/core' }` to both internal build configs.                                                    |
| 2 | SVG icons disappear after hydration                           | `renderToDom` uses `document.createElement()` for all elements, placing SVG elements (circle, line, path, svg) in HTML namespace where browsers won't render them.                                                                          | Added `SVG_TAGS` set (50+ elements) and `createElementForTag()` helper that routes SVG elements to `document.createElementNS()`.                               |
| 3 | Theme toggle doesn't switch / theme lost on navigation        | `_renderIntoShadowRoot`'s VNode branch renders DOM but never subscribes to signals. TemplateResult path has `_subscribeTemplateSignals()`; VNode path had no equivalent.                                                                    | Wrapped `render()` in an alien-signals `effect()` that auto-tracks signal accesses and re-renders DOM on changes. | 4 | `graph:check` fails: core imports signals without declaration | `dsd-element.ts` imports `effect` from `@openelement/signal` but `packages/core/deno.json` didn't declare the dependency.                                                                                                                   | Added `@openelement/signal` to core's imports; moved signals before core in CI publish order.                                                                  |

#### Pre-release Fixes

- **VNode event binding**: `_renderIntoShadowRoot` previously used `renderToString` for VNode, which silently dropped `onClick`/`onInput` handlers. Now uses `renderToDom` with `addEventListener`.
- **SSG package resolver `.tsx` subpath resolution**: `resolveVirtualLessPackageRelative` in `ssg-package-resolver.ts` blindly appended `.ts` to all relative imports, causing `.tsx` UI components to resolve to non-existent `.ts` files. Added `resolveSourcePathExtension` that consults `LESSJS_EXPORT_FILES`.
- **JSX DOM renderer signal child unwrap**: `renderToDom` now auto-unwraps Signal values in children (matching `renderToString` behavior).
- **`www` stale `.js` search imports**: All route files and `_renderer.ts` updated to `.tsx`.
- **`@openelement/runtime` missing `LessRenderer` export**: Added re-export.
- **Generated project build test alias drift**: Updated all UI component aliases to `.tsx` and added `jsx-runtime` subpath aliases.

## Migration Guide

### From `html` tagged templates to JSX

```tsx
// Before (v0.24.1, deprecated):
import { DsdElement, html } from '@openelement/core';
class MyButton extends DsdElement {
  render() {
    return html`
      <button class="btn" @click="${this._handle}">Click</button>
    `;
  }
}

// After (v0.24.1):
import { DsdElement } from '@openelement/core';
class MyButton extends DsdElement {
  render() {
    return <button className='btn' onClick={this._handle}>Click</button>;
  }
}
```

### From `@prop()` to `static props`

```tsx
// Before:
class MyComp extends DsdElement {
  @prop()
  count = 0;
}

// After:
class MyComp extends DsdElement {
  static props = { count: Number };
}
```

### From `classMap`/`when`/`repeat`

```tsx
// classMap({ active: isActive }) â†’ className={isActive ? 'btn active' : 'btn'}
// when(cond, () => a, () => b) â†’ {cond ? <A /> : <B />}
// repeat(items, (item) => x) â†’ {items.map(item => <Item key={item.id} />)}
```
