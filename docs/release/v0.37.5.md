# v0.37.5 Release Note

Protocol-First Runtime Architecture

## Highlights

- `@openelement/protocols` now owns the first accepted replacement boundaries
  for renderer, component adapter, runtime adapter, route manifest, island
  metadata, cache/storage intent, signal interoperability, and data adapter
  contracts.
- Vite + Nitro are now the documented default base engine direction: Vite owns
  module graph/build/HMR, and Nitro is the planned production
  runtime/deployment engine for v0.37.6 proof.
- openElement keeps the application, component, route, render, island, and
  light/shadow semantics. Nitro is the runtime skin, not the framework API.
- The EntryDescriptor route manifest contract is accepted for the Nitro proof
  path.
- JSR visibility is no longer a version-exit gate under ADR-0097. Publish
  attempts remain best-effort telemetry and must be recorded honestly.

## Migration Notes

- Protocol type imports should prefer `@openelement/protocols/*` subpaths for
  new code.
- Existing public package surfaces keep compatibility re-exports where the
  protocol contracts moved.
- Generated starter projects now include direct `@openelement/core` and
  `@openelement/protocols/*` imports required by the runtime-free protocol
  boundaries.

## Deferred

- Nitro runtime proof is v0.37.6.
- Public API and product surface reset is v0.38.x.
- Expanded UI interactive/form coverage remains behind the protocol/runtime
  proof.
- Live JSR registry visibility and post-publish JSR consumer smoke remain
  best-effort distribution telemetry, not version-exit evidence.
