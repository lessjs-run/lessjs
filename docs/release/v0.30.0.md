# v0.30.0 - Type Safety Cleanup

Release date: 2026-06-04\
Changelog: [v0.30.0](../changelog/v0.30.0.md)\
SOP: [SOP-011](../sop/v0.30.0/SOP-011-type-safety-cleanup.md)\
Supersedes: [v0.29.4](./v0.29.4.md)

v0.30.0 is the type safety cleanup release. It eliminates `ban-types` suppressions, simplifies the render pipeline, and removes the SSR compat stub and happy-dom dependency.

## TL;DR

- 3 `ban-types` suppressions → 0.
- `render()` → `VNode | null` (all components already return JSX).
- `_SsrHTMLElementStub` deleted, happy-dom removed.
- ~60 lines of dead code deleted.
- Zero break for existing components.

## What Changed

### VNode.tag precise types

```ts
// Before
// deno-lint-ignore ban-types
tag: string | Function | symbol;

// After
export type ComponentFn = (props: Record<string, unknown>) => unknown;
export type ComponentCtor = new (...args: unknown[]) => { render(): unknown };
tag: string | ComponentFn | ComponentCtor | symbol;
```

### render() → VNode | null

```ts
// Before
render(): string | VNode { return ''; }

// After
render(): VNode | null { return null; }
```

`_renderIntoShadowRoot` simplified from 3 branches (VNode / string / fallback) to 1.

### SSR cleanup

- `_SsrHTMLElementStub` 16-line class → inline `globalThis.HTMLElement = class {}`
- `happy-dom` removed from dependencies
- 2 test files migrated out

## What Did Not Change

- `renderDsd()` still handles string returns for backward compatibility.
- Adapter protocol unchanged.
- Public API surface unchanged.
