# v0.30.0 - Architecture Contract Freeze

Release date: 2026-06-04\
Changelog: [v0.30.0](../changelog/v0.30.0.md)\
ADR: [ADR-0080](../adr/ADR-0080-architecture-contract-freeze.md)\
SOP: [v0.30.0](../sop/v0.30.0/README.md)\
Supersedes: [v0.29.6](./v0.29.6.md)

v0.30.0 freezes the LessJS architecture contract before the next UI Shell work.
The release removes the last native string-render compatibility path, aligns the
renderer around VNode/RenderNode, and adds gates that make future drift visible.

## TL;DR

- Native components now render `VNode | null`.
- `renderDsd()` rejects non-VNode native output instead of silently accepting
  strings or adapter-native template objects.
- `trustedHtml` is the explicit trust-boundary name; `rawHtml` is gone from the
  current API surface.
- Route/nav metadata extraction is AST-backed.
- Hub CLI/snapshot helpers and adapter hydration event binding have one shared
  implementation.
- `arch:check` and expanded `graph:check` make the v0.30.0 cleanup gate-proved.
- All 19 packages are `0.30.0` with internal JSR ranges on `^0.30.0`.

## Migration

Native LessJS components must return JSX/VNode:

```tsx
import { DsdElement, type VNode } from '@openelement/core';

class MyCard extends DsdElement {
  render(): VNode {
    return (
      <section class='card'>
        <slot />
      </section>
    );
  }
}
```

Do not return HTML strings from native `render()`:

```diff
- render(): string {
-   return '<section class="card"><slot></slot></section>';
- }
+ render(): VNode {
+   return <section class='card'><slot /></section>;
+ }
```

For already-trusted HTML, use an explicit trust marker:

```tsx
<div innerHTML={htmlFromTrustedPipeline} trustedHtml />;
```

Without `trustedHtml`, `innerHTML` is treated as text.

## Architecture Notes

v0.30.0 treats completed SOPs as claims that must be backed by commands. The new
`arch:check` gate byte-reads every tracked file, checks current docs and source
for removed APIs, verifies duplicate-helper cleanup, enforces the reviewed
production type-escape allowlist, and rejects source-regex metadata parsing
regressions.

The UI Shell, Ocean-Island, and `@openelement/ui/css` work stays deferred to v0.31.0
so that product-facing shell work starts from a clean renderer and metadata
foundation.
