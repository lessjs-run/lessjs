# Release v0.27.0

**The "One Tree, One Pass" release.**

---

## What changed

This release eliminates the **dual tree traversal** in SSR rendering. Previously, VNode trees were serialized to HTML strings, then re-parsed with `parse5` to find nested custom elements. Now, a single `renderDsdTree()` traversal handles everything inline.

The public API surface also gets streamlined: three render functions (`renderDsd`, `renderDsdByName`, `renderNestedDsd`) collapse into one polymorphic `renderDsd(input, props?)`. JSX factories (`jsx`, `jsxDEV`, `jsxs`) leave the root export and stay on the `jsx-runtime` subpath where they belong.

### By the numbers

- **-335 lines** net code removed
- **-1 external dependency** (parse5)
- **2 ADRs** (ADR-0071, ADR-0072)
- **12 SOPs** in v0.27.0 cycle
- **953 tests** passing

## Breaking changes

```diff
- import { renderDsdByName } from '@lessjs/core'
+ import { renderDsd } from '@lessjs/core';  renderDsd('tag', props)

- import { jsx, jsxs } from '@lessjs/core'
+ import { jsx, jsxs } from '@lessjs/core/jsx-runtime'

- dependencies: { "parse5": "npm:parse5@7.0.0" }
+ // parse5 no longer needed
```

## New public API

```ts
// @lessjs/core — root
import { renderDsd, DsdElement, Fragment, renderToString } from '@lessjs/core'

// @lessjs/core/jsx-runtime — compiler protocol (Deno jsxImportSource)
import { jsx, jsxDEV, jsxs, For, Show } from '@lessjs/core/jsx-runtime'
```
