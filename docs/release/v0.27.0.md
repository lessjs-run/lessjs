Release date: 2026-06-01\

## TL;DR

**消灭了 SSR 渲染管线的双遍树遍历，删除 parse5 外部依赖，统一对外 API 表面。**
Net -293 lines, -1 external dependency, 3 production bugs root-fixed.

## What Changed

### Pipeline: "One Tree, One Pass"

v0.26.1 的 DSD 渲染需要三遍树遍历：

```
VNode → renderToString → HTML string → parse5 → AST → renderNested → HTML
```

v0.27.0 合并为一遍：

```
VNode → renderDsdTree → HTML
```

`renderDsdTree()` 在遍历 VNode 树时遇到 Custom Element 就内联调用 `renderDsd()`，不再需要先序列化为 HTML 再用 parse5 重新解析。

| Before                                              | After             |
| --------------------------------------------------- | ----------------- |
| `renderToString()` + `renderNestedCustomElements()` | `renderDsdTree()` |
| parse5 (npm:parse5@7.0.0)                           | — (deleted)       |
| 3 passes over the tree                              | 1 pass            |
| ~745 lines (render-nested.ts + renderToString)      | ~540 lines (-28%) |

### API: Three Into One

| Removed                                    | Replacement                                            |
| ------------------------------------------ | ------------------------------------------------------ |
| `renderDsdByName(tag, props)`              | `renderDsd(tag, props)` — 多态输入                     |
| `renderNestedDsd(node)`                    | 内部函数，不再导出                                     |
| `import { jsx } from '@lessjs/core'`       | `import { jsx } from '@lessjs/core/jsx-runtime'`       |
| `import { Show, For } from '@lessjs/core'` | `import { Show, For } from '@lessjs/core/jsx-runtime'` |

New unified API:

```ts
// @lessjs/core — root
import { DsdElement, Fragment, renderDsd, renderToString } from '@lessjs/core';

// @lessjs/core/jsx-runtime — compiler protocol
import { For, jsx, jsxDEV, jsxs, Show } from '@lessjs/core/jsx-runtime';
```

### Architecture: Ocean + Island Signal Model

确立三层架构：

```
Component (DsdElement) → Signal (alien-signals) → DOM (DSD + effects)
```

- **Ocean**: 纯 DSD 静态页面，零 JS
- **Island**: signal 驱动的交互组件，~2KB JS
- **Hydration markers**: `data-signal` / `data-signal-html` / `data-signal-attr` / `data-signal-class`
- `_hydrateSignals()` 替代原来的 `_walkAndBind()`，从位置匹配改为显式标记匹配

### Production Bugs Fixed

| Bug                      | Root Cause                                      | Fix                                                  |
| ------------------------ | ----------------------------------------------- | ---------------------------------------------------- |
| Sidebar 消失             | `<less-layout>` 走字符串拼接而非 `renderDsd()`  | `__wrapAppShell` → `__renderAppShell` 调 `renderDsd` |
| Search `[object Object]` | `routeNode`(VNode) 被 `String()` 隐式转换       | VNode 先经 `renderDsdTree()` 渲染再拼接              |
| Search theme 不跟随      | `<dialog>::backdrop` 不继承 shadow DOM CSS 变量 | `<dialog>` → `<div>`                                 |

### Brand Cleanup (SOP-009 Phase A)

- `data-less-e` → `data-eid`
- `__LESS_CLIENT_ONLY_TAGS__` → `__CLIENT_ONLY_TAGS__`
- `__LESS_HEAD_EXTRAS__` → `__HEAD_EXTRAS__`
- `LessJS ERROR` → `Render Error`
- `LessJS Island Markers` → `Island Markers`

### Dead Code Removal

- `packages/core/src/render-nested.ts` — 452 lines deleted
- `packages/core/src/dom-simulation.ts` — 236 lines deleted (happy-dom experiment, superseded by Playwright snapshots)
- `__wrapAppShell` — removed; `__renderAppShell` replaces it

### Other Changes

- **SSR Manifest Nav Fix** (SOP-007): `__ROUTE_MANIFEST__` injection removed; less-layout uses `virtual:less-nav` virtual module
- **Signal-Native www Islands**: 0 hand-written `addEventListener` / `innerHTML=` / `document.createElement` in island components
- **Event binding**: `data-on-click/input/change/submit/keydown` markers via `_bindEvents()`
- **Router**: `marked` dependency moved from `esm.sh` bare URL to `npm:marked@15.0.4`

## Breaking Changes


## Migration Guide

### 1. Update render function call sites

```ts
// Before
import { renderDsdByName } from '@lessjs/core';
await renderDsdByName('less-layout');

// After
import { renderDsd } from '@lessjs/core';
await renderDsd('less-layout');
```

### 2. Update JSX import source

```ts
// Before
import { jsx, jsxs } from '@lessjs/core';

// After
import { jsx, jsxs } from '@lessjs/core/jsx-runtime';
```

### 3. Remove parse5 dependency

```json
// deno.json — remove this line
{ "imports": { "parse5": "npm:parse5@7.0.0" } }
```

### 4. Update page component patterns

```tsx
// Before: page wraps itself in <less-layout>
export default class HomePage extends DsdElement {
  render() {
    return (
      <less-layout>
        <h1>Hello LessJS</h1>
        <less-counter client:idle />
      </less-layout>
    );
  }
}
```

```tsx
// After: pages only declare their content
export default class HomePage extends DsdElement {
  render() {
    return <h1>Hello LessJS</h1>;
  }
}
```

The App Shell (`<less-layout>`) is now applied automatically by the Builder.
