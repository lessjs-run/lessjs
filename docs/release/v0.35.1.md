# v0.35.1 Release Note: Deduplication & ADR-0079 Debt Closure

CHANGELOG: [CHANGELOG.md](../../CHANGELOG.md)\
SOP: [docs/sop/v0.35.1/README.md](../sop/v0.35.1/README.md)\
Issue: [#47](https://github.com/open-element/openelement/issues/47)\
ADR: ADR-0079

v0.35.1 is a fix patch that closes ADR-0079's four unfulfilled deduplication
commitments and addresses the highest-severity items from the 2026-06-06
full-project audit (32 items total, 11 P0 resolved).

## Highlights

### Bug Fixes

- **Hub validatorVersion drift**: `hub-submit.ts` hardcoded `'0.19.0'`, constants
  had `'0.33.0'`. Now uses `VALIDATOR_VERSION` import.
- **Version fallback removed**: `FALLBACK_OPENELEMENT_VERSION` was a silent
  0.33.0 fallback. Now throws on failure (fail-fast per Lesson 25).
- **Constants merged**: `VALIDATOR_VERSION` and `openElement_UI_VERSION` unified.

### Deduplication

- **CSS merge**: `docs-page-styles.ts` (147L) deleted. Single consumer
  switched to `page-styles.ts` (291L). ~80 lines of CSS deduplicated.
- **HTML escape dedup**: `ui/src/shared/escape.ts` deleted. All 6 UI components
  now import `escapeHtml`/`escapeAttr` from `@openelement/core`.
- **Test fixtures shared**: `client-only-cem.json` and `invalid-cem.json`
  moved to `tools/test-fixtures/`.

### Infrastructure

- Added `TAG_VALIDATION_ERROR` to `core/src/errors.ts` ErrorCode.
- Created `tools/lib/` and `tools/test-fixtures/` directories.
- AutoFlow skill updated with complete 12-step release workflow.
- Pre-commit hook properly installed and enforcing 6 checks.
