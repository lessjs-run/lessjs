# v0.37.6 Release Note

Vite + Nitro Runtime Proof

## Highlights

- openElement now has a real Nitro proof fixture that mounts the universal
  request handler into generated Nitro output.
- The Node proof builds `node-server` output, starts the generated server, and
  verifies openElement route/render behavior through Web `Request` and
  `Response` boundaries.
- The Cloudflare Workers proof builds `cloudflare-module` output and verifies
  the generated server entry, wrangler config, public asset, route/render
  markers, and Nitro route-rule cache markers.
- Static pages preserve the zero-framework-JS default unless explicit islands
  or client-only behavior are requested.
- `load()`, layouts, redirects, not-found, errors, API routes, public assets,
  island chunks, and `/isr` cache intent are covered by the proof harness.
- Nitro owns runtime delivery, platform output, serving, and supported cache
  primitives. openElement keeps application, component, route, render, island,
  DSD/shadow, light DOM opt-in, and cache intent semantics.
- JSR visibility remains best-effort distribution telemetry under ADR-0097 and
  is not a version-exit gate.

## Migration Notes

- Users should not author Nitro routes to use openElement route/render
  semantics. Nitro remains the deployment/runtime layer behind the adapter.
- Static routes continue to emit no framework JavaScript unless explicit island
  hydration or client-only behavior is requested.
- Cache intent should be expressed through openElement semantics and mapped by
  the adapter to Nitro route rules or supported runtime primitives.

## Release Truth

- `dev` non-JSR CI must pass before merge or sync to `main`.
- `main` non-JSR CI must pass before tag and GitHub release publication.
- Local or CI JSR publish should be attempted as best-effort telemetry, but
  JSR publish success and registry visibility do not block version exit.

## Deferred

- Public API and product surface reset is v0.38.x.
- Expanded starter, deploy, and consumer hardening remains on the path to the
  v0.39.0 full-stack framework RC.
