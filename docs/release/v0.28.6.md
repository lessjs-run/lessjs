# v0.28.6 - Build and Trust Boundary Cleanup

Release date: 2026-06-03\
Changelog: [v0.28.6](../changelog/v0.28.6.md)\
Supersedes: [v0.28.5](./v0.28.5.md)

v0.28.6 is the cleanup release between AppShell and the renderer IR line. It
does not add a new user-facing framework feature. It tightens the release
pipeline and removes ambiguity around metadata and raw HTML trust boundaries.

## What Changed

- Root release tasks are now package-graph driven and cover all 19 workspace
  packages.
- Hub client-only metadata is loaded through a shared module boundary. Missing
  generated data is optional; present but broken generated data is a build
  error.
- Local island metadata now uses TypeScript AST extraction instead of regex.
- Optional adapter/content/i18n stubs are centralized.
- `rawHtml` is explicit trusted HTML, not a sanitizer promise.
- Adapter registry default reset no longer wipes named adapters.

## Why It Matters

This release makes local release evidence match the real workspace graph. It
also makes the security boundary honest: LessJS core escapes text by default,
but it does not pretend to sanitize arbitrary HTML.

## Verification Target

The required release evidence is:

- `deno task fmt:check`
- `deno task lint`
- `deno task typecheck`
- `deno task test`
- `deno task build`
- `deno task dsd:check-report`
- `deno task publish:dry-run`

This note does not claim live publication until the package publish workflow has
completed.
