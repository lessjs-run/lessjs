# v0.40.0 Product-Line Cleanup Evidence

## Summary

v0.40.0 narrows the current product line to 12 packages and removes historical
Hub, RPC, CEM, compat-check, interop adapters, standalone runtime, standalone
style-sheet, and standalone SSG surfaces from the active repo shape. Public
package names are singular (`element`, `protocol`, `signal`), active code
enforces 0 explicit `any`, and AutoFlow2 historical implementations are removed
in favor of the AutoFlow3 control plane.

## Evidence Targets

- Package graph: 21 packages reduced to 11 current packages.
- Workflows: 8 workflows reduced to 4 active workflows.
- Docs: active docs limited to current truth, ADR, roadmap/status, release
  evidence, and archive index; `.github` templates no longer reference SOP/NextVersion.
- Hub: `packages/hub`, root `hub-index`, public Hub data, and Hub routes removed
  from the current product line.
- CEM/compat: SSG keeps only internal CEM compatibility code needed for package
  detection inside `@openelement/adapter-vite`.
- AutoFlow3: CI and local gates use AutoFlow3 entry points only.
- AutoFlow3 patch automation: v0.40.0 cleanup-train release execution requires
  approval id `ADR-0105/v0.40.x-cleanup-train`.
- Type safety: `type-safety:check` gate asserts 0 explicit `any` in active
  TS/TSX across packages, tests, tools, and www active code.
