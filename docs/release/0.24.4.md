# v0.24.4: API Naming Convention — Web Platform Style

> **Released**: 2026-05-29
> **Previous**: v0.24.3
> **SOP**: SOP-001 (docs/sop/v0.24.4/SOP-001-api-naming-convention.md)

## Summary

v0.24.4 applies the Web Platform naming convention across the entire public
API surface. Function names follow `verbNoun`, acronyms use `PascalCase`
(`Dsd` not `DSD`), and brand prefixes are removed. This is a mechanical
rename with no behavioral changes — all 944 tests pass identically.

- import { less } from '@lessjs/adapter-vite';

- less({ routesDir: 'app/routes', island: { upgradeStrategy: 'idle' } })
+ lessPipeline({ routes: { dir: 'app/routes' }, island: { dir: 'app/islands', upgradeStrategy: 'idle' } })

## What's New

### lessPipeline() Declarative API

- `lessPipeline(config: PipelineConfig): Plugin[]` — single public build entry
- Config shape: `{ routes, i18n, output, island, pwa, viewTransition, headExtras }`
- Maps to `FrameworkOptions` internally — all existing build code reused

### SSG DSD Output Restored

- **Root cause**: `import()`-based tagName detection silently failed for routes
  with `virtual:less-nav` dependencies
- **Fix**: Regex-based source scan on `export const tagName` — zero module execution
- DSD markers in generated pages: 3 (polyfill only) → 5 (real `<template shadowrootmode>`)

### SSG Smoke Test Hardening

- Removed `dist/` cache skip — test always does fresh build
- Prevents stale output from masking regressions across CI runs

### customElements Duplicate Registration Fix

- Added `!customElements.get(tagName)` guard to 4 island components
- Prevents `Failed to execute 'define'` errors during SSR bundle loading

### Internal Cleanup

| Before                 | After                                       |
| ---------------------- | ------------------------------------------- |
| `less()` public API    | Removed — `lessPipeline()` only             |
| `_dsdHydrated` field   | Removed — unified `_renderOrHydrate()` path |
| `__lessBindDone` guard | `__bindEventsDone`                          |
| tagName detection      | Regex source scan (was `import()`)          |
