# v0.24.3: Consolidation, Renderer Hardening & Signal Unwrap

> **Released**: 2026-05-29
> **Previous**: v0.24.2
> **SOP**: v0.24.3

## Summary

v0.24.3 is a consolidation release. It hardens the JSX renderer infrastructure,
extracts signal utilities into a neutral module, adds direct renderer tests,
and aligns the public API surface to the new JSX+Signal component model.

As of v0.24.3, **JSX + static props + Signal is the only supported component
authoring model**. Legacy `html` tagged templates and `@prop()` decorator are
removed from public API.

## Changes

### Signal Unwrap Hardening

- **Neutral `signal-like.ts` module**: `isSignalLike()` and `unwrapSignalLike()`
  extracted from `template.ts`. JSX renderers (`jsx-render-dom.ts`, `jsx-render-string.ts`)
  no longer depend on the legacy template module.
- **CSR attribute unwrap**: `applyProps()` now auto-unwraps Signal values in all
  attributes. `<div title={signal('tooltip')}>` correctly renders `title="tooltip"`.
- **SSR attribute unwrap**: `serializeAttrs()` auto-unwraps Signal values before
  HTML serialization.
- **Style object unwrap**: Nested Signal values in `style` objects are unwrapped
  before assignment/serialization in both CSR and SSR.
