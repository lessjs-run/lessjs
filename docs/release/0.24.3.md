# v0.24.3: Consolidation, Renderer Hardening & Signal Unwrap

> **Released**: 2026-05-29
> **Previous**: v0.24.2
> **SOP**: v0.24.3

## Summary

v0.24.3 is a consolidation release. It hardens the JSX renderer infrastructure,
extracts signal utilities into a neutral module, adds direct renderer tests,
aligns the public API surface to the new JSX+Signal component model, and
eliminates 12 cross-package type duplications.

As of v0.24.3, **JSX + static props + Signal is the only supported component
authoring model**. Legacy `html` tagged templates and `@prop()` decorator are
removed from public API.

## Changes

### Signal Unwrap Hardening

- **Neutral `signal-like.ts` module**: `isSignalLike()` and `unwrapSignalLike()`
  extracted from `template.ts`. JSX renderers (`jsx-render-dom.ts`, `jsx-render-string.ts`)
  no longer depend on the legacy template module.
- **Duplicate `SignalLike`/`isSignalLike` removed from `template.ts`**: The legacy
  definitions that were left behind after extraction are cleaned up. `SignalLike.value`
  changed to `readonly` for consistency.
- **CSR attribute unwrap**: `applyProps()` now auto-unwraps Signal values in all
  attributes. `<div title={signal('tooltip')}>` correctly renders `title="tooltip"`.
- **SSR attribute unwrap**: `serializeAttrs()` auto-unwraps Signal values before
  HTML serialization.
- **Style object unwrap**: Nested Signal values in `style` objects are unwrapped
  before assignment/serialization in both CSR and SSR.

### New Renderer Tests (38 tests)

- `renderToDOM`: text/Signal children, SVG namespace (circle, line, path, g, rect, svg),
  onClick binding, AbortController cleanup, Fragment, className, multiple children
- `renderToString`: Signal children, onClick exclusion, className→class, htmlFor→for,
  boolean attrs, style serialization, SVG attrs, Fragment, void elements

### Cross-Package Type Deduplication

12 types/utilities that were duplicated across 2–3 packages are now
canonically defined in `@lessjs/core`:

| Type                                                                                                                           | Removed from                                                                                                                         |
| ------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| `SignalLike`, `isSignalLike`                                                                                                   | `core/template.ts` (duplicate with `core/signal-like.ts`)                                                                            |
| `ManifestDecision`, `SsrAdmissionDecision`                                                                                     | `compat-check/types.ts`, `adapter-vite/entry-descriptor.ts`                                                                          |
| `ValidationResult`, `ValidationError`, `ValidationWarning`, `ValidationDiagnostic`, `ValidatedTag`, `ManifestValidationReport` | `compat-check/types.ts`                                                                                                              |
| `ComponentLayer`, `HydrationStrategy`                                                                                          | `cem/types.ts`, `compat-check/types.ts`                                                                                              |
| `StrategySource`                                                                                                               | `compat-check/types.ts`, `adapter-vite/entry-generators.ts`                                                                          |
| `CompatibilityTier`                                                                                                            | `cem/cem-parser.ts` (was a local `type` alias)                                                                                       |
| `isValidTagName`                                                                                                               | `cem/cem-parser.ts`, `compat-check/compatibility.ts` (two different impls → unified in `core/tag-utils.ts` with reserved-name check) |

### Nav Alias Cleanup

- Replaced 50+ usages of deprecated aliases across 34 route files:
  `filterFrameworkNav` → `filterDocsNav`, `filterEngineNav` → `filterArchitectureNav`,
  `filterRegistryNav` → `filterHubNav`
- Removed the 3 legacy alias definitions from `www/app/utils/nav-filter.ts`

### Dependencies Changed

- `@lessjs/cem` now has 1 internal dependency (`@lessjs/core`, was 0):
  enables shared type imports for `ComponentLayer`/`HydrationStrategy`/`CompatibilityTier`/`isValidTagName`

### CLI & Data Organization (SOP-003)

- **CLI rename**: `@lessjs/hub/cli/less-add` → `@lessjs/hub/cli/less-install-guide` — eliminates naming collision with `@lessjs/compat-check/cli/less-add`
- **Data relocation**: `_hub-data-full.ts` (3867L auto-generated) moved from `www/app/routes/registry/` to `www/app/data/registry/hub-data.ts`
- **Changelog**: `changelog.ts` (was 1956L inline HTML, now 200L) reads `CHANGELOG.md` at build time. Self-contained markdown renderer, zero external deps. Single source of truth.

### Public API

- `isSignalLike`, `unwrapSignalLike`, and `SignalLike` type exported from `@lessjs/core`
- `isValidTagName` exported from `@lessjs/core`
- `StrategySource` now exported from `@lessjs/core` (was defined but not exported)
- Signal utilities import path: `import { isSignalLike, unwrapSignalLike } from '@lessjs/core'`
