v0.29.6 closes architecture debt across the v0.29.x cleanup arc. Eliminates 17 type escapes, deletes 40+ lines of duplicate code, centralizes hub constants, and winds down the `render(): string` path.

## TL;DR

- `render()` → `VNode | null` with deprecation compat for strings
- 17 type escapes eliminated — WeakMap replaces property stamping
- 40+ lines of duplicate code deleted
- Hub versions centralized in `constants.ts`
- 1297 tests, 0 failures

## What Changed

### render() → VNode | null

```ts
// Before
render(): string | VNode { return ''; }

// After
render(): VNode | null { return null; }
```

String returns still work in SSR (`renderDsd` compat path). CSR path is VNode-only.

### WeakMap replaces property stamping

```ts
// Before: stamping metadata on constructor (3 type escapes)
(componentClass as unknown as Record<string, unknown>).__island = true;

// After: type-safe (0 escapes)
export function getIslandMeta(ctor) {
  return _islandMeta.get(ctor);
}
```

### Precise types

```ts
// Before
const renderFn = children[0] as unknown as ((item, idx) => unknown);

// After
export type RenderFn = (item: unknown, idx: number) => unknown;
const renderFn = children[0] as RenderFn;
```

## Breaking Changes

- `render()` returns `VNode | null`. String returns receive deprecation warning. SSR path still handles strings.
