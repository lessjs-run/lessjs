v0.35.0 delivers all three layers of AutoFlow2 mechanical autonomy in one
version: CI-blocking Harness Gate, autonomous Cell Execution with TDD and
AI cross-review, and multi-version Evolution Loop. Product development resumes
at v0.36 under full AutoFlow governance.

## Highlights

### Layer 1 — Harness Gate

- `deno task autoflow:check` — CI-invariant verification, exit 0/1.
- 8 Pnueli temporal invariants enforced in pre-commit hook and CI.
- 12-state Harel cell lifecycle state machine with 3 orthogonal dimensions.
- Durable Execution event sourcing ledger with Kahn deterministic communication.

### Layer 2 — Cell Execution

- `deno task autoflow:evolve --dry-run` — detect drift, build DAG, simulate.
- Three-phase cell execution: testgen → implement → review (ADR-0087).
- Wave-based parallel scheduling with cascade cancellation.
- Git branch isolation per cell (Kahn principle).

### Layer 3 — Evolution Loop

- `deno task autoflow:health` — agent diagnostic.
- AlphaEvolve fitness + SWE-bench 12-dimensional quality metrics.
- Multi-cycle evolution trend tracking (Lehman complexity).
- `.workbuddy/skills/autoflow/SKILL.md` — AI agent project driver.

## Defense Mechanisms

Pre-commit hook enforces 8 mandatory checks before any commit:
1. `deno fmt --check .` (full repo)
2. `deno lint .`
3. `deno check` (key entry points)
4. `deno task arch:check`
5. `deno task graph:check`
6. `deno task docs:check-strategy`
7. `deno task docs:check-current`
8. `autoflow:check` (8 Pnueli invariants)

CI (lint.yml) additionally runs `autoflow:check` on every push.
`.workbuddy/skills/autoflow/SKILL.md` drives agent behavior with mandatory
`autoflow:health` on every session start.
