# v0.29.5 - Type Safety & SSR Cleanup

Release date: 2026-06-04\
Changelog: [v0.29.5](../changelog/v0.29.5.md)\
SOP: [SOP-011](../sop/v0.29.5/SOP-011-type-safety-cleanup.md)\
Supersedes: [v0.29.4](./v0.29.4.md)

v0.29.5 is the type safety and SSR cleanup release. It eliminates `ban-types` suppressions, tightens component types, removes the SSR stub class, deletes the Happy DOM dependency, and replaces symbol-keyed prop storage with type-safe WeakMap.

## TL;DR

- 3 `ban-types` suppressions → 0
- 25 `as unknown as` type escapes eliminated
- `render()` → `VNode | null`
- `_SsrHTMLElementStub` deleted
- `happy-dom` removed
- 5 `Symbol.for()` → 5 `WeakMap`
- ~80 lines deleted

## What Changed

### VNode.tag precise types

```ts
// Before
// deno-lint-ignore ban-types
tag: string | Function | symbol;

// After
export type ComponentFn = (props: Record<string, unknown>) => unknown;
export type ComponentCtor = new (...args: unknown[]) => { render(): unknown };
tag: string | ComponentFn | ComponentCtor | symbol;
```

### prop.ts WeakMap refactor

```ts
// Before: symbol-keyed storage (17 type escapes)
(instance as unknown as _PropSignalsAccessor)[PROP_SIGNALS] = sigMap;

// After: WeakMap (0 type escapes)
_propSignals.set(instance, sigMap);
```

### render() simplification

```ts
// Before
render(): string | VNode { return ''; }

// After
render(): VNode | null { return null; }
```

### SSR cleanup

- `_SsrHTMLElementStub` → inline conditional base
- `happy-dom` removed from dependencies
- 2 test files removed

## Breaking Changes

- `render()` now returns `VNode | null`. All components already use JSX; string backwards-compat preserved.
- `onRenderError()` returns `VNode | null`.
- `happy-dom` removed from dependencies.
