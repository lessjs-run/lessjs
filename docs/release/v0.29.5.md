# v0.29.5 - Type Safety & SSR Cleanup

Release date: 2026-06-04\
Changelog: [v0.29.5](../changelog/v0.29.5.md)\
SOP: [SOP-011](../sop/v0.29.5/SOP-011-type-safety-cleanup.md)\
Supersedes: [v0.29.4](./v0.29.4.md)

v0.29.5 eliminates `ban-types` suppressions, replaces symbol-keyed prop storage with type-safe WeakMap, deletes the Happy DOM dependency, and simplifies the SSR stub class.

## TL;DR

- 3 `ban-types` suppressions → 0
- 21 `as unknown as` type escapes eliminated
- 5 `Symbol.for()` → 5 `WeakMap`
- `happy-dom` removed from dependencies
- SSR stub simplified to anonymous class

## What Changed

### VNode.tag precise types

```ts
// Before
// deno-lint-ignore ban-types
tag: string | Function | symbol;

// After
export type ComponentFn = (props: Record<string, unknown>) => unknown;
export type ComponentCtor = new (...args: unknown[]) => { render(): unknown };
tag: string | ComponentFn | ComponentCtor | symbol;
```

### prop.ts WeakMap refactor

```ts
// Before: symbol-keyed storage (17 type escapes)
(instance as unknown as _PropSignalsAccessor)[PROP_SIGNALS] = sigMap;

// After: WeakMap (0 type escapes)
_propSignals.set(instance, sigMap);
```

### SSR stub cleanup

```ts
// Before: named class with globalThis side effect
const _SsrHTMLElementStub = class { hasAttribute() { return false; } ... };
(globalThis.HTMLElement = _SsrHTMLElementStub as unknown as typeof HTMLElement);

// After: anonymous inline class
const _Base = typeof HTMLElement !== 'undefined'
  ? HTMLElement
  : (class { hasAttribute(_: string) { return false; } ... } as unknown as typeof HTMLElement);
if (typeof HTMLElement === 'undefined') (globalThis as Record<string, unknown>).HTMLElement = _Base;
```

## What Did Not Change

- `render()` still returns `string | VNode` — not narrowed in this release.
- No public API surface changes.
- No breaking changes.
