# v0.26.1 — DSD-First Real DOM Signal Architecture

Release date: 2026-05-30\
ADR: [ADR-0062](../adr/ADR-0062-dsd-first-rdom-signal-architecture.md)\
SOP: [SOP-002](../sop/v0.26.1/SOP-002-signal-to-css-driven-visual.md)\
Issue: [#28](https://github.com/lessjs-run/lessjs/issues/28) — Chromium DSD layout bug

## TL;DR

**修复了 ADR-0058 移除 `effect(() => render())` 导致的白屏，确立了 DSD-first RDOM 信号架构。**

```
HTML = 骨架（DSD 一次生成，render() 只调一次）
CSS  = 视觉（data-theme={signal} + CSS 选择器，零 JS 切换）
JS   = 数据 + 原子更新（signal → effect → setAttribute / textContent）
```

## Root Cause

Commit `7989e8e8` (ADR-0058) removed `effect(() => render())` from DSD hydration, assuming per-prop signal bindings sufficed. Three factors combined to cause the blank screen:

1. **Chromium DSD layout bug** — DSD shadow DOM renders correct DOM but 0×0 host bounding rect. Only full DOM replacement triggers correct layout.
2. **G1 effect leak** — per-prop effects never disposed on disconnect.
3. **G2 consumeContext dead copy** — provider updates never reached consumers.

## What Changed (v0.26.1 vs v0.26.0)

| Dimension      | v0.26.0                                 | v0.26.1                                                    |
| -------------- | --------------------------------------- | ---------------------------------------------------------- |
| Signal→DOM     | `effect(() => render())` full re-render | `_layoutWorkaroundReRender()` one-time + per-prop bindings |
| Theme toggle   | `signal.value` in render()              | `data-theme={signal}` + CSS `[data-theme]`                 |
| Effect cleanup | Manual `_signalUnsubscribers` array     | `effectScope` — one `dispose()` call                       |
| Context        | Dead copy (`signal(value)`)             | Live reference to source signal in Map                     |
| Render calls   | 1 + N per lifecycle                     | 1 (plus 1 for layout workaround)                           |
| Code size      | dsd-element.ts 575 lines                | dsd-element.ts ~535 lines (-40)                            |

## Signal System Architecture

```
alien-signals (third-party)
  ↓
@lessjs/signals (signal + computed + effect + effectScope)
  ↓                    ↓                   ↓
signal-context.ts  jsx-render-dom.ts   dsd-element.ts
(~35 lines)         applyProps          (~535 lines)
                    signal→setAttribute   lifecycle + effectScope
```

## Components Audit

9 `less-*.tsx` components — zero `signal.value` in `render()`\
2 `www/islands` components — zero `signal.value` in `render()` (after fix)\
Total migration cost: ~10 lines (theme-toggle CSS rewrite)

## Known Issues

- **Chromium DSD layout bug** — `_layoutWorkaroundReRender()` will be removed when Chrome fixes DSD layout recalculation on custom element upgrade. [Issue #28](https://github.com/lessjs-run/lessjs/issues/28)
- **G4 Show/For DSD hydration** — deferred to P2. Currently works through the one-time re-render.
- **G5 batch/untrack exposure** — alien-signals primitives exist but `@lessjs/signals` facade doesn't yet re-export them.
