# v0.38.0 Migration Notes

v0.38.0 is a product surface reset. It keeps the current package names and does
not physically remove workspace packages.

## Starter Import Map

New projects generated by `@openelement/create` no longer predeclare
`@openelement/protocols` or protocol subpath aliases in `deno.json`.

Existing projects do not need to remove those aliases immediately. They can
keep them if application code imports protocol contracts directly.

For new or cleaned projects, prefer this starter surface:

- `@openelement/app`
- `@openelement/app/vite`
- `@openelement/runtime`
- `@openelement/core`
- `@openelement/core/jsx-runtime`
- `@openelement/ui`

Add `@openelement/protocols` or a protocol subpath only when application or
advanced tooling code imports the contract directly.

## Public Docs Surface

Root docs, live website guide pages, and internal/archived package READMEs now
separate:

- product surfaces: `create`, `app`, `app/vite`, `runtime`, `core`, `ui`,
  `protocols`;
- advanced surfaces: `adapter-vite`, `content`, `i18n`, `router`, `signals`,
  `style-sheet`, and adapter packages;
- internal surfaces: `ssg`, `cem`, `compat-check`;
- archived surfaces: `hub`, `rpc`.

This is a documentation and template contract first. Physical package removal
or new package names still require ADR approval.

## RPC

`@openelement/rpc` remains published and tested, but v0.38.0 classifies it as
archived/advanced. API route docs now recommend Hono plus platform
Request/Response APIs for ordinary API routes.

## SSG

`@openelement/ssg` remains in the package graph as internal build
infrastructure. Applications should use `@openelement/app/vite` or generated
`@openelement/create` tasks instead of importing `@openelement/ssg` directly.
