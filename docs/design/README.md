# LessJS Design Notes

This directory owns design thinking for the public `www` experience.

It covers:

- site information architecture;
- layout density and navigation;
- visual language and art direction;
- interaction states and component presentation;
- how the website should demonstrate LessJS without becoming a marketing-only
  landing page.

It does not own framework architecture, package boundaries, release gates, or
runtime API decisions. Those live under `docs/arch/`, `docs/adr/`, and
`docs/sop/`.
